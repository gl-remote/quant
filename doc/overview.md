# 天勤量化交易系统

> 版本: 0.2.0-dev | 更新日期: 2026-05-25

---

## 项目定位

天勤量化交易系统是一个基于双均线交叉策略的自动化交易平台，覆盖从数据获取、策略回测到实盘执行的完整量化交易流程。系统采用"核心逻辑 + 桥接器"架构，将纯业务算法与具体交易框架解耦，确保策略逻辑在不同运行环境（回测、模拟、实盘）中保持一致。

## 核心能力

| 能力 | 说明 |
|------|------|
| **数据获取** | 从天勤量化 SDK 拉取历史 K 线数据，智能去重合并，导出为标准 CSV 格式 |
| **策略回测** | 基于 vn.py 框架的三阶段回测流水线：训练集 → 验证集 → 测试集，独立执行并对比 |
| **过拟合评估** | 自动分析三阶段收益递减、风险递增、策略稳定性，输出 0-100 过拟合风险评分 |
| **实盘交易** | 通过天勤网关连接交易所，执行真实的开仓、平仓操作 |
| **运行监控** | 操作日志持久化至 SQLite 数据库，支持审计追溯 |

## 系统组成

```
┌──────────┐   ┌───────────┐   ┌───────────┐
│ main.py  │──▶│  backtest  │──▶│strategies │
│ 命令行入口 │   │  回测引擎   │   │  策略模块   │
└──────────┘   └───────────┘   └───────────┘
                     │                │
                     ▼                ▼
              ┌───────────┐   ┌───────────┐
              │   data    │   │  config   │
              │ 数据/存储  │   │ 配置管理   │
              └───────────┘   └───────────┘
```

- **`backtest/`** — 回测引擎、数据加载、报告生成、对比分析
- **`strategies/`** — 策略核心算法 + vn.py / 天勤双桥接器
- **`data/`** — CSV 数据导出、SQLite 元数据与日志管理
- **`config/`** — YAML 分层配置管理，支持本地覆盖

详细架构说明参见 [系统架构设计](architecture.md)。

## 技术栈

- **Python** 3.10+
- **数据**: pandas, numpy
- **数据验证**: Pandera (DataFrame Schema) + Pydantic (单条记录)
- **ORM**: peewee
- **回测框架**: vn.py BacktestingEngine
- **数据源**: 天勤量化 SDK (tqsdk)
- **配置**: YAML 分层合并 (PyYAML)
- **存储**: SQLite (操作日志与元数据), CSV/JSON (行情与报告)

## 运行模式

| 命令 | 模式 | 说明 |
|------|------|------|
| `export` | 数据导出 | 从天勤拉取历史 K 线，导出为 CSV |
| `test` | 策略测试 | 离线验证策略逻辑，不联网 |
| `backtest` | 统一回测 | 自动选择引擎：单标的→TqSdk(GUI)，批量→vn.py(文字报告) |
| `report` | 报告生成 | 从数据库生成回测报告（不执行回测） |
| `live` | 实盘交易 | 对接真实交易所执行交易 |

## 快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 导出历史数据
python main.py export --symbol DCE.m2509 --start 2025-01-01 --end 2026-01-01

# 3. 运行三阶段回测
python main.py backtest --symbol DCE.m2509

# 4. 查看回测报告（控制台输出 + JSON 文件）
```

完整使用说明参见 [使用指南](usage-guide.md)。

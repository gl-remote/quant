# 回测系统功能说明

> 版本: 1.0.0 | 更新日期: 2026-05-23

---

## 1. 系统简介

本回测系统基于 vn.py 量化交易框架构建，提供完整的策略回测流水线：从本地 CSV 加载历史数据，按科学比例划分训练集/验证集/测试集，在三个数据集上独立运行回测，生成详细交易报告，并对三阶段结果进行对比分析，评估策略的泛化能力和过拟合风险。

## 2. 核心功能

### 2.1 数据接入

从项目本地路径 `.quant_shared_data/csv` 读取 CSV 格式的历史 K 线数据文件，支持以下文件命名模式：

- `{symbol}.csv`
- `{symbol}_qlib.csv`
- `{symbol}_*.csv`

CSV 文件需包含以下列：`datetime`、`open`、`high`、`low`、`close`、`volume`。

### 2.2 数据随机划分

将完整历史数据集按科学比例分割为三个子数据集：

| 数据集 | 默认比例 | 用途 |
|--------|---------|------|
| 训练集 (Train) | 60% | 策略参数优化 |
| 验证集 (Val) | 20% | 策略选择与超参调优 |
| 测试集 (Test) | 20% | 最终性能评估 |

支持两种划分模式：
- **时间顺序划分** (`shuffle: false`) — 前 60% 训练，中间 20% 验证，后 20% 测试。避免未来信息泄露，推荐用于时间序列数据。
- **随机打乱划分** (`shuffle: true`) — 随机采样划分，适合跨品种/跨时段稳健性检验。

### 2.3 三阶段独立回测

分别在训练集、验证集、测试集上独立执行完整的回测流程，确保各阶段回测环境完全隔离。每个阶段都包含：

- 完整的 OHLCV 行情驱动
- 策略信号生成 (金叉买入 / 死叉 / 止损 / 止盈卖出)
- 模拟交易执行 (含手续费和滑点)
- 资金曲线追踪

### 2.4 交易报告生成

为每个数据集的回测生成详细的交易报告，内容包括：

- **资金概况**: 初始资金、最终权益、总收益率、年化收益
- **交易统计**: 交易次数、盈利/亏损次数、胜率、平均盈亏、盈亏比
- **风险评估**: 最大回撤、夏普比率、日波动率、收益回撤比
- **交易明细**: 每笔交易的时间、方向、价格、数量、盈亏、原因

报告以 JSON 格式保存至 `.quant_shared_data/reports/` 目录，同时输出格式化的控制台报告。

### 2.5 三阶段对比分析

对训练集、验证集、测试集的回测结果进行多维度对比，重点分析：

| 分析维度 | 说明 |
|---------|------|
| **收益递减分析** | 检测 train → val → test 收益率是否递减 (过拟合信号) |
| **风险递增分析** | 检测回撤是否在测试集上显著增大 |
| **稳定性分析** | 使用变异系数 (CV) 衡量各指标在三阶段上的波动 |
| **过拟合综合评估** | 0-100 评分体系，综合收益、回撤、夏普、胜率变化 |

## 3. 技术特性

- **vn.py 框架兼容**: 主引擎使用 vnpy BacktestingEngine，策略继承 CtaTemplate
- **自动降级**: vnpy 不可用时自动切换至内置回测引擎，保证系统可用性
- **可复现性**: 支持设置随机种子，确保数据划分和回测结果可复现
- **可配置化**: 所有参数通过 conf.yaml 统一管理
- **报告持久化**: JSON 格式报告、交易记录、资金曲线自动保存

## 4. 系统要求

- Python >= 3.8
- 依赖包: numpy, pandas, pyyaml, matplotlib (可选)
- vnpy >= 3.8.0, vnpy_ctastrategy >= 1.2.0 (可选，推荐)
- 天勤量化 SDK (tqsdk)，用于旧版兼容模式

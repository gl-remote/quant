# Workbench 工作台

本目录用于存放研究、实验、调试过程中正在形成但尚未归档的文档。

## 用途

AI 或人工在进行策略研究时，产生的中间文档应先放到这里，而不是直接写入 roadmap 或 archive。

适合放入本目录的内容包括：

- 策略实验记录；
- 参数对照和初步结果；
- 临时研究结论；
- 尚未确认是否有长期价值的方向；
- 修复框架问题后需要重新评估的实验记录；
- 准备归档前的压缩版总结。

## 工作流

1. **从 roadmap 领取方向**  
   roadmap 只描述阶段目标、评价标准和候选方向。

2. **实验文档先写入 workbench**  
   AI 研究过程中产生的文档默认先放在本目录，文件名应包含实验版本和研究方向。

3. **实验过程中持续更新**  
   记录关键假设、结果、结论、受框架 issue 影响的部分。

4. **发现底层框架问题时暂停**  
   框架问题应转入 [`docs/issues`](../issues/) 记录，不在实验文档中展开大量排障细节。

5. **确认有长期价值后再归档**  
   只有结论稳定、文本压缩、引用修正后，才移动到 archive。

## 与其他目录的区别

| 目录 | 用途 |
|------|------|
| `docs/roadmap` | 阶段规划、研究方向、评价标准 |
| `docs/workbench` | 当前研究中的实验记录和临时结论 |
| `docs/issues` | 实验中发现的底层框架问题 |
| `docs/archive` | 已稳定、可长期引用的归档文档 |

## 文档命名建议

```text
<topic>-<version>-<direction>.md
```

例如：

```text
low-validation-cost-r1-bollinger-squeeze-breakout.md
```

命名应尽量体现：

- 研究主题；
- 实验版本；
- 当前尝试的具体方向。

## 重要约定

- AI 研究中生成的实验文档，默认先放到本目录。
- workbench 文档可以持续修改、压缩、重命名。
- 不要把未验证的实验结果直接写入 roadmap。
- 不要把尚未稳定的中间结论直接放入 archive。
- 归档前应清理冗余过程，只保留长期有价值的信息。

# va-asymmetry-composite · 策略规格与 P0–P9 实验总结

> 本文**兼具两份职能**：一是 `va-asymmetry-composite` 策略的**数学规格（策略行为唯一契约）**，单独阅读即可复现策略，不依赖代码实现或聊天上下文；二是 **P0–P9 全部实验的结论汇总**，各 Phase 的判定与数字散见于对应章节 callout，文末 §13 给出收尾总览。实验过程与排程见 `research/themes/va-asymmetry-composite/experimental-plan.md`。
> 本规格在 B0 冻结版（归档 `2026-07-10-va-asymmetry-composite`，v1.0）之上，曾列入「逐 tier 可选日内开仓时机 `entry_mode`」候选层（§3.1.2）。**P2 已全族回测并否决该候选层**（择时为零/负轴，见 §11），故本契约**仅保留 `baseline`（即刻入场）为唯一入场方式**。
> **B0 承诺路径**：全 tier `baseline`（不择时，见 §3.1.1），锁定为契约行为；§3.1.2 的 7 种时机已证伪、仅作审计留存，不属 B0 契约、不得声称择时有效。

策略在物理上分为两层，后者依赖前者：

- **A 层 · 盘前分类器**：每个交易日开盘前，给每个合约算出"今天做多候选 / 做空候选 / 不参与"，结果当天固定。
- **B 层 · 日内交易执行**：交易日当天，按 A 层给出的清单执行开仓、止损、平仓与仓位管理。

## §1 分类器：输入、输出与归类（A 层 · 盘前 · 日频）

### §1.1 原始输入（3 个原始量 · 定义方式）

对每个合约，基于近 **10 日**行情（偏度取前一日 5m K 线）算出 3 个**原始量**。注意：这些原始量**本身不是排名**——排名只是 §1.3 里多种归一化手段之一，可替换；原始量未经归一化、量纲/分布随品种与时段而异，不能直接用于阵营判定。

三个原始量及其**定义方式**（计算口径继承 poc 分类器，见 `classifier-math-spec`）：

1. **量加权价格偏度 A3_skew（原始）**：取前一交易日 `skew_win` 指定粒度的 K 线（见 §0.1），在该粒度 K 线集合上按成交量加权估计价格分布的三阶中心矩标准化偏度（poc §4）：
   `A3_skew := m_3 / m_2^{3/2}`，其中 `m_k = Σ_j v_j (p_j − μ_v)^k / Σ_j v_j`，`μ_v = Σ_j v_j p_j / Σ_j v_j`，`{p_j, v_j}` 为各 K 线价格与成交量。
   语义：核心信号，"昨天成交量堆在价格高位还是低位"。`A3_skew ∈ ℝ`，实证范围 ≈ [−3.1, +2.2]、`|A3_skew| < 3` 为典型（见 §1.3.0 D2）。
2. **真实波幅 ATR（原始）**：该合约前 `L`（=`atr_entry_win`，见 §0.1）日真实波幅均值，原始单位为价格或 bps（poc §5.2，`daily_atr_L_bps`）：
   `TR_i = max(H_i − L_i, |H_i − C_{i−1}|, |L_i − C_{i−1}|)`，`ATR = (1/L)·Σ_{i=d−L+1}^{d} TR_i`，`H_i/L_i/C_i` 为第 i 日日高/低/收。
3. **收益率方向 / 趋势（原始）**：该合约前 `M`（=`trend_entry_win`，见 §0.1）日累计对数收益 `trend_ret_M = log(C_d / C_{d−M+1})`（poc §5.3），原始取值为收益本身（可正可负）。

> 注：策略名含 "value-area"，但该成分不参与实际计算；真正使用的是上述量加权偏度。三者均为原始量，是否及如何归一化、映射到可比尺度由 §1.3 决定。定义方式中的窗口——原始量窗口 `atr_entry_win`/`trend_entry_win`（即 `L`/`M`）、`skew_win`（偏度 K 线粒度），以及三者各自的 **rank 窗口 `skew_rank_win` / `atr_rank_win` / `trend_win`（天数可独立设定）**——均为可调超参，具体值与候选集见 §0.1。

### §1.2 主攻阵营（6 类 · 标签 + 逻辑，不含参数选择）

基于上述三维特征，本策略主攻 **6 个互斥阵营（tier）**，由 (偏度域, 波动率制度, 趋势制度) 三维组合唯一确定，再加 transition 轴决定"稳定/转换期"适用。6 类 = 多头 3 + 空头 3。下面**只讲每类的逻辑（为什么这个组合是可交易环境），不写具体参数区间**——参数区间在 §1.3 按归一化方式给出。

**元逻辑（来自 poc 阶段研究，非本 spec 杜撰）**：

- **偏度是"顺势放大器"而非独立均值回归**（KF-05）：偏度本身不单独产生 alpha，必须叠加趋势 + 波动过滤；无 trend filter 会在跌段亏钱。
- **ATR 是过滤器而非放大器**，与偏度近乎独立（KF-06，Spearman |ρ| < 0.05）；低 ATR 空头甚至会反转。
- **ATR × Trend 正交**（KF-19 洞察 S）：ATR = 强度，Trend = 方向，联合筛选是核心 filter 架构。
- **多头 3 种反弹机制**（KF-16 洞察 P）：低 ATR = 日常均值回归、中 ATR = 震荡后秩序恢复、高 ATR = 恐慌 V 反弹。
- **空头单机制**（KF-17 洞察 Q）：只有"崩盘前奏"一种，必须高 ATR；低 ATR 空头反转。

**各阵营逻辑（标签 + 为什么可交易）**：

| 阵营 | 方向 | 逻辑 | transition 适用 |
|:---|:---:|:---|:---|
| **L_seg3_lowmid_up** | 多 | 温和右偏（量堆在价格中低位、非极端底厚/顶厚）+ 低中波动 + 强涨势。偏度作顺势放大器确认涨势可持续性；含 KF-23 段3·低ATR 甜蜜点（平静市 + 底厚 → 慢反弹）。 | 全期（稳定/转换均可） |
| **L_seg12_high_up** | 多 | 近中性/极端底厚偏度 + **高波动** + 强涨势。对应洞察 P 的高 ATR"恐慌 V 反弹"机制：高波动中极端底厚 + 强涨的急反弹。 | 仅转换期（稳定期失效） |
| **L_seg2_low_flat** | 多 | 低偏度 + **低波动** + 平稳趋势。低波动平稳市的温和多头；KF-26 表明平稳期 alpha 仅存在于转换期。（注：旧 POC 验证表显示 full/trans 有正 alpha（IR≈+0.17），但 Composite 白名单曾以"塑形后 IR 为负"排除；该结论基于旧管线、且被 workbench 标为"漏判"。本实验未重验，列为**待验证候选**，与 §0.1/§4.2 六类全活跃一致） | 仅转换期 |
| **S_seg12_high_dn** | 空（核心） | 极端右偏（**顶厚**=量堆在价格高位=上方供给堆积）+ **高波动** + 走弱/跌。空头唯一机制"崩盘前奏"（KF-17），高波动中趋势转弱时最强反转做空信号；三 period 全 A，最铁复合信号（KF-29）。 | 全期，偏好 |τ| 大（转换期更强，KF-28） |
| **S_seg34_high_dn** | 空 | 次极端右偏（顶厚但略弱）+ 高波动 + 跌。同"崩盘前奏"机制、顶厚程度稍低的主力空头。 | 全期，偏好 |τ| 大 |
| **S_seg2_mid_dn** | 空 | 中高右偏（顶厚，程度弱于 S_seg12）+ **中波动** + 跌。高波动以外的顶厚做空（KF-23 段3·ATR中 新甜蜜点）；仅转换期有效。注：其 skew 区间 `(0.81,0.91]`（§1.3.1）为 S_seg12 `[0.81,1]` 的偏弱子集，故偏度程度低于 S_seg12，称"中高"而非"极端"。 | 仅转换期 |

> 注：阵营逻辑依据见 poc `research-status.md`（KF-05/06/16/17/19/26/28/29）与 `parameter-selection-spec §8 下游指南`。**互斥性**：6 类在 (skew × ATR × trend) 三维空间内完全互斥，同一事件最多命中 1 类（边界刻意不重叠，区间见 §1.3）。多头与空头严格分域：偏度 ≤ 0.30 多域、≥ 0.60 空域、中间 [0.30, 0.60] 不参与（具体边界见 §1.3.1）。

### §1.3 参数归一化与阵营参数范围

阵营判定在**归一化后的特征**上做。归一化手段是可替换超参；不同手段下，同一阵营对应的参数范围不同。下面先给每个参数的候选归一化方式，再在每种方式下给出各阵营的参数范围。

#### §1.3.0 归一化候选总览（主特征）

| 候选 | 名称 | 做法 | 输出尺度 | 适用原始量 |
|:---|:---|:---|:---|:---|
| A（默认 B0） | 滚动排名 rank | 对原始量在 `*_rank_win` 窗口（§0.1，skew/atr/trend 天数独立）内滚动排名：`r_t(x) = #{x_{t−N+1…t} ≤ x_t} / N ∈ (0,1]`（N = 窗口天数） | **0~1（均匀）** | 偏度 / ATR / 趋势 均可（分布稳健） |
| B | z-score 标准化 | `(x − μ)/σ`，μ、σ 取窗口内 | **无界 ℝ（中心 0）** | 偏度 / ATR / 趋势 均可；但 D2 已实证**跨 0、窗内混合符号、跨合约尺度差 2×**，固定 z 阈值不可移植，仅作下游 ℝ 尺度变体（详见 §1.3.0 注） |
| C | 分位数 / 百分位 | `percentile(x)` 在窗口经验分布中的位置 | **0~1（均匀）** | 偏度 / ATR / 趋势 均可（≈A） |
| D | 绝对阈值分档 | 直接对原始量设硬阈值（如 ATR 以 bps 绝对档） | **离散档** | ATR 最自然（有绝对量纲）；偏度/趋势需先定尺度 |

> **尺度一致性**：§1.3.1~1.3.3 的阵营范围写在 **0~1 尺度**上，故仅 **A / C 可直接套用**。B（ℝ）、D（离散）须先对齐尺度（见各小节）。
> **分布假设（已实证，基于 `dataset_full.parquet` 的 n=36,625 个 A3_skew 事件 = D2）**：区分两层分布——(D1) 偏度所概括的**日内量加权价格分布**（偏度是其统计量）；(D2) **偏度统计量自身的时间序列** `{ŝ_i}`，即归一化对象。归一化作用在 D2 上、与 D1 形状无关；"用对称假设归一化偏度自相矛盾"为**范畴错误**（两者非同一分布）。对 D2 的实证：
> - 位置/对称性：`P(ŝ>0)=0.493`，`E[ŝ]≈0`，`median(ŝ)≈0`，`range(ŝ)=[−3.1,+2.2]` ⇒ "ŝ>0/右偏"假设不成立，log 等正支撑变换失效；
> - 形状：`skew(ŝ)≈+0.02`（近似对称），`excess kurt(ŝ)=+0.77`（中度重尾），`ŝ ∼_fit t(ν≈12)`（t 与正态在中间分位几乎重合，差异仅在极尾）⇒ z-score 对称前提满足，但重尾使 σ 不稳定；
> - 跨合约：`std(ŝ)∈[0.32,0.66]` 且 JB 普遍拒绝正态 ⇒ 绝对阈值(D)不具跨品种可移植；
> - 窗内：`∀N`，窗口内 D2 普遍同现正负（中心≈0）⇒ 固定 z/绝对阈值在窗内语义混乱。
> **结论**：A/C（排名/百分位）是 D2 上唯一经得起检验的归一化（非参数、零分布假设、跨窗口/跨合约稳健）；B 仅作下游 ℝ 尺度待校准变体（对称前提满足，但窗内跨 0 + 跨合约尺度差使其**不能用于 tier 判定**）；D 仅单品种内可作绝对档。注：`N=10` 仅约 10 个日频观测，`scipy.t.fit` 在多个合约退化至天文数字，故参数化 PIT 不纳入候选。
> **同合约窗内归一化（A 层盘前 · 日频 · N 天参数化）**：本策略归一化即**逐合约（within-contract）滚动**，且按 spec 设计在 **A 层盘前、日频**完成——§1.4 明确 A 层"开盘前基于前 N 日行情算好、当日清单当天不再变化"，B 层（日内执行）全天复用、**不重新计算 skew**。故归一化作用在**每日 1 个 skew 观测**构成的序列上（每合约每天 1 值、日内恒定），§1.3.0 的跨合约尺度差（std∈[0.32,0.66]）不影响判定。这恰与被否决的**旧实现**相反：旧管线（stage4/stage2）对 `A3_skew` 按**逐事件行**滚 `ROLLING_EVENTS=100`，因每日约 6 行且日内 skew 恒定，等效仅 ~17 交易日、且同日重复行引入 ≤0.05 的秩 jitter——混淆了"日内事件"与"日频信号"，是已知缺陷；正确实现须先 `drop_duplicates(event_date)` 再去按 N 日滚（与 atr/trend 同口径；该正确口径将在后续独立脚本实现，不沿用被否决的旧 stage4 实现）。**窗口长度 N 为已参数化超参（`skew_rank_win` / `atr_rank_win` / `trend_win` 三者天数独立可调，见 §0.1），后续微调将遍历各天数**。

> 结合 D2 已实证为对称、中度重尾 `t(ν≈12)`、中心≈0 且归一化落在每日序列上，选型结论：
> (i) **首选 = 逐合约滚动百分位/排名（A/C）**：与 spec 以百分位定义的阵营边界（0.09..0.91）精确对应、模型零风险；
> (ii) **尾部分位分辨率（供天数微调）**：边界为秩坐标 `p ∈ {0.09, 0.19, 0.30, 0.60, 0.81, 0.91}`，单侧尾观测数 `n_tail(p) := p·N`。要求 `n_tail(0.09) ≥ n_min`，取 `n_min = 5` ⇒ `N ≥ ⌈5/0.09⌉ = 56` ⇒ 推荐 `skew_rank_win ≥ 60`（`N=20 ⇒ n_tail≈1.8` 偏短；`N=60 ⇒ ≈5.4`、`N=100 ⇒ 9` 更稳）。atr/trend 边界居中（0.33/0.67），`N=20` 已满足 `n_tail(0.33)≈6.6`，不在此限；

> (iv) **参数化 t-PIT 的增量区间**：仅在 `N ≲ 40`（约 10–20 天窗口）时相对 rank 有增量（借已知 t 形状对极值收缩、稳 0.09/0.91）；常规 `N≥60` 天窗下 rank 已够平滑，参数化只增模型风险。

> - **修订结论**：原始 z-score B 仍**不能**用于 tier 判定（阈值漂移）；**t-PIT 对齐版 B 可且已实测 in-sample 占优**。**但 OOS / walk-forward 验证（2026-07-11，`va_p03b_b_walkforward.py`，冻结引擎，4 折时间展开，Cap=4.0）未过门禁**：OOS(后50% 时段) ΔSharpe(B−A)=**+1.50**、μ_true+13.24%、**P(B>A)=0.838 < 0.95**；4 折前向中最早 25% 折0 反为负（ΔSh=−1.76、P=0.194）——B 的 in-sample 优势在样本外**方向仍正、幅度可观，但置信不足且非平稳（早期一段失效）**，**故默认归一化升为 B(t-PIT)**（2026-07-11 决策：采用 80% 单侧置信门槛——OOS P(B>A)=0.837≥0.80 且 ΔSh=+1.50≥0.2 双双通过；原 95% 门未过，故该决策为**风险偏好决定而非 95% 铁证**，A/C 仍等价可作回退）。B 不再仅为候选，仍建议更大样本/跨期复验或 ν∈{8,12,20} 鲁棒扫描以巩固。本修订不改注(i)(ii) 选型逻辑、不改阵营边界坐标 `I_τ`（t-PIT 与 A/C 同 0~1 尺度、直接套用）。详见 `experimental-plan.md` P3 段、B 轨 in-sample `project_data/ai_tmp/p3_b_track/summary.md`、OOS `project_data/ai_tmp/p3b_b_walkforward/summary.md` 与对称头对头 `project_data/ai_tmp/p3b_b_headtohead/summary.md`。

> ⚠ **短窗口分辨率（skew 专属，仅影响统计功效、不使边界失效）**：边界为分位坐标，但当 `skew_rank_win` 过短，每侧尾观测数 `0.09×N` 不足——N=10 时每侧尾仅 ≈0.9 个、`S_seg12 (0.81,0.91]` 与 `S_seg2 (0.81,1]` 等细分在 10 日窗下**分辨不出**；N=20≈1.8、N=30≈2.7、N=60≈5.4。故 §0.1 的 `P_skew10/20/30` 属**短窗探测档**，其 tier 细分分离度随窗口缩短而下降；各窗口的 tier alpha 是否仍成立须在该窗口下重验（重跑 tier 显著性验证，而非重标边界坐标）。N≥60 更稳，故 skew 微调应将 ≥60 天纳入候选（§1.3.0）。

#### §1.3.4 转换轴(transition)归一化与阵营适用范围

`transition_flag` / `transition_intensity_signed` 是对 ATR 维度的**另一种归一化**（波动率制度切换检测），独立于上面 A~D 主特征，单独列出。与主特征同理支持多候选，**B0 默认 T1（session-ATR 桶 crossover，继承 poc §6.4）**：

| 候选 | 名称 | 做法 | 输出 |
|:---|:---|:---|:---|
| T1（默认 B0） | session-ATR 桶 crossover | 相邻交易日 session-ATR 档位（low/mid/high）跨档切换检测 + 3 日衰减 | `flag` ∈{0,1} + `τ_signed` ∈[−1,+1] |
| T2 | ATR 原始动量 | 对 session-ATR 水平/rank 做归一化差分 `m = tanh(k·Δ/W)` | 有符号连续 ∈[−1,+1]；需另设阈值 `|m| > Θ_m` 才得二元 `flag`，衡量 ATR 动量/突变量（非严格 crossover，单日尖峰也会被记为转期） |
| T3 | ATR z-score 突破 | ATR 滚动 z-score 跨 ±Θ_z 即 flag=1，符号表方向 | `flag` ∈{0,1} + 符号 |

> 已排除 HMM（原 T4）：数据量（需数年日线，单合约换月仅数月）、批量 vs 在线、概念错位（平滑回望式滞后 vs 即时 crossover）三项冲突，不构成合理备选。
> 候选只改变"ATR → 转期信号"映射，**不改变下方阵营适用范围语义**。

> **【P8 结论（2026-07-11，`va_p08_transition.py`，冻结 P1 引擎，Cap=4.0）】** T2/T3 **未稳定优于 T1**，保持 T1（生产 `transition_flag`，B0）：转期占比 T1=44.6% / T2@0.3=5.4% / T3@1.5=15.0%；转期子集 per-trade edge T1 夏普 2.01、T2@0.3 仅 1.19（样本过小不可靠）、T3@1.5 2.32（仅略高、覆盖仅 15% 且样本小，不构成稳定优势）。`trans_only vs full` 配对 ΔSh 的强负为**构造性偏差**（非转期日 pnl=0 而 full 仍有正贡献），不能据此判 T 方法差；公平看 per-trade 质量，T3 单档微弱占优但不稳健、T2 全面劣于 T1。结论：保持 T1（B0）。**遗留观察**：既 trans 子集 edge 弱于 stable、且 P0.5 已证伪"偏好 |τ| 大"，自然问题是"是否从 whitelist 剔除转期只留 stable"——但涉及结构性改动生产 whitelist（用户"先不动生产代码"），且 B0 当前 full 已最优，故 P8 不采纳，留待 P9 治理裁剪评估。详见 `project_data/ai_tmp/p8_transition/summary.md`。

**候选 T1（默认）详细定义**：session 级 ATR 桶近 3 日是否 crossover，产出 `transition_flag` 与 `τ_signed ∈ [−1,+1]`（0=稳定，+1=低→高波动扩张，−1=高→低波动收缩，3 日衰减）：

`transition_intensity_signed(s,t) := Δ_recent × max(0, 1 − age / W)`

- `Δ_recent = level(current) − level(prev_at_crossover)`，档位 `level(low)=0, mid=0.5, high=1` ⇒ `Δ_recent ∈ {−1, −0.5, 0, 0.5, 1}`；
- `age` = 距最近 crossover 交易日数，`W = n_transition_window_days`（=3，可延长）；
- **关键不变性**：`τ_signed = 0 ⟺ 稳定期`（距最近 crossover **≥ W 日**、含 age=W 时 `max(0,1−W/W)=0` 已归零）。故 `τ_signed ≠ 0` 仅在 crossover 后不足 W 日（age < W）成立、稳定期恒为 0，**非恒成立**；
- `flag = 1 ⟺ τ_signed ≠ 0`；供 B 层仓位缩放与定向出场。

**各阵营的 transition 适用范围（在 T1 及任意候选下语义一致）**：

| 阵营 | transition 适用范围（τ_signed） |

| L_seg3_lowmid_up | `τ_signed ∈ [−1, +1]`（full，含稳定期 τ=0） |
| L_seg12_high_up | `τ_signed ≠ 0`（= transition_flag=1，仅转期） |
| L_seg2_low_flat | `τ_signed ≠ 0`（仅转期） |

#### §1.3.4.1 spec↔数据 taxonomy 映射表（坐标取向 = 互补）

⚠ **重要勘误（2026-07-11 · 已实证修正前版"spec 反向严重错误"误读）**：初版曾称"将 spec `tier()` 直接套数据 rank 列得 100% ∅，故 spec 坐标反转/不兼容"。**该结论错误**——误读根因是把数据列 `signed_skew_rank_roll` **直接**喂入 spec `tier()`，而未做取向对齐。实证：先取 **`spec_r_s = 1 − signed_skew_rank_roll`**（互补变换），再套 spec `tier()` → **NA=0%、spec 判 short 与数据 DN 一致率=100%**，且交叉表干净恢复生产阵营。故 **spec 坐标框架自洽、未被写反**；它用的是「短侧秩」取向（高 `r_s` = 极端跌 skew = short），而数据列 `signed_skew_rank_roll` 是「带符号 skew 秩」（高 = 涨 skew = long = `UP`），二者为**互补关系**，非错误。

- **取向定义（务必区分）**：
  - 数据 `signed_skew_rank_roll`：`DN`(short) 落**低值(0–0.30)**、`UP`(long) 落**高值(0.70–1.0)**（NEUTRAL 居中 0.30–0.70）——生产分类器 `poc_va.py._skew_label` 即按此：低 rank→DN、高 rank→UP。
  - spec `r_s`（§1.3.1–1.3.4）：`S`(short) 阵营在**高 r_s([0.60,1])**、`L`(long) 阵营在**低 r_s([0,0.30])**——即"高 = 极端跌 skew = 短"。
  - 二者关系：**`spec_r_s = 1 − signed_skew_rank_roll`**（或等价地，spec 的 `r_s` 是"短侧秩"，数据是"带符号 skew 秩"）。
- **segment 粒度不一致（已实证：不影响分类，非"重叠歧义"）**：数据按 `skew_label` 切 **4 段空头(DN_1–DN_4) + 4 段多头(UP_1–UP_4)**（本数据集实际出现 7 段：DN1–4 + UP1–3，UP4 无事件）；spec 仅 **3 段空头(S_seg12/S_seg34/S_seg2_mid) + 3 段多头(L_seg3/L_seg12/L_seg2) = 6 营**。粗看 spec 区间在 `r_s` 单维有重叠（`S_seg12[0.81,1]` 与 `S_seg2_mid(0.81,0.91]`、`L_seg3(0.09,0.30]` 含 `L_seg2(0.09,0.19]`），**但实证检验（互补变换后套 spec `tier()`，全量 312 事件）：同时满足 ≥2 个阵营条件的事件 = 0 个（0.00%）——重叠仅在 `r_s` 单维，被各自的 `r_a`/`r_t` 条件完全解耦，无真歧义，分类结果不依赖分支顺序。** 数据段并入 spec 营为干净 1:1（除 DN2 因含 `atrHigh`+`atrMid` 两变体而分别落入 `S_seg12_high_dn` / `S_seg2_mid_dn`，属按 ATR 正确拆分，非歧义）：`DN1→S_seg12`、`DN3→S_seg34`、`DN4→S_seg34`、`UP1→L_seg12`、`UP2→L_seg3`、`UP3→L_seg3`；`L_seg2_low_flat` 在本数据集命中 0 事件（该 spec 营未触发，非缺陷）。故**粒度差异是 spec 比数据更粗（合并 DN3+DN4、UP2+UP3），不影响分类器可用性**。
- **两套 regime 命名**：数据用 `atr_regime∈{high,mid,low}` + `trend_regime∈{up,down}` + `period∈{stable,trans}`；spec 用秩阈值区间。ATR/trend 语义可直接对齐，**仅 skew 须先取互补**。

|:---|:---|:---|:---|

```text
  L_seg2_low_flat   if  r_s ∈ (0.09, 0.19]∧ r_a ≤  0.33        ∧ r_t ∈ (0.20, 0.75)
  S_seg12_high_dn   if  r_s ∈ [0.81, 1]   ∧ r_a >  0.67        ∧ r_t ≤ 0.20
  S_seg34_high_dn   if  r_s ∈ [0.60, 0.81)∧ r_a >  0.67        ∧ r_t ≤ 0.20
  S_seg2_mid_dn     if  r_s ∈ (0.81, 0.91]∧ r_a ∈ (0.33, 0.67) ∧ r_t ≤ 0.20
  ∅（不参与）        otherwise
```

> 此判定仅给出 tier 分类；各 tier 的 transition 适用范围由 §1.3.4 独立给出，作为二层过滤（tier 命中后，当日 `τ_signed` 须落在其适用范围方参与）。

配套域函数（由上式各 tier 的 skew 区间之并直接推出）：
`domain(r_s) = long if r_s ≤ 0.30,  short if r_s ≥ 0.60,  ∅ if r_s ∈ (0.30, 0.60)`.

**性质（非规则，由区间构造保证）**：上式六个 `(r_s, r_a, r_t)` 区间互不重叠 ⇒ 任一合约至多命中一类（互斥）；并非所有组合被覆盖 ⇒ 空隙归 `∅`（partial 分类）。多/空域由 `r_s ≤ 0.30 / r_s ≥ 0.60` 严格不相交界定。区间刻度见 §1.3.0（A/C 下为固定 0~1 秩坐标；B/D 对齐规则见各小节）。

### §1.4 分类结果的时效

A 层在**开盘前**基于各指标前 N 日行情（见 §0.1 各 `*_rank_win`，skew/atr/trend 天数可独立设定）算好，输出一份"当日清单"：哪些合约是多头候选、哪些是空头候选、
各自属于哪个 tier。这份清单**当天不再变化**，供 B 层全天使用。

### §1.5 波动率（ATR）的计量单位（硬约束）

策略中所有"距离 / 止损 / 仓位 / 成本"的统一度量，都是**入场前 `atr_entry_win` 日化 ATR**（窗口长度见 §0.1），并归一化为
"占价格的万分之几（bps）"。所有后续计算都使用同一把尺子，便于直接比较。

该日化 ATR 同时充当**日波动基准 σ_day**：§3.3 的持仓退出以它为时钟单位，把"日历时间"换成"波动率时间"，
使快 / 慢品种在风险尺度上对齐（scale-invariant）。

## §3 交易执行 · 塑形（B 层 · 日内 · 小时级）

### §3.1 入场

- 时钟：交易日内以 `base_tf` 为基准周期推进；入场判定在 `entry_tf` 候选周期上检查（B0 默认见 §0.1）。
- **入场 = 双层条件同时满足**：(1) A 层当日清单里该合约是某 tier 的多/空候选；(2) 该 tier 的 `entry_mode(tier)` 在 `entry_tf` K 线上触发（见 §3.1.1）。
- **方向**：由 tier 决定（多域 tier → 做多，空域 tier → 做空）；`entry_mode` 的触发方向须与 tier 方向一致（映射见 §3.1.2，禁止反向开仓）。
- **数量**：由 §5 的仓位公式决定（B0 下 `W(tier)=W0`，见 §0.1；只受 §7.2 名义上限压仓影响）。

#### §3.1.1 极限入场（`baseline` · B0 承诺路径）

`baseline` 是 B0 锁定的**唯一入场方式**，不做任何日内择时过滤：

```text
A_candidate(s,t) := (s 在交易日 d 的 A 层当日清单中，属于某 tier τ(s))
tier_dir(τ)      := long  , 若 τ 为多域 tier
                    short , 若 τ 为空域 tier

baseline_enter(s,t) := A_candidate(s,t)
                    ∧ ( tier_dir(τ(s)) = long  => 开多 )
                    ∧ ( tier_dir(τ(s)) = short => 开空 )
```

即：A 层清单命中后，在该合约当日的**首根 `entry_tf` K 线**即按 tier 方向开仓。`entry_mode(tier) ≡ baseline`（∀ tier）时，策略退化为「候选即开」的基线，与 B0 冻结版行为一致；此即本规格的**承诺路径**。

#### §3.1.2 候选入场时机（`entry_mode` 候选集 · **已证伪，见 §11**）

除 `baseline` 外，下列 7 种时机为**候选触发条件**，按 tier 可独立选用，均在 `entry_tf` K 线上计算。统一前置记号：`t` = `entry_tf` 的 bar 索引，`C_t/H_t/L_t/O_t` = 该 bar 的 OHLC，`idx(·)` 显式区分 bar 引用、`time(·)` 显式区分时钟时刻、`prev_session` = 前一交易日日线。

```text
# 通用入场谓词骨架（方向恒与 tier 一致，禁止反向）
enter(s,t; mode) := A_candidate(s,t)
                 ∧ pred_mode(s,t)
                 ∧ ( tier_dir(τ(s)) = long  => 开多 )
                 ∧ ( tier_dir(τ(s)) = short => 开空 )
# 其中 pred_mode ∈ { pred_boll, pred_macd, pred_kdj, pred_rsi,
#                    pred_breakout, pred_prevhi, pred_openrange }

# boll（布林带触带，boll_win/boll_k 见 §0.1）
BB_mid_t := SMA(C, boll_win)_t ; BB_std_t := std(C, boll_win)_t
BB_up_t  := BB_mid_t + boll_k·BB_std_t ; BB_low_t := BB_mid_t − boll_k·BB_std_t
pred_boll(s,t) := ( tier_dir=long ∧ C_t ≤ BB_low_t )
               ∨ ( tier_dir=short ∧ C_t ≥ BB_up_t )

# macd（DIF/DEA 交叉，macd_f/s/sig 见 §0.1）
DIF_t := EMA(C, macd_f)_t − EMA(C, macd_s)_t ; DEA_t := EMA(DIF, macd_sig)_t
pred_macd(s,t) := ( tier_dir=long ∧ DIF_t > DEA_t ∧ DIF_{t-1} ≤ DEA_{t-1} )
               ∨ ( tier_dir=short ∧ DIF_t < DEA_t ∧ DIF_{t-1} ≥ DEA_{t-1} )

# kdj（超卖/超买区金叉/死叉，kdj_n/m/s/os/ob 见 §0.1）
K_t, D_t := KDJ(C, kdj_n, kdj_m, kdj_s)_t
pred_kdj(s,t) := ( tier_dir=long ∧ K_t > D_t ∧ K_{t-1} ≤ D_{t-1} ∧ K_t < kdj_os )
              ∨ ( tier_dir=short ∧ K_t < D_t ∧ K_{t-1} ≥ D_{t-1} ∧ K_t > kdj_ob )

# rsi（极值，rsi_n/os/ob 见 §0.1）
RSI_t := RSI(C, rsi_n)_t
pred_rsi(s,t) := ( tier_dir=long ∧ RSI_t < rsi_os ) ∨ ( tier_dir=short ∧ RSI_t > rsi_ob )

# breakout（前 breakout_n 根高低突破）
HH_t := max{ H_i | i = t−breakout_n .. t−1 } ; LL_t := min{ L_i | i = t−breakout_n .. t−1 }
pred_breakout(s,t) := ( tier_dir=long ∧ C_t > HH_t ) ∨ ( tier_dir=short ∧ C_t < LL_t )

# prevhi / prevlo（前一交易日高低突破；多=破 PH，空=破 PL）
PH := prev_session_high ; PL := prev_session_low
pred_prevhi(s,t) := ( tier_dir=long ∧ C_t > PH ) ∨ ( tier_dir=short ∧ C_t < PL )

# openrange（开盘 openrange_min 分钟区间突破，openrange_min 见 §0.1）
open_idx := idx( session 首根 entry_tf bar ) ; k := ceil( openrange_min / entry_tf_minutes )
OR_high  := max{ H_i | i = open_idx .. open_idx+k−1 }
OR_low   := min{ L_i | i = open_idx .. open_idx+k−1 }
pred_openrange(s,t) := ( t ≥ open_idx+k )
                    ∧ ( ( tier_dir=long ∧ C_t > OR_high ) ∨ ( tier_dir=short ∧ C_t < OR_low ) )
```

> 上述 7 种**均已回测验证并否决（P2）**：相对 B0 全 `baseline` 净夏普增量均不过采用门禁，且延迟入场/择时空间测试表明 edge 在持仓第 1–8h 缓慢累积、入场时机为零/负轴。故 `entry_mode(tier) ≡ baseline`（即刻入场）为**唯一有效入场方式**，下列公式仅作审计留存，**不再作为待采用候选**。**B0 全部 tier 取 `baseline`，本小节不影响 B0 契约。**

**tier → `entry_mode` 候选挂法（待回测验证）**：

| tier | 方向 | 逻辑风味 | 候选 `entry_mode` |
|:---|:---:|:---:|:---|
| L_seg3_lowmid_up | 多 | 动量延续 | `macd` · `breakout` · `prevhi` · `openrange` · `rsi`（实验） |
| L_seg12_high_up | 多 | 反转（底厚·恐慌 V 反弹） | `boll` · `kdj` · `rsi`（均超卖侧） |
| L_seg2_low_flat | 多 | 温和多头（低波动平稳市） | 待回测验证 |
| S_seg12_high_dn | 空 | 反转（顶厚·逢高做空） | `boll` · `kdj` · `rsi`（超买侧）· `breakout` · `prevhi` · `openrange`（均实验） |
| S_seg34_high_dn | 空 | 反转（次极端顶厚） | 同 S_seg12 候选集 |
| S_seg2_mid_dn | 空 | 反转（中高顶厚） | 同 S_seg12 候选集 |

> `boll`/`kdj`/`rsi` 属反转极值类（与顶厚/底厚阵营同向），`macd`/`breakout`/`prevhi`/`openrange` 属趋势确认/突破类（与动量阵营同向）；`rsi` 与 `kdj`/`boll` 风味重叠、`breakout`/`prevhi`/`openrange` 与 S_* 反转逻辑存在风味冲突，均标实验、须回测验证。

### §3.2 止损（SL）

各 tier 止损距离为 `K_SL(tier) × ATR`（ATR 见 §1.5），`K_SL(tier)` 的 B0 取值与候选见 §0.1（多域 tier 与空域 tier 分别取多/空域候选）。止损价与触发谓词：
   `SL_price = entry_price − K_SL(τ)·ATR`（多域 τ），`SL_price = entry_price + K_SL(τ)·ATR`（空域 τ）；
   `exit_SL(s,t) := (long(τ(s)) ∧ P_t ≤ SL_price) ∨ (short(τ(s)) ∧ P_t ≥ SL_price)`.

### §3.3 时间退出（主平仓机制 · 波动率时间）

持仓以**波动率时间**（非日历时间）计量。波动率时钟（单位 σ_day）定义如下。令 `r_k = ln(C_k/C_{k−1})` 为第 k 根 `base_tf` bar 的对数收益，单根波动增量
`Δv_k := |r_k| / σ_day`（σ_day 见 §1.5，单位：日波动倍数）。持仓累积波动率
`V(s,t) := Σ_{k: entry≤k≤t} Δv_k`。触发条件：
`V(s,t) ≥ H_vol(τ(s))  ⇒  时间退出（于下一根 base_t bar 收盘平仓）`。

| tier | 波动率预算 `H_vol(tier)`（× σ_day） |
|:---|:---:|
| L_seg3_lowmid_up | `B_L` |
| L_seg12_high_up | `B_L` |
| L_seg2_low_flat | `B_L` |
| S_seg12_high_dn | `B_S` |
| S_seg34_high_dn | `B_S` |
| S_seg2_mid_dn | `B_S` |

```text
B_L := arg_solve { median_{τ∈L_*}[ T_hold(τ, B) ] = 8h_calendar }   （B0 ATR 时钟、无 SL/TP 干扰）
B_S := arg_solve { median_{τ∈S_*}[ T_hold(τ, B) ] = 10h_calendar }
# 其中 T_hold(τ, B) 为在 H_vol(τ)=B、无 SL/TP 下的日历持仓时长（由 §3.3 波动率时钟反推）
```

即 `B_L`/`B_S` 由「复现原 8h/10h 中位持仓」这一锚点反解得到，是 **B0 锁定基线**；P2 曾显示不同 tier 的最优持仓时长分化（前载 S_seg34/L_seg3≈6h ≈ B_S×0.6，L_seg12≈10h），故把 `H_vol(tier)` 列为 P6 候选搜索轴。**但 P6 walk-forward（2026-07-11，冻结引擎，4 折时间展开，Cap=4.0）显示：in-sample 仅 ΔSh=+0.09（P=0.523，不显著），OOS 反向 ΔSh=−1.10、P(μ_true>0)=0.161（远低于 0.95 门禁），维持 B0 统一 8/10h 占优**——P2 的"强轴"信号为同批挑峰乐观上界且换冻结引擎后大幅衰减，未过门，不升格。B0 仍锁定 B_L/B_S；快品种单位 σ_day 内积累波动更快、日历持仓自动更短，慢品种更长。

优先级：**止损 > 时间退出**（同时触发取止损）。

### §3.4 移动止损（不启用）

本策略默认不启用移动止损。（P2 择时空间测试：trailing-stop 平均 pnl 仅 5.8/6.5/8.0 bps ≪ 固定持有 45.3bps，系统砍掉 67.6% 后段赢家，与「edge 需持有穿越噪声至真实峰值」机制自洽，支持保持关闭。）

### §3.5 硬止盈（不启用）

本策略默认不设硬止盈目标。（P2 表明 edge 在持仓数小时后才累积至峰值、前段多为逆信号/噪声，硬 TP 会提前截断 drift，支持保持关闭。）

> 与早期 S2（按 A/B/C 类型全局筛选）的区别：S2 是把"类型"当优化轴去挑，实测净夏普增量

### §4.2 默认配置（B0）

所有 6 个 tier 的白名单均 = 全品种：

```

```

即等价于旧 S1。参数仍只按 tier 分组（§0.1 的 `K_SL / H_vol / W`），品种不入参。

## §5 仓位与加权（B0 = 等权 × 等权）

### §5.1 信号强度加权 → 等权

强度权重按 tier 索引为 `W(tier)`。B0 下所有 tier 均取等权方案 `W0`，即 `W(tier) ≡ 1`（不做按信号强弱的加权）。

### §5.2 多空权重 → 等权

B0 下多头与空头的权重相同（均为 1），不做多空倾向性加权。

### §5.3 单品种单 tier 的目标仓位（压仓前）

目标名义仓位（压仓前）：
   `Notional_target(s,τ) = W_dir(τ) · W(τ) · (RiskPerTrade · Equity) / (K_SL(τ) · ATR_bps(s))`
   其中 `W_dir(τ)` = 方向权重（B0 下 `VW0` ⇒ 多空均 1，见 §5.2）、`W(τ)` = 强度权重（B0 下 `W0` ⇒ 1，见 §5.1）、`ATR_bps(s) = ATR(s)/price(s)` 为入场 ATR 占价比例（§1.5）。上界约束见 §7.1。

## §6 交易成本（硬约束 · 真实成本口径）

### §6.1 单边成本

每笔单边成本统一以 bps 计（占名义本金的万分之几，与 §1.5 的 ATR 同口径）：
`cost_oneway_bps = comm_bps + slip_bps`，其中 `comm_bps = fee_bps(contract)`（交易所手续费 ×(1+broker_markup) 折算），`slip_bps = n_ticks · tick_size · multiplier / price · 1e4`（滑点档数 × 最小变动价位 × 合约乘数 ÷ 价格）。具体数值查合约规格表 `contract_specs.py`，按下述方式计算：

- 佣金：**交易所手续费 ×（1 + broker_markup）**，再折算为 bps。
  - 交易所手续费：固定值（元/手）或成交额费率（`is_rate=True`，即价格 × 数量 × 乘数 × 费率）。
  - 期货公司加收比例由各品种在 `ContractSpec`（`contract_specs.py`）中携带，属实现/config 数据、非策略旋钮，不在本规格的参数表里。
    例外：`ap/cu/lh/j/pm/ic/im/au/wh/ec/zc` 等 11 个交易所基准手续费本就偏高、4 跳难覆盖成本的品种已下调至优惠档，
    避免高基准上失真。以上均为占位值，实盘以账户结算单为准逐品种校准。
- 滑点：按"滑点档数 × 最小变动价位 × 数量 ÷ 价格"折算为 bps（具体档数见 `contract_specs.py`，非本规格关心）。

### §6.2 成本相对波动的倍率（诊断用）

`κ(s) = cost_oneway_bps / ATR_bps(s)`，用于诊断摩擦相对波动的大小。

### §6.3 净收益

`Net_bps = Gross_bps − cost_oneway_bps − cost_oneway_bps = Gross_bps − 2·cost_oneway_bps`（进出各一次单边）。

## §7 风控

### §7.1 单笔止损 = `RiskPerTrade` × 权益（硬约束，见 §0.1）

这是 §5.3 仓位公式的硬约束。由 §5.3 公式推出理想止损损失恒等式 `Notional_target·K_SL(τ)·ATR_bps = RiskPerTrade·Equity`，故单笔名义仓位被显式设为使理想止损损失恰为 `RiskPerTrade·Equity`。实际损失上界：
   `Loss ≤ RiskPerTrade·Equity + slippage`（滑点使实损略超；B0 默认无 trailing/TP，见 §3.4/3.5）。

### §7.2 总名义暴露上限（默认见 §0.1 之 `Cap`）

所有持仓的名义金额之和 ≤ **`Cap` × 账户权益**。超出时按以下**确定性**优先级砍仓（每次砍至满足上限为止）：

设持仓集合 `O`，按复合键升序排序后从首部逐笔平仓，直到 `Σ_{(s,τ)∈O} Notional(s,τ) ≤ Cap·Equity`：
`key(pos) = ( W(τ),  t_entry(pos),  ord(τ) )`，其中 `ord(τ)` 为固定枚举序 `L_seg3(1) < L_seg12(2) < S_seg12(3) < S_seg34(4) < S_seg2(5)`。

> 砍仓只减少名义暴露，不改变其余持仓的任何参数或状态。

### §7.3 保证金约束（自动满足）

期货保证金率约 `margin_rate ∈ [5%, 12%]`；名义暴露 `Cap·Equity` 对应保证金占用 `Cap·margin_rate`（取默认 `Cap=4.0` 时∈`[20%, 48%]`），低于约束上限，不主动管理。

### §7.4 单日最大损失熔断（可选 · 默认见 §0.1 之 `cb`）

若当日累计亏损触限则停止开仓：`if Σ_{i∈day} Net_i ≤ −cb·Equity then open_new(s,t) := false  (∀ t 剩余当日)`（已有持仓按原规则退出）。

## §8 事件并发与去重

### §8.1 合约内去重

同一合约在去重窗口内只接受第一个有效信号，后续信号忽略。B0 默认窗口见 §0.1 之 `dedup`。

### §8.2 多 tier 并发

同一时刻同一合约由 §1.3 判定函数唯一确定一个 tier，故不存在多 tier 并发。
跨品种可并发，但受 §7.2 名义上限聚合压仓。

### §8.3 多空互斥

`domain(r_s)`（§1.3）使多域（`r_s ≤ 0.30`）与空域（`r_s ≥ 0.60`）严格不相交，故同一合约不可能同时多/空，
不做自对冲；跨品种多空可并存。

## §9 归因口径（硬约束）

任何声称"某方案平均收益为正"的结论，必须同时报告以下四项（记单笔毛收益序列为 `{g_i}`，单位 bps）：

- `μ_g = mean(g_i)`（平均毛收益）；
- `σ_g² = var(g_i)`（毛收益方差）；
- `μ_true = μ_g − σ_g²/2`（Itô 修正后的真实漂移）；
- `P(μ_true > 0)`（基于以"合约—日期"为聚类单元的自助法估计）。

判定：`edge ⇔ μ_true > 0 ∧ P(μ_true > 0) ≥ 0.95`。

## §11 本策略明确不采用的方案（审计）

以下方案经测试未通过，不作为 B0 的活动配置，仅留作审计：

- **日内择时入场模式 `entry_mode`（boll/macd/kdj/rsi/breakout/prevhi/openrange 共 7 种）**：P2 全族回测净夏普增量均不过采用门禁（择时为零/负轴，edge 在持仓第 1–8h 缓慢累积、不在开盘瞬间兑现）。已否决；B0 全部 tier 锁定 `baseline`（候选即首根 `entry_tf` K 线开仓），禁止任何日内择时过滤。公式与挂法仅存 §3.1.2 作审计。
- **全局按类型筛选（旧 S2）**：把各 tier 白名单整体收窄到单一 A/B/C 类型，相对 B0 净夏普增量为负，反向拖累；已由 `sym(tier)` 默认全品种取代，不采用。
- **信号强度加权（W1/W2/W3）**：按偏度距离 / 波动匹配 / 三维乘积加权，均未达增量门槛，不采用。
- **多空加权（VW1/VW2）**：按信息比率比例 / 频率平衡加权，未达增量门槛，不采用。
- **持仓时长分化 `H_vol(tier)`（按 tier 固定长度）**：P6 walk-forward（2026-07-11，冻结 P1 引擎，4 折时间展开，Cap=4.0）未过采用门禁——in-sample 配对 ΔSharpe=+0.09、P(μ_true>0)=0.523（不显著，且为同批挑峰乐观上界），OOS 反向 ΔSharpe=−1.10、P=0.161（远低于 0.95）；维持 B0 统一 8/10h 持仓。P2 workbench 所报 in-sample ΔSharpe+0.77 系非冻结引擎/挑峰口径，换冻结引擎后衰减至 +0.09 且 OOS 转负，属过拟合，不采用。详见 `project_data/ai_tmp/p6_hvol/summary.md`。

> **L_seg2_low_flat 不在本"已否决"清单**：它仍是 B0 活跃 tier（§0.1/§4.2）。旧 POC 证据矛盾——验证表显示 full/trans 正 alpha（IR≈+0.17），但 Composite 白名单曾以"塑形后 IR 为负"排除（基于旧管线、被 workbench 标为"漏判"）。其 alpha 须在新管线重验后定论，重验确为负再移出活跃集。

**跨 Phase 元结论（收尾）**：

- **"优化跑不赢 B0 夏普"先验被全程印证**：所有局部塑形轴（P3 C/A 等价、P5 8h、P6 H_vol 未过门、P7 风控全关、P8 T1）均保持默认，**唯一正式采纳的变动是 `Cap=4.0`** ——而它是"放大暴露换更高绝对收益/回撤（年化 +9.7pp、回撤 ≈4×），夏普基本持平"的**风险预算扩张**，并非夏普优化。
- **真实杠杆所在**：在**名义上限（`Cap`）** + **事件层选择（治理/事件选择，P9）**，而**非**日内择时（`entry_mode` 已证伪、P2）或持仓时长分化（`H_vol` 过拟合、P6）。这也解释了"朴素 1/N 等权谜题"式结论——加权/择时/分化类轴在 B0 上无增量。
- **失败回溯协议生效**：P2 全 7 模式 0/N 被正确诊断为"轴本身选错/弱轴"而非"基线神奇最优"，据此把杠杆转向 H_vol(P6)/治理(P9)，而非继续拧择时旋钮。
- **归一化 B(t-PIT) 待落地**：2026-07-11 已决策默认升 B，但生产 `poc_va.py` 仅产 A(rank) 系，B 变换仍只在研究脚本借壳运行；用户决策"暂不改动生产基准，待 B 的 edge 巩固后正式搬入 `norm="B"` 分支并解除借壳"——此为该轮**唯一未闭环项**，其余局部轴全部"保持 B0"。

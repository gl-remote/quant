## 1. 核心问题

R1 的“布林带收缩后直接突破”没有证明成本后有效，但止损放宽曾显示一定改善。R2 改为验证：

```text
布林带收缩后首次突破边界
→ 回踩边界附近但不重新接受压缩区
→ 再次向突破方向启动
```

要回答的问题是：突破后的边界回踩，是否比直接追突破更适合作为低验证成本价格区间。

## 2. 实验定义

| 项目 | 定义 |
| --- | --- |
| 候选价格区间 | 布林带收缩后被突破的上轨 / 下轨附近 |
| 传统解释 | 压缩后突破，回踩边界不破，边界可能完成支撑 / 阻力转换 |
| 结构塑形解释 | 不预测突破必然延续，只观察边界是否提供清晰接受 / 失败判断 |
| 方向假设 | 突破后回踩边界不重新接受原压缩区，再次向外启动 |
| 严格失败边界 | 多头为重新跌回突破上轨下方；空头为重新回到突破下轨上方 |
| 入场方式 | 首次突破后等待回踩触碰边界容忍区，再突破回踩后的极值入场 |
| 实际止损边界 | 入场到严格失败边界距离 × `stop_widen_multiplier` |
| 目标止盈 | 严格失败距离 × `take_profit_r` |
| 时间退出 | 入场后 `max_hold_bars` 根 1m K 未验证成功则退出 |

本轮最小实现只把多头作为主证据，`allow_short=false`。

## 5. 结论

R2 固定参数组未通过。

主要证据：

1. 交易机会不算极低频，但样本仍偏少：三个合约分别为 22 / 40 / 28 笔；
2. `stop_widen_multiplier` 没有稳定购买到胜率：
   - `DCE.m2601` 从 `1.0` 到 `1.5` 完全无改善；
   - `DCE.m2603` 胜率不变但净亏损扩大；
   - `DCE.m2605` 胜率从 38.46% 提升到 46.15%，净亏损收窄，但仍不足以覆盖成本；
3. 成本压力很高，单合约滑点 + 手续费合计约 8,019 / 14,514 / 10,729，足以吞掉小目标结构；
4. 改善只出现在 `DCE.m2605`，不满足“不依赖单一合约”的要求。

综合判断：

```text
布林带突破后回踩边界再启动
→ 比直接突破更少交易
→ 但没有稳定改善成本后期望
→ 止损放宽只在单一合约上改善，不能视为结构有效
```

## 6. 对后续研究有用的信息

不要继续围绕同一入口做大规模参数搜索。若继续研究布林带边界，优先改结构定义，而不是继续扩大 `stop_widen_multiplier`。

后续更值得检查的方向：

1. 严格失败边界是否应改为“回踩低点 / 高点”，而不是突破上轨 / 下轨；
2. 回踩条件是否过严或过松，导致样本质量不足；
3. `take_profit_r=1.0` 的小目标是否太接近成本，导致即使胜率改善也无法转正；
4. 是否应先过滤低波动窗口，避免目标空间天然不足以覆盖手续费和滑点；
5. 如果只在单一合约改善，不应视为结构有效，应先跨合约检查再进入参数邻域。

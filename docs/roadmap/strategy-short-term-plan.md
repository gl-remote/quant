# 策略短期实验规划

> 类型：Roadmap / 短期实验规划\
> 状态：执行中\
> 创建日期：2026-06-27\
> 研究框架：[策略长期共识：低验证成本价格区间下的结构塑形框架](./strategy-research-framework.md)

## 1. 短期目标

短期目标不是盲选 Donchian、布林带或其他入口信号，也不是继续扩大 MA/ATR 参数搜索，而是把研究框架落到一组可迭代的短期实验中。

短期实验不是单轮决策，而是多轮候选价格区间验证。每轮实验完成后，实验记录归档到 [`docs/archive/strategy-research`](../archive/strategy-research/)，短期计划只保留下一步实验方向和共同评价标准。

当前阶段实验优先验证一件事：

```text
是否存在一类低验证成本价格区间，
能够用清晰失败边界判断方向选择是否错误，
并且通过适度放宽止损提升胜率，
使胜率提升覆盖盈亏比下降和交易成本。
```

本阶段不要求立即得到可上线策略，也不直接追求“高胜率”。每一轮都必须先证明：候选价格区间确实比随机入场更容易低成本知道自己错了。

## 2. 实验哲学

本阶段延续 [策略长期共识：低验证成本价格区间下的结构塑形框架](./strategy-research-framework.md) 的交易哲学，采用“低验证成本价格区间验证”作为短期实验哲学。

| 组成      | 定位                       |
| ------- | ------------------------ |
| 候选价格区间  | 研究对象，不是预测信号              |
| 方向假设    | 用于定义交易方向，不假设能稳定预测未来      |
| 严格失败边界  | 理论上证明方向假设失败的位置           |
| 实际止损边界  | 为吸收正常噪声后设置的退出位置          |
| 止损放宽    | 用更低盈亏比购买胜率，必须事先定义        |
| 小目标止盈   | 用于观察高胜率低盈亏比结构是否成立        |
| 时间退出    | 入场后无法快速验证则退出，避免长期占用样本    |
| 冷却 / 暂停 | 连续亏损或异常波动后停止交易，保护小资金生存能力 |

核心判断不是：

```text
这个入场信号是否更准？
```

而是：

```text
这个价格区间是否更容易低成本判断方向选择是否失败？
止损放宽带来的胜率提升，是否足以覆盖盈亏比下降和成本？
```

## 3. 当前实验命名

当前优先实验命名为：

```text
low-validation-cost-r1
```

含义：

```text
用一类清晰价格结构作为候选区间，
定义方向假设、严格失败边界和实际止损边界，
验证该区间是否具备低验证成本，
以及止损放宽是否能改善成本后期望。
```

N 周期突破、布林带、均线回踩等都只是候选价格区间表达，不是研究主角。

## 4. 每轮实验核心问题

每轮实验按顺序回答以下问题：

```text
是否存在明确的候选价格区间
→ 该区间的方向假设是否清楚
→ 严格失败边界是否可定义
→ 入场到严格失败边界的验证成本是否足够低
→ 实际止损边界是否只是有限放宽
→ 放宽止损后胜率是否提高
→ 胜率提升是否覆盖盈亏比下降和成本
→ 单笔尾部亏损和亏损簇是否仍可承受
→ 收益是否不依赖单品种、单参数或单时段
```

如果这些问题无法回答，或回答是否定的，不进入参数优化。

## 5. 候选区间来源

### 5.1 成熟体系只作为结构语言

每轮可以借用成熟交易体系寻找候选区间，但不能直接采纳它们的预测解释。

| 来源体系                     | 可提取的候选区间                         | 本项目中的用途        |
| ------------------------ | -------------------------------- | -------------- |
| Price Action             | 前高前低、区间边界、趋势线、K 线反转区             | 定义接受 / 拒绝和失败边界 |
| Wyckoff                  | spring、upthrust、test、吸筹 / 派发区边界  | 定义假突破或测试失败     |
| Market Profile / Auction | VAH / VAL、POC、Initial Balance 边界 | 定义价值区接受 / 拒绝   |
| 均值回归                     | 布林带外、VWAP 偏离、Z-score 极端区         | 定义继续扩散即失败      |
| 突破交易                     | 通道边界、前高前低、压缩后突破位                 | 定义突破接受 / 失败    |
| 趋势回踩                     | 均线回踩、前突破位回踩、趋势通道回踩               | 定义趋势延续失败边界     |

每个候选区间都必须完成结构塑形翻译：

```text
传统交易理由是什么
→ 对应哪个价格结构
→ 方向假设是什么
→ 严格失败边界在哪里
→ 入场到失败边界的验证成本是多少
→ 是否需要放宽止损吸收正常噪声
→ 放宽止损后的胜率提升是否覆盖盈亏比下降和成本
```

### 5.2 当前优先候选

每轮只选一个最简单区间，不同时堆叠多个复杂体系。

当前优先候选：

| 候选区间      | 验证问题                        |
| --------- | --------------------------- |
| 布林带收缩后边界  | 压缩边界是否提供清晰失败点，短距离扩张是否足以覆盖成本 |
| 前高 / 前低附近 | 价格突破后接受或失败是否能快速判断           |
| 趋势背景下回踩区域 | 回踩区是否靠近趋势失效点，验证趋势延续成本是否较低   |
| 通道边界附近    | 通道延续或回落的失败边界是否清楚            |

当前优先从“布林带收缩后边界”或“趋势背景下回踩区域”二选一，避免一开始直接测试普通突破。

## 6. 实验结构

### 6.1 区间定义

每个实验必须显式定义：

| 项目     | 说明                  |
| ------ | ------------------- |
| 候选区间   | 例如布林带收缩后上轨 / 下轨附近   |
| 方向假设   | 例如突破边界后短距离扩张        |
| 严格失败边界 | 例如重新跌回压缩区内部、跌破回踩低点等 |
| 严格失败距离 | 入场价到严格失败边界的距离       |
| 实际止损边界 | 为吸收正常噪声后设置的止损位置     |
| 止损放宽倍数 | 实际止损距离 / 严格失败距离     |
| 目标止盈   | 固定小目标、波动倍数或区间目标     |
| 时间退出   | 未在预期时间内验证成功则退出      |

### 6.2 止损放宽测试

每轮至少比较两类设置：

| 设置       | 目的               |
| -------- | ---------------- |
| 严格失败边界止损 | 观察原始低验证成本区间的基础表现 |
| 适度放宽止损   | 观察吸收正常噪声后胜率是否提升  |

重点不是找到最优止损，而是回答：

```text
止损放宽是否真的购买到了足够胜率？
```

### 6.3 暂不处理

每轮不做：

- 大规模 Optuna 搜索；
- Walk-Forward；
- 仓位优化；
- 多体系条件叠加；
- 亏损后加仓摊低；
- 为了提高胜率取消或无限放宽止损；
- 为了凑交易数放宽候选区间定义；
- 根据单次收益结果频繁切换候选区间。

## 7. 评价顺序

每轮按以下顺序判断：

```text
候选价格区间是否清晰
→ 严格失败边界是否可定义
→ 严格失败距离是否足够低
→ 实际止损放宽是否可控
→ 放宽后胜率是否提高
→ 盈亏比下降是否可接受
→ 成本后期望是否改善
→ 单笔尾部亏损是否可承受
→ 交易机会是否足够评价
→ 参数和品种是否不依赖单点偶然性
```

收益率、夏普、最大回撤和最优参数只作为后置参考，不作为每轮主证据。

## 8. 关键观察指标

每轮实验完成后，需形成实验记录并归档到 [`docs/archive/strategy-research`](../archive/strategy-research/)。

| 字段          | 说明                          |
| ----------- | --------------------------- |
| 实验版本        | 例如 `low-validation-cost-r1` |
| 候选价格区间      | 例如布林带收缩后边界                  |
| 结构来源        | Price Action、布林带、趋势回踩等      |
| 传统解释        | 该体系通常如何解释这个位置               |
| 结构塑形解释      | 本项目为什么认为它是低验证成本区间           |
| 方向假设        | 本轮交易方向假设                    |
| 严格失败边界      | 理论上方向假设失效的位置                |
| 严格失败距离      | 入场到严格失败边界的距离                |
| 实际止损边界      | 真实止损位置                      |
| 实际止损距离      | 入场到实际止损的距离                  |
| 止损放宽倍数      | 实际止损距离 / 严格失败距离             |
| 候选信号数       | 区间定义后产生的候选机会                |
| 实际成交数       | 真实开仓次数和完整交易数                |
| 严格止损胜率      | 使用严格失败边界止损时的胜率              |
| 放宽止损胜率      | 使用实际止损边界时的胜率                |
| 胜率提升        | 放宽止损后胜率提升幅度                 |
| 盈亏比下降       | 放宽止损后平均亏损扩大幅度               |
| 盈亏平衡胜率      | 根据盈亏比和成本计算的最低胜率             |
| 胜率安全边际      | 实际胜率 - 盈亏平衡胜率               |
| 平均盈利        | 金额、tick、相对成本倍数              |
| 平均亏损        | 金额、tick、相对平均盈利倍数            |
| 最大单笔亏损      | 会回吐多少笔平均盈利                  |
| 亏损簇         | 最大连续亏损次数、最差连续亏损金额           |
| 成本影响        | 成本 / 平均盈利、毛收益到净收益差异         |
| MAE         | 进场后最大不利波动，用于判断止损是否合理        |
| MFE         | 进场后最大有利波动，用于判断止盈是否过早或过晚     |
| 期望净变化       | 止损放宽后成本后期望是否改善              |
| exit reason | 止盈、严格失败、实际止损、时间退出、冷却暂停等分布   |
| 结论          | 区间有效、需调整边界、需调整止损放宽、还是淘汰     |

## 9. 通过标准

每轮不是要求立即盈利，而是要求看到“低验证成本区间 + 止损放宽”的结构塑形迹象。

| 维度    | 通过标准                           |
| ----- | ------------------------------ |
| 区间清晰度 | 候选区间可以客观定义，不依赖主观画线             |
| 失败边界  | 严格失败边界清楚，且距离入场较近               |
| 止损放宽  | 放宽倍数有限，且事先定义                   |
| 胜率提升  | 放宽止损后胜率有明确提升                   |
| 期望净变化 | 胜率提升覆盖盈亏比下降和成本                 |
| 单笔质量  | 平均盈利显著大于手续费和滑点                 |
| 尾部亏损  | 最大亏损不会回吐过多平均盈利                 |
| 亏损簇   | 连续亏损在小资金可承受范围内                 |
| 交易机会  | 不依赖极低频单笔样本，也不靠放宽区间定义凑交易数       |
| 参数邻域  | 边界和止损放宽小幅变化不应断崖式退化             |
| 可解释性  | 能说明收益来自价格区间、失败边界、止损放宽、止盈还是时间退出 |

未通过时，不继续围绕同一入口调参，而是先判断失败来自哪一层：

```text
候选区间无法客观定义
→ 严格失败边界不清楚
→ 验证距离过大
→ 放宽止损没有提升胜率
→ 胜率提升不足以覆盖盈亏比下降
→ 平均盈利太接近成本
→ 止损放宽后尾部亏损不可承受
→ 品种或周期不适合小资金
```

## 10. 短期执行顺序

```text
实现 low-validation-cost-r1
→ 选择一个候选价格区间：布林带收缩后边界，或趋势背景下回踩区域
→ 完成结构塑形翻译：传统解释、方向假设、严格失败边界
→ 跑严格失败边界止损版本
→ 跑适度放宽止损版本
→ 输出严格失败距离、实际止损距离、止损放宽倍数
→ 输出严格止损胜率、放宽止损胜率、胜率提升、盈亏比下降
→ 输出盈亏平衡胜率、胜率安全边际、成本后期望净变化
→ 判断低验证成本区间是否有效
→ 若有效，再做边界邻域和多窗口检查
→ 若无效，先更换候选价格区间，而不是扩大参数搜索
```

## 11. 区间替换原则

只有在当前区间完成完整诊断后，才比较不同候选区间。

区间替换的目的不是寻找神奇信号，而是寻找更适合结构塑形的低验证成本实验场。

可替换区间包括：

| 区间                | 用途                 |
| ----------------- | ------------------ |
| 布林带收缩后边界          | 检查压缩边界是否提供低成本失败判断  |
| 前高 / 前低附近         | 检查突破接受 / 失败是否能快速判断 |
| 趋势背景下回踩区域         | 检查趋势延续失败边界是否足够近    |
| 通道边界附近            | 检查通道延续或回落的验证距离是否较短 |
| 支撑 / 阻力 / 密集成交区边缘 | 检查价格接受 / 拒绝是否有清晰边界 |

若收益分布只在单一入口、单一参数、单一品种或单一时段上成立，不视为结构有效。

## 12. 暂停条件

出现以下情况时，不继续投入当前结构：

1. 找不到可客观定义的低验证成本价格区间；
2. 严格失败边界无法定义，或距离入场过远；
3. 放宽止损不能带来足够胜率提升；
4. 胜率提升无法覆盖盈亏比下降和成本；
5. 平均盈利太小，成本 / 平均盈利过高；
6. 单笔最大亏损会回吐过多平均盈利；
7. 亏损簇超出小资金可承受范围；
8. 为维持胜率需要无限放宽止损、加仓摊低或长期扛亏；
9. 交易数只能靠放宽区间定义获得；
10. 收益主要来自单合约、单时段或尖峰参数；
11. 无法解释收益分布改善来自哪一层结构。

## 13. 近期产出物

| 产出物                           | 说明                               |
| ----------------------------- | -------------------------------- |
| `low-validation-cost-r1` 策略实现 | 候选价格区间 + 严格失败边界 + 可控止损放宽的最小可运行版本 |
| 固定参数组回测结果                     | 不使用大规模优化                         |
| 区间定义说明                        | 候选区间、传统解释、结构塑形解释                 |
| 失败边界分析                        | 严格失败边界、严格失败距离、实际止损边界             |
| 止损放宽分析                        | 止损放宽倍数、胜率提升、盈亏比下降                |
| 期望净变化分析                       | 盈亏平衡胜率、胜率安全边际、成本后期望变化            |
| 尾部亏损分析                        | 最大亏损 / 平均盈利、亏损簇、异常退出             |
| 成本敏感性分析                       | 成本 / 平均盈利、滑点上升后的稳定性              |
| MAE / MFE 分析                  | 判断止损和止盈设置是否合理                    |
| 方向判断                          | 区间有效、调整边界、调整止损放宽，或暂停方向           |

> 创建日期：2026-06-27

## 1. 核心共识

在当前数据规模、算力条件、资金规模和研究资源下，本项目不以原创 alpha discovery 为主线，也不适合采用依赖广泛分散和长期大数定律的传统 CTA 右尾框架。

长期方向从：

```text
寻找符合目标约束的 alpha 因子
```

调整为：

```text
承认我们不能稳定预测未来市场变化
→ 寻找能够低成本验证方向选择的特殊价格区间
→ 在这些区间建立明确的失败判定和退出边界
→ 通过适度放宽止损换取胜率提升
→ 在胜率提升、盈亏比下降和成本之间塑造正期望结构
```

结构塑形仍然是主线，但具体目标不是“低胜率高盈亏比、靠少数大趋势覆盖多数小亏”，也不是“靠更强信号预测未来”，而是：

```text
低验证成本入场 × 明确失败边界 × 可控止损放宽 × 胜率安全边际 × 成本后正期望
```

这更符合小资金约束：账户不能长期等待少数右尾行情，也无法依赖大量品种分散来平滑亏损簇；但可以优先寻找那些“更容易知道自己错了”的价格区间，在这些区间用更低验证成本做方向选择。

## 2. 为什么不做原创 alpha discovery

| 原因 | 说明 |
|------|------|
| 数据不足 | 当前样本长度和品种覆盖不足以支持可靠的新因子发现 |
| 算力有限 | 大规模特征搜索、组合筛选和稳健性验证成本过高 |
| 资金有限 | 账户无法承受大量试错、长亏损簇和广泛分散前的波动 |
| 过拟合风险高 | 参数搜索容易在短样本中拟合噪声 |
| 归因困难 | 复杂组合信号很难解释收益来源和失效原因 |
| 与当前优势不匹配 | 当前更适合做回测框架、成本约束、风控、报告和执行层面的工程化验证 |

因此，参数搜索、复杂指标组合和多重过滤不应被当作发现 alpha 的工具。Optuna / Bayesian search 只能在结构已经成立后用于局部参数平台验证，不能承担“创造策略优势”的职责。

## 3. 我们选择的交易哲学

本项目长期采用以下交易哲学作为主线：

```text
不假设自己能稳定预测未来市场方向；
假设市场存在某些特殊价格区间，在这些区间更容易判断方向选择是否失败；
入场点的价值不在于预测更准，而在于验证成本更低、失败边界更清楚；
在低验证成本的前提下，适度放宽止损，用较低盈亏比换取更高胜率；
最终通过胜率、盈亏比、成本和尾部亏损之间的结构塑形获得正期望。
```

对应的成熟思想包括：

| 体系 | 与本项目的关系 |
|------|----------------|
| 结构塑形 | 不追求神奇信号，而是设计胜率、平均盈利、平均亏损和成本之间的关系 |
| 低验证成本入场 | 选择失败边界清晰、验证距离短的价格区间，而不是追求预测未来 |
| 高胜率交易 | 通过低验证成本价格区间和止损容忍度提升命中率，接受较低盈亏比 |
| 风险优先 | 小资金首要目标是避免尾部亏损和连续亏损导致账户失去继续试错能力 |
| 状态过滤 | 只在更容易验证方向选择的市场状态下交易，宁可少交易 |
| 资金生存 | 策略必须先适配账户规模、保证金、滑点和连续亏损承受力 |

## 4. 正期望来源

后续研究围绕收益分布拆解：

```text
成本后期望 = 胜率 × 平均盈利 - 败率 × 平均亏损 - 单笔成本
```

本框架下，胜率提升不来自“预测未来更准”，而来自两层结构：

```text
特殊价格区间降低方向验证成本
+ 适度放宽止损吸收正常噪声
```

若平均盈利小于平均亏损，则最低胜率门槛会显著上升：

```text
盈亏比 = 平均盈利 / 平均亏损
最低胜率 = 1 / (1 + 盈亏比)
```

例如：

| 盈亏比 | 不含成本的最低胜率 |
|--------|--------------------|
| 0.5 | 66.7% |
| 0.6 | 62.5% |
| 0.7 | 58.8% |
| 0.8 | 55.6% |

加入手续费、滑点和偶发跳空后，真实胜率要求还要更高。因此，放宽止损只有在带来的胜率提升超过盈亏平衡胜率上升时才有意义。

## 5. 关键理论约束

### 5.1 大数定律可能不站在我们这边

传统 CTA 依赖大量品种、长周期、多市场状态和长期重复交易。有限资金下，我们可能无法同时满足：

- 足够多的品种；
- 足够低的单品种仓位；
- 足够长的亏损承受期；
- 足够多的独立交易机会；
- 足够低的交易成本占比。

因此，不能把“长期来看右尾会覆盖亏损”作为默认前提。策略必须在更短交易序列内体现可承受性，否则账户可能在理论优势兑现前已经失败。

### 5.2 入场点是验证位置，不是预测位置

本框架不认为某个入场点能稳定预测未来涨跌。入场点的核心价值是：

```text
用更小的代价验证方向选择是否错误
```

一个好的入场区域应满足：

- 方向假设清楚；
- 失败边界清楚；
- 从入场到失败边界的距离较短；
- 正常噪声与真正失败可以区分；
- 一旦失败，可以快速退出，不需要长期扛亏；
- 成功时的可捕获空间足以覆盖成本和止损放宽带来的盈亏比下降。

因此，后续讨论入场不应问：

```text
这个点是不是能预测上涨 / 下跌？
```

而应问：

```text
这个价格区间是否更容易低成本判断方向选择是否失败？
```

### 5.3 特殊价格区间是研究对象

我们假设市场中存在一些特殊价格区间，在这些区域做方向选择比随机价格更容易知道成败。

候选价格区间包括：

| 区间 | 可能优势 |
|------|----------|
| 前高 / 前低附近 | 突破或失败的判定更清楚 |
| 通道边界附近 | 方向延续或回落的验证距离较短 |
| 布林带收缩后边界 | 压缩后扩张或假突破较容易定义 |
| 趋势背景下回踩区域 | 趋势延续失败边界相对清晰 |
| 波动归一后的异常压缩区 | 小止损即可验证是否进入扩张 |
| 支撑 / 阻力 / 密集成交区边缘 | 价格接受或拒绝更容易观察 |

这些区间不是 alpha 因子本身，而是更适合做方向验证的“实验场”。

### 5.4 成熟交易体系只提供结构语言，不提供预测依据

Price Action、Wyckoff、Market Profile、均值回归、突破交易、Scalping 等成熟体系可以连接到本框架，但连接方式不是直接采纳它们的方向预测解释。

本项目使用这些体系的目的有两个：

1. 防止研究重新滑回“寻找更准信号”；
2. 借用成熟体系的价格结构语言，有目的地寻找更多低验证成本价格区间。

因此，成熟交易体系在本项目中的定位是：

```text
价格结构语言库
→ 候选低验证成本区间来源
→ 失败边界定义工具
→ 收益分布塑形素材
```

而不是：

```text
方向预测理论
→ 直接交易理由
→ alpha 因子来源
```

不同交易体系需要被翻译成结构塑形语言：

| 成熟体系 | 常见传统解释 | 本项目中的结构塑形解释 | 可提取的候选区间 |
|----------|--------------|------------------------|------------------|
| Price Action | 支撑反弹、阻力回落、形态确认 | 关键位附近更容易观察价格接受 / 拒绝，失败边界较清楚 | 前高前低、区间边界、趋势线、K 线反转区 |
| Wyckoff | Spring / Upthrust 说明供需转换 | 假突破或测试失败提供清晰的方向失败证据 | 吸筹 / 派发区边界、spring、upthrust、test |
| Market Profile / Auction | 价值区接受或拒绝 | 价值区边缘能观察市场是否接受更高 / 更低价格 | VAH / VAL、POC、Initial Balance 边界 |
| 均值回归 | 极端偏离后价格回归均值 | 极端偏离区可以定义“继续扩散即失败”的边界 | 布林带外、VWAP 偏离、Z-score 极端区 |
| 突破交易 | 突破后趋势延续 | 突破边界能观察市场是否接受新价格，失败时退出边界清楚 | 通道边界、前高前低、压缩后突破位 |
| 趋势回踩 | 回踩后趋势继续 | 回踩区域距离趋势失效点较近，验证趋势延续成本较低 | 均线回踩、前突破位回踩、趋势通道回踩 |
| Scalping / 微结构 | 短期订单流有方向优势 | 短期结构能快速验证对错，适合小目标和严格退出 | 日内高低点、盘口失衡、短周期区间边缘 |

后续引用任何成熟体系时，都必须完成一次“结构塑形翻译”：

```text
该体系的传统交易理由是什么
→ 它对应哪个价格结构
→ 这个结构的方向假设是什么
→ 严格失败边界在哪里
→ 入场到失败边界的验证成本是多少
→ 是否需要放宽止损吸收正常噪声
→ 放宽止损后的胜率提升是否覆盖盈亏比下降和成本
```

如果一个成熟体系只能提供“这里可能涨 / 跌”的解释，但不能帮助定义低验证成本区间、失败边界、止损放宽空间和成本后期望，则不应纳入本项目主线。

### 5.5 放宽止损是购买胜率，不是放弃风控

在低验证成本价格区间中，可以有意提高止损距离，以吸收正常噪声、减少过早出局，从而换取一定胜率提升。

但这必须被视为一笔明确成本：

```text
更宽止损 = 更低盈亏比 + 更高盈亏平衡胜率
```

因此，止损放宽必须满足：

- 放宽幅度事先定义，不能盘中随意扩大；
- 放宽后单笔最大亏损仍在账户承受范围内；
- 胜率提升必须超过盈亏平衡胜率上升；
- 最大亏损 / 平均盈利 不能恶化到吞掉大量小盈利；
- 不能通过长期扛亏、加仓摊低或取消止损来换胜率。

需要记录：

| 指标 | 用途 |
|------|------|
| 严格失败边界 | 方向假设理论上失效的位置 |
| 实际止损边界 | 为吸收噪声后设置的退出位置 |
| 止损放宽倍数 | 实际止损距离 / 严格失败距离 |
| 胜率提升 | 放宽止损后胜率增加多少 |
| 盈亏比下降 | 放宽止损后平均亏损扩大多少 |
| 期望净变化 | 胜率提升是否覆盖盈亏比下降和成本 |

### 5.6 高胜率不是免费午餐

高胜率通常伴随低盈亏比，核心风险是少数亏损单吞掉大量盈利单。

必须长期记录：

- 胜率是否稳定高于成本后盈亏平衡胜率；
- 单笔最大亏损相当于多少笔平均盈利；
- 是否存在一次亏损回吐多日盈利；
- 亏损是否集中出现在特定市场状态；
- 是否存在滑点、跳空或流动性导致的超预期亏损。

高胜率结构的死亡方式通常不是“慢慢亏”，而是“长期小赚后被少数大亏打穿”。

### 5.7 尾部亏损是第一风险

低盈亏比策略必须把尾部亏损作为第一约束。

禁止把以下做法当成提高胜率的手段：

- 亏损后加仓摊低；
- 无限放宽止损等待回归；
- 用浮亏换胜率；
- 在异常波动中继续补仓；
- 为了维持高胜率不承认失效。

高胜率可以接受低盈亏比，但不能接受不受控的左尾。

### 5.8 交易频率要服务于验证质量，不服务于凑样本

交易数太少无法评价，但为了凑交易数降低场景质量也会破坏本框架。

后续应优先寻找：

```text
少而清晰、失败边界明确、验证成本低的交易场景
```

而不是：

```text
大量边缘信号 + 依赖大数定律修复
```

交易机会不足时，应先判断是否真的存在适合小资金的低验证成本场景，而不是继续放宽入口。

### 5.9 成本门槛更严格

高胜率低盈亏比结构的单笔盈利较小，因此成本侵蚀更敏感。

每个候选结构必须检查：

- 成本 / 平均盈利；
- 成本 / 平均亏损；
- 成本后胜率是否仍高于盈亏平衡胜率；
- 滑点上升后是否立刻失效；
- 小盈利是否只是被成本前收益伪装出来。

如果平均盈利只略高于手续费和滑点，该结构不适合作为主线。

### 5.10 状态过滤用于寻找低验证成本区间

在本框架下，状态过滤不是为了预测未来，而是为了找到更适合验证方向选择的价格环境。

状态过滤应回答：

- 当前是否处于适合定义失败边界的价格结构；
- 当前波动是否足以覆盖成本，但又没有高到使止损失控；
- 当前是否避开跳空、快速扩散等高尾部风险状态；
- 当前是否能减少低质量边缘交易，而不是只提高历史回测收益。

### 5.11 止盈可以更主动，但不能制造脆弱性

高胜率低盈亏比结构允许更主动地兑现利润，但主动止盈不能建立在“亏损无限忍耐”的基础上。

合理方向：

- 小目标止盈；
- 分时或时间窗口内快速退出；
- 达到预期波动后退出；
- 不达预期则时间止损；
- 亏损触发明确退出。

错误方向：

- 小赚就跑，大亏硬扛；
- 止盈固定，止损动态放宽；
- 为了提高胜率取消止损；
- 亏损单持仓时间显著长于盈利单且无明确上限。

### 5.12 参数平台仍然重要

高胜率结构更容易被过拟合成漂亮胜率。若有效，它不应只在单个阈值、单个品种、单个时段成立。

应检查：

- 目标止盈略微变化后胜率是否断崖；
- 止损距离略微变化后期望是否反转；
- 严格失败边界和实际止损边界的小幅变化是否仍有相似结构；
- 时间退出参数变化后是否仍有相似结构；
- 不同品种是否需要完全不同规则；
- 优势是否只来自某个局部行情。

## 6. 目标约束的角色

交易频率、持仓周期、成本占比和品种一致性是策略准入条件，不是因子生成条件。

在有限资金条件下，目标约束需要重新排序：

```text
资金生存能力
→ 低验证成本价格区间是否存在
→ 严格失败边界是否清楚
→ 实际止损边界是否可承受
→ 止损放宽是否换来足够胜率提升
→ 成本后盈亏平衡胜率
→ 实际胜率安全边际
→ 交易机会是否足够
→ 参数平台和多窗口验证
```

错误顺序：

```text
先规定交易频率
→ 再放宽信号凑样本
→ 靠大数定律假设长期会修复
```

正确顺序：

```text
先确认资金能承受单笔和连续亏损
→ 再寻找低验证成本价格区间
→ 再定义严格失败边界和实际止损边界
→ 再验证止损放宽是否提高成本后期望
→ 最后才扩大样本和参数验证
```

## 7. 长期评价框架

后续策略实验统一按以下顺序判断：

```text
是否符合有限资金约束
→ 是否存在明确的低验证成本价格区间
→ 严格失败边界是否清楚
→ 实际止损边界是否可承受
→ 止损放宽是否带来足够胜率提升
→ 成本后盈亏平衡胜率是多少
→ 实际胜率是否有足够安全边际
→ 平均盈利是否显著大于成本
→ 尾部亏损是否受控
→ 交易机会是否足够评价
→ 参数是否存在稳定平台
→ 多窗口验证
```

收益率、夏普、最大回撤和最优参数可以参考，但不能替代验证成本 / 胜率 / 盈亏比 / 尾部亏损 / 成本安全边际诊断。

优先诊断指标包括：

| 指标 | 用途 |
|------|------|
| 严格失败距离 | 入场到理论失效位置的距离，衡量验证成本 |
| 实际止损距离 | 入场到真实止损位置的距离，衡量实际风险 |
| 止损放宽倍数 | 实际止损距离 / 严格失败距离 |
| 成本后胜率 | 判断是否真的具备高胜率特征 |
| 盈亏平衡胜率 | 根据盈亏比和成本计算最低要求 |
| 胜率安全边际 | 实际胜率 - 盈亏平衡胜率 |
| 最大单笔亏损 / 平均盈利 | 判断一次亏损会回吐多少笔盈利 |
| 亏损簇 | 评估账户生存能力 |
| 成本 / 平均盈利 | 判断小盈利是否被成本吞噬 |
| MAE | 观察入场后最大不利波动，评估止损是否合理 |
| MFE | 观察入场后最大有利波动，评估止盈是否过早或过晚 |
| exit reason 分布 | 判断收益来自止盈、止损、时间退出还是异常退出 |

## 8. 模块角色定位

| 模块 | 后续定位 |
|------|----------|
| MA | 可作为趋势背景或低验证成本区间的状态描述，不继续作为主线调参对象 |
| ATR | 保留为波动归一、止损距离、异常波动过滤和成本门槛判断模块 |
| MACD/KDJ | 仅作为辅助状态描述，避免叠加成过拟合确认 |
| DMI/ADX | 可作为趋势/震荡状态过滤器，但必须证明能降低验证成本或提高成本后期望 |
| Optuna/Bayesian search | 只在结构、胜率安全边际和尾部约束初步成立后启用 |
| Walk-Forward | 只在样本量和参数平台初步成立后启用 |

## 9. 长期退出条件

策略主线满足以下任一情况，应暂停或降级为 baseline：

1. 找不到清晰的低验证成本价格区间；
2. 严格失败边界无法定义，或定义后交易机会过少；
3. 放宽止损不能带来足够胜率提升；
4. 成本后胜率无法稳定高于盈亏平衡胜率；
5. 平均盈利过小，成本 / 平均盈利过高；
6. 单笔最大亏损会回吐过多平均盈利；
7. 亏损簇超出账户承受范围；
8. 为了维持高胜率需要无限放宽止损、加仓摊低或长期扛亏；
9. 正收益依赖单合约、单时段或尖峰参数；
10. 参数附近没有稳定平台，轻微扰动即断崖式退化；
11. 无法解释收益来自价格区间、失败边界、止损放宽、止盈、时间退出、成本还是市场状态变化。

## 10. 总结

后续研究应坚持：

```text
不做原创 alpha discovery，承认无法稳定预测未来；
不依赖广泛分散和长期大数定律，先保证账户生存；
不把入场当预测点，而把入场当低成本验证方向选择的价格区间；
不追求低胜率高盈亏比，转向高胜率低盈亏比但尾部受控；
不靠无限扛亏提高胜率，止损放宽必须事先定义且可度量；
不只看胜率，必须同时看验证成本、盈亏平衡胜率、成本、安全边际和亏损簇；
不为凑样本放宽场景，优先寻找少而清晰的低验证成本交易机会；
先验证资金约束和收益分布，再做参数搜索和多窗口验证。
```

短期实验应服务于这套长期共识：先验证“低验证成本价格区间 + 适度止损放宽”是否能改善成本后期望，再讨论具体入口信号和参数优化。

## 1. 核心共识

在当前数据规模、算力条件、资金规模和研究资源下，本项目不以原创 alpha discovery 为主线，也不适合采用依赖广泛分散和长期大数定律的传统 CTA 右尾框架。

长期方向从：

```text
寻找符合目标约束的 alpha 因子
```

调整为：

```text
承认我们不能稳定预测未来市场变化
→ 寻找市场共同关注、便于验证方向选择失败的共识价格区间
→ 在这些区间定义严格失败边界和短期盈利上界
→ 将严格失败边界换算为 2%~3% 单次账户风险
→ 先评估价格原始盈亏比和账户原始盈亏比
→ 再通过主动止盈、时间退出和有限止损放宽做胜率 / 盈亏比转化
→ 在长期成本后期望和安全边际之间塑造正期望结构
```

结构塑形仍然是主线，但具体目标不是“低胜率高盈亏比、靠少数大趋势覆盖多数小亏”，也不是“靠更强信号预测未来”，而是：

```text
共识价格区间 × 明确失败边界 × 短期盈利上界 × 2%~3% 账户风险预算 × 价格/账户原始盈亏比 × 胜率/盈亏比转化效率 × 成本后长期正期望
```

这更符合小资金约束：账户不能长期等待少数右尾行情，也无法依赖大量品种分散来平滑亏损簇；但可以优先寻找市场共同关注、失败边界和盈利上界相对清晰的共识价格区间，在这些区间用明确账户风险预算做方向选择实验。

## 2. 我们对 Alpha 与 Beta 的选择

本项目并不是不追求 alpha，而是不以原创预测型 alpha discovery 为主线；同时也不把主要收益来源建立在长期 beta 暴露、广泛分散和大数定律兑现之上。

| 路径 | 本项目选择 | 原因 |
|------|------------|------|
| 原创预测型 Alpha | 不作为主线 | 当前样本长度、品种覆盖、算力和研究资源不足以支持可靠的新因子发现；参数搜索、复杂指标组合和多重过滤容易在短样本中拟合噪声，且收益归因困难 |
| 长期 Beta / 风险溢价 | 不作为主线 | 小资金账户难以承受长期暴露带来的长亏损簇、尾部回撤和广泛分散前的波动，也无法依赖大量品种和长期大数定律平滑收益路径 |
| 结构型 Alpha | 作为主线 | 当前更适合围绕共识价格区间、明确失败边界、短期盈利上界、账户风险预算、成本约束和执行质量，验证是否存在可重复的成本后正期望结构 |

因此，参数搜索、复杂指标组合和多重过滤不应被当作发现 alpha 的工具。Optuna / Bayesian search 只能在结构已经成立后用于局部参数平台验证，不能承担“创造策略优势”的职责。

本项目追求的 alpha，不是“更早预测未来方向”的信号优势，而是：在市场共同关注、失败边界清晰、验证成本可控的价格区间内，通过价格原始盈亏比、账户风险预算、胜率 / 盈亏比转化和成本后期望，塑造可验证的结构性优势。

## 3. 我们选择的交易哲学

本项目长期采用以下交易哲学作为主线。它由基本假设和推论组成，后续实验应优先检验推论是否成立，而不是把推论当成已证明事实。

基本假设：

```text
1. 我们不能稳定预测未来市场方向。
2. 市场可能存在某些特殊价格区间，在这些区间更容易判断方向选择是否失败。
3. 许多市场参与者也会关注这类特殊价格区间，因此这些区间可能形成更强的共识边界和接受 / 拒绝行为。
4. 价格不会无限单边运行，单次方向判断在有限持仓周期内的可捕获盈利存在短期上界。
5. 在具备稳定正期望和足够重复次数的前提下，每次相对独立的投机判断失败时，将账户损失控制在 2%~3% 区间，作为符合凯利公式思想和大数定律约束的资金管理假设。
```

可直接推出的研究原则：

```text
1. 入场点的价值不在于预测更准，而在于验证成本更低、失败边界更清楚。
2. 低验证成本只说明更容易知道自己错了，不等于结构具备正期望。
3. 因为盈利存在短期上界，入场前必须同时定义严格失败边界和预期盈利上界。
4. 价格原始盈亏比 = 预期盈利上界 / 入场到严格失败边界的验证成本。
5. 严格失败边界必须能换算为明确的单次账户风险，失败时账户损失应控制在 2%~3% 区间。
6. 账户原始盈亏比 = 预期账户盈利 / 单次账户风险，其中单次账户风险按 2%~3% 预算约束计算。
```

需要实证验证的市场行为假说：

```text
1. 特殊价格区间因为被更多市场参与者关注，可能出现更高的共识拥挤。
2. 共识拥挤可能吸引试探性交易、止损单和突破追单。
3. 这些行为可能导致假突破、反复穿越、快速回撤，并使订单、止损和持仓压力在区间附近堆积。
4. 当堆积压力被释放时，市场可能具备更强短期动能；一旦价格有效离开拥挤区，并被市场重新接受在区间外，方向选择可能被进一步确认。
5. 被确认后的方向可能出现一段明显价格变动。
```

策略选择偏向：

```text
1. 当前市场行为假说使研究优先借用 Wyckoff、Auction / Market Profile、Price Action failed breakout 等结构语言，关注共识边界、假突破、接受 / 拒绝和压力释放。
2. 优先研究假突破后的重新接受 / 拒绝、价值区边缘 failed auction、Initial Balance / 前日高低点突破失败、密集成交区边缘压力释放等结构。
3. 普通突破追随、普通指标边界突破、单纯均线趋势回踩和 MACD / KDJ / 均线等指标确认叠加，暂时降低优先级。
4. 布林带、普通 Donchian、均线、MACD / KDJ 等指标仍可作为诊断工具，用于观察压缩、扩张、趋势背景、动量变化和市场行为，但它们不是市场逻辑本身。
5. 这些成熟体系和指标只提供候选区间、接受质量、失败边界、趋势背景和盈利上界的结构语言，不直接提供方向预测依据。
```

策略评价约束：

```text
1. 接受 / 拒绝质量、盈利上界、账户风险预算和尾部亏损控制，是判断结构是否可交易的评价条件，不是基本假设。
2. 每个策略结构必须证明：在合约乘数、最小手数、滑点、跳空和止损放宽后，单次账户风险仍可稳定控制在 2%~3%。
3. 明显价格变动是否足以在 2%~3% 单次账户风险约束下形成优质价格原始盈亏比和账户原始盈亏比，必须通过数据验证。
4. 主动止盈、时间退出和止损放宽，只能在价格原始盈亏比和账户原始盈亏比均足够的结构上使用。
5. 止损放宽会扩大价格风险，因此必须同步降低仓位，不能让账户风险超过 2%~3%。
6. 止损放宽、主动止盈和时间退出，都是把原始盈亏比部分转化为胜率的策略选择；提高胜率通常必然牺牲盈亏比，因此重点评估转化效率、二者稳定性、长期期望和安全边际。
7. 若结构需要超过 2%~3% 的单次账户风险才能体现优势，则不符合当前资金管理假设。
```

对应的成熟思想包括：

| 体系 | 与本项目的关系 |
|------|----------------|
| 结构塑形 | 不追求神奇信号，而是设计胜率、价格原始盈亏比、账户原始盈亏比、成本和风险预算之间的关系 |
| 低验证成本入场 | 选择失败边界清晰、验证距离短的价格区间，用更低成本判断方向选择是否失败 |
| 共识边界与接受 / 拒绝 | 借用 Wyckoff、Auction / Market Profile、Price Action failed breakout 等结构语言，寻找候选区间、接受质量、失败边界和盈利上界 |
| 原始盈亏比 | 入场前同时定义严格失败边界和预期盈利上界，分别评估价格原始盈亏比和账户原始盈亏比 |
| 胜率 / 盈亏比转化 | 在原始盈亏比足够时，用止损放宽、主动止盈和时间退出把部分盈亏比转化为胜率，并评估转化效率、长期期望和安全边际 |
| 账户风险预算 | 单次独立判断失败时账户损失控制在 2%~3%，止损放宽必须同步降低仓位 |
| 风险优先 | 小资金首要目标是避免尾部亏损、连续亏损和超出风险预算的单笔亏损导致账户失去继续试错能力 |
| 状态过滤 | 只在共识边界、接受 / 拒绝、压力释放等市场行为可被观察和验证时交易，宁可少交易 |
| 资金生存 | 策略必须先适配账户规模、合约乘数、最小手数、保证金、滑点、跳空和连续亏损承受力 |

## 4. 正期望来源

后续研究围绕账户风险口径拆解收益分布，而不是只看价格信号是否盈利：

```text
成本后期望 = 胜率 × 平均账户盈利 - 败率 × 平均账户亏损 - 单笔成本
```

在本框架下，单笔平均亏损首先受账户风险预算约束：

```text
单次独立判断失败的账户损失 ≈ 2%~3%
```

因此，策略优势不来自“预测未来更准”，而来自以下结构是否同时成立：

```text
共识价格区间
→ 清晰严格失败边界
→ 可估算的短期盈利上界
→ 足够的价格原始盈亏比
→ 可执行的 2%~3% 单次账户风险
→ 足够的账户原始盈亏比
→ 通过主动止盈 / 时间退出 / 有限止损放宽进行胜率与盈亏比转化
→ 成本后长期正期望和安全边际
```

经过 `low-validation-cost-r1/r2` 多轮实验后，需要把“低验证成本”升级为四层检查：

```text
边界清楚
→ 接受 / 拒绝质量足够
→ 盈利上界和原始盈亏比足够
→ 账户风险预算可执行
```

仅有边界清楚和验证距离短，不足以说明结构有效。如果边界缺少市场共识、接受质量不足、盈利上界太近，或在合约乘数 / 最小手数 / 滑点约束下无法把失败损失控制在 2%~3%，则该结构不适合作为主线。

若主动止盈、时间退出或止损放宽用于提高胜率，必须接受盈亏比下降，并检查转化效率：

```text
价格原始盈亏比 = 预期盈利上界 / 入场到严格失败边界的验证成本
账户原始盈亏比 = 预期账户盈利 / 单次账户风险
盈亏平衡胜率 = 1 / (1 + 成本后盈亏比)
胜率安全边际 = 实际胜率 - 盈亏平衡胜率
```

例如：

| 盈亏比 | 不含成本的最低胜率 |
|--------|--------------------|
| 0.5 | 66.7% |
| 0.6 | 62.5% |
| 0.7 | 58.8% |
| 0.8 | 55.6% |

加入手续费、滑点、跳空和止损放宽后的仓位调整后，真实胜率要求还要更高。因此，胜率提升只有在长期成本后期望为正、胜率和盈亏比稳定、且安全边际足够时才有意义。

## 5. 关键理论约束

### 5.1 大数定律可能不站在我们这边

传统 CTA 依赖大量品种、长周期、多市场状态和长期重复交易。有限资金下，我们可能无法同时满足：

- 足够多的品种；
- 足够低的单品种仓位；
- 足够长的亏损承受期；
- 足够多的独立交易机会；
- 足够低的交易成本占比。

因此，不能把“长期来看右尾会覆盖亏损”作为默认前提。策略必须在更短交易序列内体现可承受性，否则账户可能在理论优势兑现前已经失败。

### 5.2 入场点是验证位置，不是预测位置

本框架不认为某个入场点能稳定预测未来涨跌。入场点的核心价值是：

```text
用更小的代价验证方向选择是否错误
```

一个好的入场区域应满足：

- 处于市场共同关注的价格区间，而不是只有指标计算意义；
- 方向假设清楚；
- 严格失败边界清楚；
- 从入场到严格失败边界的价格验证成本较低；
- 严格失败边界可以换算为 2%~3% 单次账户风险；
- 价格对边界的接受 / 拒绝证据足够强，不只是机械触碰指标线；
- 正常噪声与真正失败可以区分；
- 严格失败边界不应近到被正常噪声反复触及；
- 一旦失败，可以快速退出，不需要长期扛亏；
- 成功时的可捕获盈利上界清楚，足以形成可接受的价格原始盈亏比和账户原始盈亏比。

因此，后续讨论入场不应问：

```text
这个点是不是能预测上涨 / 下跌？
```

而应问：

```text
这个价格区间是否更容易低成本判断方向选择是否失败？
严格失败边界和盈利上界在哪里？
在 2%~3% 账户风险预算下，价格原始盈亏比和账户原始盈亏比是否足够？
```

### 5.3 特殊价格区间是研究对象

我们假设市场中存在一些特殊价格区间，在这些区域做方向选择比随机价格更容易知道成败。

候选价格区间包括：

| 区间 | 可能优势 |
|------|----------|
| 前高 / 前低附近 | 突破或失败的判定更清楚，但普通前 N 高低点需要额外证明接受质量 |
| 通道边界附近 | 方向延续或回落的验证距离较短 |
| 布林带收缩后边界 | 压缩后扩张或假突破较容易定义，但指标边界不等于市场共识边界 |
| 趋势背景下回踩区域 | 趋势延续失败边界相对清晰，但需要过滤趋势末端和弱重新接受 |
| 波动归一后的异常压缩区 | 小止损即可验证是否进入扩张 |
| 支撑 / 阻力 / 密集成交区边缘 | 价格接受或拒绝更容易观察 |
| Initial Balance / 前日高低点 / 开盘区间边界 | 更可能被市场共同观察，适合验证接受 / 拒绝和假突破 |
| VAH / VAL / POC 等价值区边缘 | 可观察市场是否接受更高 / 更低价值区，适合 Auction 结构翻译 |

这些区间不是 alpha 因子本身，而是更适合做方向验证的“实验场”。

### 5.4 成熟交易体系只提供结构语言，不提供预测依据

Price Action、Wyckoff、Market Profile、均值回归、突破交易、Scalping 等成熟体系可以连接到本框架，但连接方式不是直接采纳它们的方向预测解释。

本项目使用这些体系的目的有两个：

1. 防止研究重新滑回“寻找更准信号”；
2. 借用成熟体系的价格结构语言，有目的地寻找更多共识价格区间、接受 / 拒绝证据、严格失败边界和盈利上界。

因此，成熟交易体系在本项目中的定位是：

```text
价格结构语言库
→ 候选共识价格区间来源
→ 接受 / 拒绝质量描述工具
→ 严格失败边界和盈利上界定义工具
→ 账户风险预算和收益分布塑形素材
```

而不是：

```text
方向预测理论
→ 直接交易理由
→ alpha 因子来源
```

不同交易体系需要被翻译成结构塑形语言：

| 成熟体系 | 常见传统解释 | 本项目中的结构塑形解释 | 可提取的候选区间 |
|----------|--------------|------------------------|------------------|
| Price Action | 支撑反弹、阻力回落、形态确认 | 关键位附近更容易观察价格接受 / 拒绝，失败边界较清楚 | 前高前低、区间边界、趋势线、K 线反转区 |
| Wyckoff | Spring / Upthrust 说明供需转换 | 假突破或测试失败提供清晰的方向失败证据 | 吸筹 / 派发区边界、spring、upthrust、test |
| Market Profile / Auction | 价值区接受或拒绝 | 价值区边缘能观察市场是否接受更高 / 更低价格 | VAH / VAL、POC、Initial Balance 边界 |
| 均值回归 | 极端偏离后价格回归均值 | 极端偏离区可以定义“继续扩散即失败”的边界 | 布林带外、VWAP 偏离、Z-score 极端区 |
| 突破交易 | 突破后趋势延续 | 突破边界能观察市场是否接受新价格，失败时退出边界清楚 | 通道边界、前高前低、压缩后突破位 |
| 趋势回踩 | 回踩后趋势继续 | 回踩区域距离趋势失效点较近，验证趋势延续成本较低 | 均线回踩、前突破位回踩、趋势通道回踩 |
| Scalping / 微结构 | 短期订单流有方向优势 | 短期结构能快速验证对错，适合小目标和严格退出 | 日内高低点、盘口失衡、短周期区间边缘 |

后续引用任何成熟体系时，都必须完成一次“结构塑形翻译”：

```text
该体系的传统交易理由是什么
→ 它对应哪个共识价格结构
→ 这个结构的方向假设是什么
→ 严格失败边界在哪里
→ 入场到失败边界的价格验证成本是多少
→ 失败边界能否换算为 2%~3% 单次账户风险
→ 价格接受 / 拒绝证据是什么
→ 严格边界是否容易被正常噪声快速再触及
→ 预期盈利上界在哪里
→ 价格原始盈亏比和账户原始盈亏比是否足够
→ 是否需要用主动止盈 / 时间退出 / 有限止损放宽进行胜率与盈亏比转化
→ 转化后长期期望是否为正，安全边际是否足够
```

如果一个成熟体系只能提供“这里可能涨 / 跌”的解释，但不能帮助定义共识价格区间、接受 / 拒绝质量、严格失败边界、盈利上界、账户风险预算和成本后长期期望，则不应纳入本项目主线。

### 5.5 低验证成本陷阱

`low-validation-cost-r1/r2` 的多轮实验说明，低验证成本本身不是优势来源，它只是降低试错成本的必要条件。

后续必须显式排查四类陷阱：

| 陷阱 | 表现 | 处理 |
|------|------|------|
| 边界太近 | 严格失败距离很低，但持仓后很快被正常噪声反复触及 | 记录严格边界快速再触及率，若过高则不能靠放宽止损硬修 |
| 边界无共识 | 指标边界清楚，但价格没有表现出真实接受 / 拒绝 | 增加接受质量证据，优先测试更有市场共识的边界 |
| 盈利上界不足 | 小止损对应小盈利空间，价格原始盈亏比不足 | 入场前定义预期盈利上界，原始盈亏比不足则不进入实验 |
| 账户风险不可执行 | 价格结构看似合理，但合约乘数、最小手数、滑点或跳空使单次风险无法控制在 2%~3% | 在回测前做账户风险预算预筛，风险不可执行则不进入实验 |

因此，候选区间需要先通过：

```text
边界清楚
→ 接受 / 拒绝质量足够
→ 盈利上界和原始盈亏比足够
→ 账户风险预算可执行
```

再讨论主动止盈、时间退出、止损放宽和参数平台。

### 5.6 放宽止损是诊断工具，不是默认优化方向

在原始盈亏比足够的结构中，可以有意提高价格止损距离，以吸收正常噪声、减少过早出局，从而换取一定胜率提升。

但止损放宽应首先被视为胜率 / 盈亏比转化诊断工具，而不是默认优化方向。若放宽止损只提高胜率，却没有改善长期成本后期望、没有稳定转化效率，或只在单一合约改善，则应判定为购买噪声。

放宽止损必须被视为一笔明确成本：

```text
更宽价格止损 = 更低价格盈亏比 + 更高盈亏平衡胜率
```

同时，由于本框架采用 2%~3% 单次账户风险预算：

```text
更宽价格止损 → 必须同步降低仓位 → 账户风险不能超过 2%~3%
```

因此，止损放宽必须满足：

- 放宽幅度事先定义，不能盘中随意扩大；
- 放宽后单次账户风险仍在 2%~3% 预算内；
- 胜率提升必须覆盖盈亏比下降、仓位下降和交易成本；
- 胜率、盈亏比和转化效率在多窗口 / 多品种下稳定；
- 不能通过长期扛亏、加仓摊低或取消止损来换胜率。

需要记录：

| 指标 | 用途 |
|------|------|
| 严格失败边界 | 方向假设理论上失效的位置 |
| 实际止损边界 | 为吸收噪声后设置的退出位置 |
| 止损放宽倍数 | 实际止损距离 / 严格失败距离 |
| 仓位调整倍数 | 放宽止损后为维持 2%~3% 账户风险所需的仓位变化 |
| 胜率提升 | 放宽止损后胜率增加多少 |
| 价格盈亏比下降 | 放宽止损后价格层面的盈亏比下降多少 |
| 账户盈亏比变化 | 降仓后账户层面的盈亏比是否仍足够 |
| 期望净变化 | 胜率提升是否覆盖盈亏比下降、仓位下降和成本 |

### 5.7 高胜率不是免费午餐

高胜率通常伴随低盈亏比。若严格止损和 2%~3% 账户风险预算执行到位，核心问题不再只是“会不会出现单笔大亏”，而是：

```text
牺牲多少盈亏比
→ 换来多少胜率
→ 转化后长期成本后期望是否为正
→ 正期望是否有足够安全边际
```

必须长期记录：

- 胜率是否稳定高于成本后盈亏平衡胜率；
- 价格盈亏比和账户盈亏比是否稳定；
- 胜率 / 盈亏比转化效率是否稳定；
- 单次失败是否稳定控制在 2%~3% 账户风险预算内；
- 是否存在一次亏损回吐过多平均账户盈利；
- 亏损是否集中出现在特定市场状态；
- 是否存在滑点、跳空或流动性导致的超预算亏损。

高胜率结构的死亡方式通常不是“胜率不够高”，而是“胜率提升不值得所牺牲的盈亏比，或少数超预算亏损破坏长期正期望”。

### 5.8 尾部亏损是第一风险

低盈亏比策略必须把尾部亏损和超预算亏损作为第一约束。

禁止把以下做法当成提高胜率的手段：

- 亏损后加仓摊低；
- 无限放宽止损等待回归；
- 用浮亏换胜率；
- 在异常波动中继续补仓；
- 为了维持高胜率不承认失效；
- 止损放宽后不降低仓位，导致单次账户风险超过 2%~3%。

高胜率可以接受较低盈亏比，但不能接受不受控的左尾，也不能接受长期依赖超出账户风险预算的单笔风险。

### 5.9 交易频率要服务于验证质量，不服务于凑样本

交易数太少无法评价，但为了凑交易数降低场景质量也会破坏本框架。

后续应优先寻找：

```text
少而清晰、失败边界明确、验证成本低的交易场景
```

而不是：

```text
大量边缘信号 + 依赖大数定律修复
```

交易机会不足时，应先判断是否真的存在适合小资金的低验证成本场景，而不是继续放宽入口。

### 5.10 成本门槛更严格

高胜率低盈亏比结构的单笔盈利较小，因此成本侵蚀更敏感。

每个候选结构必须在回测前先做价格空间和账户风险预筛，检查：

- 预期盈利上界 / 单笔成本；
- 价格原始盈亏比是否足够；
- 账户原始盈亏比是否足够；
- 严格失败距离是否小到导致盈利上界天然不足；
- 合约乘数、最小手数、滑点和跳空是否使 2%~3% 风险预算不可执行；
- 最小可接受平均账户盈利是否显著大于手续费和滑点；

回测后继续检查：

- 成本 / 平均账户盈利；
- 成本 / 单次账户风险；
- 成本后胜率是否仍高于盈亏平衡胜率；
- 滑点上升后是否立刻失效；
- 小盈利是否只是被成本前收益伪装出来。

如果平均账户盈利只略高于手续费和滑点，或结构需要超出 2%~3% 单次账户风险才能体现优势，该结构不适合作为主线。

### 5.11 状态过滤用于寻找低验证成本区间

在本框架下，状态过滤不是为了预测未来，而是为了找到更适合验证方向选择、估算盈利上界和执行账户风险预算的价格环境。

状态过滤应回答：

- 当前是否处于市场共同关注的共识边界附近；
- 当前是否存在可观察的接受 / 拒绝、假突破、压力释放等市场行为；
- 当前是否处于适合定义严格失败边界的价格结构；
- 当前是否能估算短期盈利上界；
- 当前波动是否足以覆盖成本，但又没有高到使 2%~3% 账户风险预算不可执行；
- 当前是否避开跳空、快速扩散等高尾部风险状态；
- 当前是否能减少低质量边缘交易，而不是只提高历史回测收益。

### 5.12 止盈可以更主动，但不能制造脆弱性

在价格原始盈亏比和账户原始盈亏比足够的结构中，可以更主动地兑现利润，把部分盈亏比转化为胜率。但主动止盈不能建立在“亏损无限忍耐”或“超出账户风险预算”的基础上。

合理方向：

- 在预期盈利上界附近主动止盈；
- 分时或时间窗口内快速退出；
- 达到预期波动后退出；
- 不达预期则时间止损；
- 亏损触发严格失败边界或实际止损边界后明确退出；
- 止损放宽时同步降低仓位，保持 2%~3% 单次账户风险。

错误方向：

- 小赚就跑，大亏硬扛；
- 止盈固定，止损动态放宽但不降仓；
- 为了提高胜率取消止损；
- 亏损单持仓时间显著长于盈利单且无明确上限；
- 主动止盈后没有检查胜率 / 盈亏比转化效率和长期正期望。

### 5.13 参数平台仍然重要

高胜率结构更容易被过拟合成漂亮胜率。若有效，它不应只在单个阈值、单个品种、单个时段成立。

应检查：

- 严格失败边界小幅变化后，价格原始盈亏比是否仍足够；
- 预期盈利上界小幅变化后，账户原始盈亏比是否仍足够；
- 目标止盈略微变化后胜率是否断崖；
- 止损距离略微变化后期望是否反转；
- 止损放宽后按 2%~3% 风险预算降仓，长期期望是否仍为正；
- 严格失败边界和实际止损边界的小幅变化是否仍有相似结构；
- 时间退出参数变化后是否仍有相似结构；
- 不同品种是否需要完全不同规则；
- 优势是否只来自某个局部行情。

## 6. 目标约束的角色

交易频率、持仓周期、成本占比和品种一致性是策略准入条件，不是因子生成条件。

在有限资金条件下，目标约束需要重新排序：

```text
资金生存能力
→ 单次账户风险是否能控制在 2%~3%
→ 是否存在市场共同关注的共识价格区间
→ 严格失败边界是否清楚
→ 预期盈利上界是否可定义
→ 价格原始盈亏比是否足够
→ 账户原始盈亏比是否足够
→ 接受 / 拒绝质量是否足够
→ 实际止损边界是否可承受，放宽止损后是否同步降仓
→ 胜率 / 盈亏比转化效率是否稳定
→ 成本后长期期望和安全边际
→ 交易机会是否足够
→ 参数平台和多窗口验证
```

错误顺序：

```text
先规定交易频率
→ 再放宽信号凑样本
→ 再通过止损放宽追求表面胜率
→ 靠大数定律假设长期会修复
```

正确顺序：

```text
先确认账户能承受单笔和连续亏损
→ 再确认严格失败边界能映射到 2%~3% 单次账户风险
→ 再寻找共识价格区间和接受 / 拒绝证据
→ 再定义严格失败边界和预期盈利上界
→ 再评估价格原始盈亏比和账户原始盈亏比
→ 再设计主动止盈 / 时间退出 / 有限止损放宽
→ 再验证胜率 / 盈亏比转化效率、长期正期望和安全边际
→ 最后才扩大样本和参数验证
```

## 7. 长期评价框架

后续策略实验统一按以下顺序判断：

```text
是否符合有限资金约束
→ 单次账户风险是否能控制在 2%~3%
→ 是否存在明确的共识价格区间
→ 严格失败边界是否清楚
→ 预期盈利上界是否可定义
→ 价格原始盈亏比是否足够
→ 账户原始盈亏比是否足够
→ 严格失败边界快速再触及率是否可接受
→ 价格接受 / 拒绝质量是否足够
→ 实际止损边界是否可承受，放宽后是否同步降仓
→ 胜率 / 盈亏比转化效率是否稳定
→ 成本后长期期望是否为正
→ 正期望是否有足够安全边际
→ 平均账户盈利是否显著大于成本
→ 尾部亏损和超预算亏损是否受控
→ 交易机会是否足够评价
→ 参数是否存在稳定平台
→ 多窗口验证
```

收益率、夏普、最大回撤和最优参数可以参考，但不能替代验证成本 / 原始盈亏比 / 账户风险预算 / 胜率转化效率 / 成本安全边际诊断。

优先诊断指标包括：

| 指标 | 用途 |
|------|------|
| 严格失败距离 | 入场到理论失效位置的价格距离，衡量验证成本 |
| 严格边界快速再触及率 | 判断边界是否近到只是在承受正常噪声 |
| 接受 / 拒绝质量 | 判断价格是否真的在边界处形成结构证据，而不是机械触碰指标线 |
| 预期盈利上界 | 估算单次方向判断在有限持仓周期内的可捕获空间 |
| 价格原始盈亏比 | 预期盈利上界 / 严格失败距离，判断价格结构是否有足够空间 |
| 单次账户风险 | 严格失败边界触发时的账户损失，目标控制在 2%~3% |
| 账户原始盈亏比 | 预期账户盈利 / 单次账户风险，判断资金口径是否成立 |
| 实际止损距离 | 入场到真实止损位置的距离，衡量实际价格风险 |
| 止损放宽倍数 | 实际止损距离 / 严格失败距离 |
| 仓位调整倍数 | 止损放宽后为维持 2%~3% 账户风险所需的仓位变化 |
| 成本后胜率 | 判断是否真的具备高胜率特征 |
| 成本后盈亏比 | 扣除手续费和滑点后的平均盈利 / 平均亏损 |
| 盈亏平衡胜率 | 根据成本后盈亏比计算最低要求 |
| 胜率安全边际 | 实际胜率 - 盈亏平衡胜率 |
| 胜率 / 盈亏比转化效率 | 判断牺牲多少盈亏比换来多少胜率 |
| 最大单笔亏损 / 平均账户盈利 | 判断一次亏损会回吐多少笔平均账户盈利 |
| 亏损簇 | 评估账户生存能力 |
| 成本 / 平均账户盈利 | 判断小盈利是否被成本吞噬 |
| MAE | 观察入场后最大不利波动，评估止损是否合理 |
| MFE | 观察入场后最大有利波动，评估止盈是否过早或过晚 |
| exit reason 分布 | 判断收益来自止盈、止损、时间退出还是异常退出 |

## 8. 模块角色定位

| 模块 | 后续定位 |
|------|----------|
| MA | 仅作为趋势背景、均值位置或市场行为诊断工具，不作为市场逻辑本身，不继续作为主线调参对象 |
| ATR | 保留为波动归一、严格失败距离、实际止损距离、异常波动过滤和账户风险预算换算模块 |
| MACD/KDJ | 仅作为动量变化和状态诊断工具，避免叠加成方向预测确认 |
| DMI/ADX | 可作为趋势/震荡状态过滤器，但必须证明能提高接受 / 拒绝质量、原始盈亏比或成本后长期期望 |
| 布林带 / Donchian | 可作为压缩、扩张、通道边界和假突破诊断工具；不是市场逻辑本身，不能单独作为主线入口理由 |
| Optuna/Bayesian search | 只在共识边界、严格失败边界、盈利上界、账户风险预算、胜率安全边际和尾部约束初步成立后启用 |
| Walk-Forward | 只在样本量、参数平台、胜率 / 盈亏比转化效率和账户风险预算初步成立后启用 |

## 9. 长期退出条件

策略主线满足以下任一情况，应暂停或降级为 baseline：

1. 找不到清晰的共识价格区间；
2. 严格失败边界无法定义，或定义后交易机会过少；
3. 严格失败边界无法映射到 2%~3% 单次账户风险；
4. 预期盈利上界无法定义，或价格原始盈亏比不足；
5. 账户原始盈亏比不足，或平均账户盈利过小；
6. 严格失败边界快速再触及率过高，说明边界太近或接受质量不足；
7. 价格接受 / 拒绝证据不足，只是机械触碰指标边界；
8. 合约乘数、最小手数、滑点或跳空导致账户风险预算不可执行；
9. 主动止盈、时间退出或止损放宽不能改善胜率 / 盈亏比转化效率；
10. 胜率和盈亏比不稳定，长期成本后期望无法转正；
11. 正期望安全边际不足，轻微成本或滑点上升即失效；
12. 尾部亏损或超预算亏损不可控；
13. 为了维持高胜率需要无限放宽止损、加仓摊低或长期扛亏；
14. 正收益依赖单合约、单时段或尖峰参数；
15. 参数附近没有稳定平台，轻微扰动即断崖式退化；
16. 无法解释收益来自共识价格区间、接受 / 拒绝质量、失败边界、盈利上界、账户风险预算、止损放宽、止盈、时间退出、成本还是市场状态变化。

## 10. 短期计划归档目录约定

`docs/roadmap/strategy-short-term-plan.md` 永远表示当前活跃短期研究计划。

当一个短期研究阶段结束时，不再把归档重点放在单个历史计划文件名上，而是把该阶段的短期计划、实验归档和相关复盘统一放入带日期和主题的目录：

```text
docs/archive/strategy-research/<YYYY-MM-DD>-<阶段主题>/
```

示例：

```text
docs/archive/strategy-research/2026/06/2026-06-27-low-validation-cost/
docs/archive/strategy-research/2026-06-28-structural-alpha/
docs/archive/strategy-research/2026-07-xx-auction-structure/
```

目录内文件按内容命名即可，不要求某个文件承担强索引职责：

```text
<阶段主题>-short-term-plan.md
<实验版本或结构主题>.md
<必要复盘或补充说明>.md
```

含义：

| 路径 | 含义 |
|------|------|
| `docs/roadmap/strategy-short-term-plan.md` | 当前活跃短期研究计划 |
| `docs/archive/strategy-research/<YYYY-MM-DD>-<阶段主题>/` | 某个历史研究阶段的归档集合 |

因此，后续归档短期计划时，应先根据该阶段的核心研究主题建立 `<YYYY-MM-DD>-<阶段主题>` 目录，再把该阶段相关计划、实验摘要和必要复盘放入该目录，并同步修正长期框架、实验归档和 issues 中的相对链接。

## 11. 研究顺序：广度优先，深度在后

对具体策略假设，采用广度优先验证而非深度优先调参。先用最简规则在尽可能多的品种上扫描，确认基本假设（方向 edge）是否普遍存在、策略的生效边界在哪里，只在有信号的范围 内继续深入。不拿在少量品种上深度调参后的完整策略去扩品种——那样的扩样几乎必然失败。

## 12. 总结

后续研究应坚持：

```text
不做原创 alpha discovery，承认无法稳定预测未来；
不依赖广泛分散和长期大数定律，先保证账户生存；
不把入场当预测点，而把入场当低成本验证方向选择的共识价格区间；
不把低验证成本等同于结构有效，必须同时检查接受 / 拒绝质量、盈利上界和账户风险预算；
不把布林带、Donchian、均线、MACD / KDJ 等指标当作市场逻辑本身，只把它们作为诊断工具；
不追求单纯高胜率，而是评估胜率 / 盈亏比转化效率、长期正期望和安全边际；
不靠无限扛亏提高胜率，止损放宽必须事先定义、同步降仓且账户风险不超过 2%~3%；
不只看胜率，必须同时看价格原始盈亏比、账户原始盈亏比、成本、安全边际和亏损簇；
不为凑样本放宽场景，优先寻找少而清晰、共识边界明确、失败边界和盈利上界可定义的交易机会；
先验证资金约束、接受质量、盈利上界、账户风险预算和收益分布，再做参数搜索和多窗口验证。
```

短期实验应服务于这套长期共识：先验证“共识价格区间 + 接受 / 拒绝质量 + 严格失败边界 + 盈利上界 + 2%~3% 账户风险预算 + 胜率 / 盈亏比转化效率”是否能形成成本后长期正期望和足够安全边际，再讨论具体入口信号和参数优化。

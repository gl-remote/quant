# 策略正期望研究知识沉淀

> 类型：Roadmap / 策略研究方法论沉淀  
> 状态：指导后续策略实验  
> 创建日期：2026-06-27  
> 来源文档：[MA 策略正期望优化复盘](./ma-positive-expectancy.md)、[ATR 策略正期望优化复盘](./strategy-atr-tuning.md)

## 1. 背景

MA 与 ATR 两条主线已经完成多轮短样本实验。两者虽然策略形态不同，但暴露出相同的核心问题：当前阻塞不是优化器、trial 数、仓位或单个止损止盈参数，而是 **alpha / 触发密度 / 持仓释放** 尚未进入可评价区间。

因此，后续策略研究需要从“调参数找收益”转向“先验证交易节奏和样本形态”。只有当交易频率、持仓周期、成本占比和跨品种一致性进入可观察区间后，参数搜索、Walk-Forward 和风险仓位才有意义。

## 2. 当前共识

| 共识 | 说明 |
|------|------|
| 参数搜索必须后置 | 交易数不足时，Optuna/Bayesian search 只会拟合噪声，不能创造交易机会 |
| 单笔或个位数交易不是正期望证据 | 短样本正收益如果来自 1~5 笔交易，不能证明策略有可迁移能力 |
| 交易节奏是前置硬约束 | 当前一个多月样本下，单品种每月约 10~20 次进场才具备初步观察价值 |
| 持仓释放和入场触发同等重要 | 信号出现不等于真实成交，长持仓、冷却和反向退出规则都会影响实际交易数 |
| 趋势背景是降噪器，不是频率修复器 | `15m` 背景能降低 `1m` 噪声，但无法单独创造交易机会 |
| 趋势过滤应后置 | 基础信号稀疏时，DMI/ADX、trend gap 等过滤会进一步压低样本 |
| ATR 更适合作为风控模块 | ATR 止损、止盈、移动止盈和波动归一有价值，但不应被误当作独立 alpha |
| MA 更适合作为 baseline 或背景状态 | 当前 MA 主线没有稳定参数平台，适合保留为对照基准 |

## 3. MA 与 ATR 的共同教训

### 3.1 不要把“没有足够交易”误判为“参数还没调好”

MA 与 ATR 的早期结果都出现过单品种只有 1 笔交易的情况。此时不应扩大搜索空间或 trial 数，因为优化器只能在已有交易机会中筛选参数，不能凭空生成有效样本。

正确处理方式：

```text
交易数不足 → 拆解信号与出场 → 检查候选信号和持仓阻塞 → 再决定是否调参
```

错误处理方式：

```text
交易数不足 → 扩大 trial → 选择单次收益最高参数 → 误判为正期望
```

### 3.2 入场放宽不等于成交密度提升

交易数由两部分共同决定：

```text
实际成交数 = 候选信号数 - 被持仓/冷却/规则阻塞的机会
```

如果单笔持仓过久，即使后续出现候选信号，也无法形成新交易。因此，每次讨论信号密度时必须同步检查：

- 平均持仓时间；
- 最长持仓时间；
- 1 日内退出比例；
- 超过 2 日退出比例；
- exit reason 分布；
- 被已有持仓阻塞的候选信号数；
- 平仓后是否允许同方向再入场。

### 3.3 反向退出能恢复样本，但容易制造噪声

MA 实验表明，同级别即时反向退出可以显著提升完整交易数，但也容易带来高频反复开平、手续费放大和品种分化。

更稳健的方向是：

- 更高周期反向退出；
- 连续 N 根反向确认；
- 最短持仓约束；
- 平仓后冷却；
- 时间止损释放持仓。

但这些机制只负责改善交易形态，不负责创造 alpha。若基础触发本身无效，出场机制无法把策略变成正期望。

### 3.4 慢周期背景降低噪声，也会压低样本

`15m` 背景 + `5m` 触发比 `1m`/`5m` 混合确认更干净，但它本质上是过滤器。它可以减少噪声交易和爆仓风险，也可能压低本就不足的交易数。

因此，慢周期背景只适合在基础触发足够多时使用。若基础触发稀疏，应先修复触发事件，而不是继续叠加过滤。

### 3.5 成本敏感性必须前置

当前目标是中低频 CTA，不是高频噪声交易。若策略通过加快指标参数获得交易数，但平均持仓只有数小时、单笔平均盈亏接近手续费和滑点，则不能作为可投入主线。

每个候选结构都应检查：

- 单笔平均盈亏；
- 平均盈利交易 / 平均亏损交易；
- 手续费和滑点占单笔盈亏比例；
- 毛收益与净收益差异；
- 交易数提升是否由噪声反复开平造成。

## 4. 推荐的研究判定顺序

后续策略实验统一按以下顺序判断：

```text
候选信号数是否足够
→ 实际成交数是否足够
→ 持仓周期是否达标
→ 成本后是否不过度恶化
→ 胜率、盈亏比和回撤是否可接受
→ 参数平台是否稳定
→ Walk-Forward / 多品种验证
```

在前三项没有达标前，不应把收益率、夏普、最大回撤或单次最优参数作为主要证据。

## 5. 目标样本形态

当前本地样本约一个多月，策略若要进入可评价区间，应尽量满足：

| 维度 | 目标 |
|------|------|
| 交易频率 | 单品种每月约 10~20 次进场 |
| 持仓周期 | 平均持仓 1~2 天 |
| 单笔质量 | 单笔平均盈亏显著大于手续费和滑点 |
| 品种一致性 | 不依赖单个合约或单笔尖峰收益 |
| 参数稳定性 | 附近参数不应断崖式退化 |
| 归因清晰 | 能说明改善来自入场、出场、风控、过滤还是成本 |

若某策略长期停留在单品种个位数交易，或只能通过极快参数制造大量数小时内交易，应暂停主线优化。

## 6. 后续策略角色定位

| 模块 | 后续定位 |
|------|----------|
| MA | 保留 `ma8` baseline，用于趋势类策略对照；可作为背景状态，不继续主线调参 |
| ATR | 保留为风控模块，用于止损、止盈、移动止盈和波动归一 |
| MACD/KDJ | 可作为辅助触发或状态描述，但不宜继续叠加为过严的多重确认 |
| DMI/ADX | 暂不前置；基础信号足够后再作为过滤器测试 |
| Optuna/Bayesian search | 只在交易节奏和持仓形态达标后启用 |
| Walk-Forward | 只在样本量和参数平台初步成立后启用 |

## 7. 下一阶段优先方向

### 7.1 ATR 最后一轮诊断

ATR 当前从 `1 / 1 / 1` 改善到 `3 / 1 / 5`，说明结构仍有一轮诊断价值，但不应进入 ATR 参数搜索。

优先补充：

- 平均持仓、最长持仓；
- 1 日内退出比例、超过 2 日退出比例；
- exit reason 分布；
- 候选信号数；
- 被已有持仓阻塞的信号数；
- 被冷却阻塞的信号数；
- 平仓后同方向再入场情况；
- MFE / MAE。

诊断目标是判断：当前结构是“没有候选信号”，还是“有信号但被出场/再入场机制卡住”。

若补完释放和再入场机制后仍无法接近单品种每月 `10~20` 次进场，ATR 触发主线应降级，仅保留 ATR 风控能力。

### 7.2 新触发结构

若 MA/ATR 继续无法达标，应优先尝试更明确的价格行为或波动事件触发：

| 方向 | 设计意图 |
|------|----------|
| Donchian / 通道突破 | 直接捕捉趋势启动，比 MA 交叉更接近价格行为 |
| 布林带突破 | 捕捉低波动向高波动切换，适合结合 ATR 风控 |
| 布林带收缩后突破 | 先定义压缩状态，再交易扩张事件 |
| 回踩均线后再启动 | 使用 MA 作为背景，不交易 MA 交叉，而交易趋势延续中的再启动 |
| N 周期高低点突破 + ATR trailing | 结构简单、归因清楚、与 ATR 风控天然兼容 |

这些结构的重点不是马上追求收益，而是先验证是否能形成目标交易节奏和更清晰的入场事件。

## 8. 实验记录规范

后续每轮实验至少记录：

| 字段 | 说明 |
|------|------|
| 策略版本 | 例如 `ma8`、`atr-r13`、`donchian-r1` |
| 样本范围 | 合约、时间窗口、bar 周期 |
| 核心假设 | 本轮想验证什么，不要只写参数 |
| 参数设置 | 入场、出场、风控、冷却、持仓约束 |
| 候选信号数 | 未考虑持仓和冷却前的信号数量 |
| 实际成交数 | 真实开仓次数和完整交易数 |
| 阻塞原因 | 持仓阻塞、冷却阻塞、风控阻塞等 |
| 持仓周期 | 平均、最长、分布比例 |
| 成本影响 | 毛收益、净收益、手续费/滑点占比 |
| 收益质量 | 胜率、盈亏比、最大回撤、单品种分化 |
| 设计结论 | 本轮证明了什么，下一轮是否继续 |

失败实验也必须记录设计含义。若某轮实验只证明“这个方向不值得继续”，也是有效结论。

## 9. 退出条件

策略主线满足以下任一情况，应暂停或降级为 baseline：

1. 多轮调整后，单品种每月仍只有个位数交易；
2. 交易数只能通过极快参数或即时反向退出获得，且成本吞噬明显；
3. 频率、持仓周期和品种一致性无法同时满足；
4. 正收益依赖单合约、单笔交易或尖峰参数；
5. 参数附近没有稳定平台，轻微扰动即断崖式退化；
6. 无法解释改善来自入场、出场、过滤、风控还是成本变化。

退出不是失败，而是避免把研究资源投入到没有样本形态支撑的结构上。

## 10. 总结

MA 与 ATR 的阶段性复盘共同说明：当前最重要的不是继续寻找最优参数，而是建立一套更严格的策略候选筛选流程。

后续研究应坚持：

```text
先样本形态，后收益指标；
先交易节奏，后参数搜索；
先结构归因，后 Walk-Forward；
先证明有可交易事件，再讨论风控优化。
```

在这个框架下，MA 保留为 baseline，ATR 保留为风控模块；下一阶段的主要增量应来自更明确的价格行为触发，例如 Donchian、通道突破、布林带突破或回踩均线后再启动。

## 1. 目标形态

ATR 不是独立 alpha，它主要用于风险控制、止损止盈、移动止盈和波动归一。当前要验证的是：以 ATR 风控承载的“趋势背景 + 触发入场”结构，是否能形成可观察、可迁移、扣除成本后仍有潜力的中低频 CTA 策略。

当前本地样本约一个多月，目标策略形态是：

| 维度 | 目标 |
|------|------|
| 交易频率 | 单品种每周约 3~5 次进场，一个月约 12~20 次进场 |
| 持仓周期 | 平均持仓 1~2 天 |
| 成本敏感性 | 单笔平均盈亏显著大于手续费和滑点 |
| 鲁棒性 | 不依赖单品种、单窗口、单笔持仓或尖峰参数 |
| 可解释性 | 能说明改善来自入场、出场、ATR 风控、持仓释放还是成本控制 |

当前判断顺序：

```text
交易节奏达标 → 持仓周期达标 → 成本后仍不过度恶化 → ATR 参数搜索 → Walk-Forward
```

因此，在交易频率和持仓周期未进入可评价区间前，不把收益率或单次 Optuna 最优结果作为正期望证据。

## 2. 当前结论

当前 ATR 在本地豆粕短样本上**还没有进入可做参数搜索的候选区域**，但已经证明主要问题不是单纯自然时间窗口不足，而是信号密度和持仓释放仍未达到目标节奏。

核心判断：

1. 原始 5-AND 结构每个合约只有 1 笔交易，不能作为正期望证据。
2. P1 的 `15m` 背景 + `5m` MACD/KDJ 结构降低了噪声，但仍只有 `1 / 1 / 1` 笔交易。
3. 并行 Bayesian search 工具链已跑通，但交易数不足时搜索结果只验证流程，不验证策略能力。
4. P1.5 放宽到 `15m` 背景 + `5m` KDJ 中线触发，并加入 `time_stop_bars=48` 后，交易数提升到 `3 / 1 / 5`。
5. `time_stop_bars` 能释放持仓，但当前成交密度仍不足，且收益质量分化明显。
6. 当前阻塞是 **alpha / 触发密度 / 持仓释放不足**，不是 ATR 参数、trial 数或优化器能力不足。

因此：**不建议现在继续扩大 ATR 参数搜索。** 下一轮应先补持仓周期统计，检查出场释放和再入场机制；如果再做 1~2 轮仍无法接近每月 `12~20` 笔，应考虑把 ATR 保留为风控模块，转向通道突破、Donchian、布林带突破、回踩均线后再启动等新触发结构。

## 3. 关键实验摘要

| 阶段 | 设置 | 交易数 | 结果摘要 | 设计含义 |
|------|------|--------|----------|----------|
| Baseline | 原始 5-AND：5m SMA + 1m/5m MACD + 1m/5m KDJ | `1 / 1 / 1` | 三个合约收益看似为正 | 单笔样本不能作为正期望证据；5-AND 入场过严 |
| P1 | `15m` 趋势背景 + `5m` MACD/KDJ 回调触发 | `1 / 1 / 1` | 降噪但不增频 | 慢趋势背景不是频率修复器；MACD 与 KDJ 同时满足仍过严 |
| 并行搜索验证 | Bayesian search，`3 trials × 3 合约` | `1 / 1 / 1` | 工具链跑通，最佳参数不可用 | 优化器只能避免退化解，不能创造有效交易机会 |
| P1.5 / `r7` | 加入 `time_stop_bars=48` | `3 / 3 / 1` | 持仓释放开始生效 | 时间止损能释放仓位，但不能创造稳定 alpha |
| P1.5 / `r13` | 修正 KDJ lookback，使用 `15m` 背景 + `5m` KDJ 中线触发 | `3 / 1 / 5` | 较 `1 / 1 / 1` 改善，但未达标 | 数据自然时间不是唯一问题；信号密度仍不足 |

`r13` 明细：

| 合约 | 交易数 | 总收益 | 最大回撤 | 判断 |
|------|--------|--------|----------|------|
| `DCE.m2601` | 3 | -4.30% | -5.70% | 密度改善但收益为负 |
| `DCE.m2603` | 1 | -8.82% | -8.82% | 仍未形成交易密度 |
| `DCE.m2605` | 5 | 9.26% | -7.80% | 接近目标低端但回撤偏大 |

## 4. 设计沉淀

1. **数据不够要拆成两个问题**  
   自然时间长度和目标交易节奏要分开判断。若策略目标是一周 `3~5` 次机会、平均持仓 `1~2` 天，一个月样本理论上应有 `12~20` 笔；当前个位数交易首先说明信号密度或持仓释放不匹配目标周期。

2. **入场放宽不等于真实成交变多**  
   交易频率由入场触发和出场释放共同决定。若单笔持仓占用过久，后续候选信号即使出现也无法成交。

3. **ATR 参数搜索必须后置**  
   交易数不足时，Optuna/Bayesian search 只会拟合噪声。应先让交易节奏和持仓周期进入可评价区间，再搜索 ATR 止损、止盈和移动止盈参数。

4. **指标 window 和 warmup 不是一回事**  
   KDJ/STOCH 需要额外预热，不能简单用业务窗口推导运行时 lookback。该框架问题已单独记录在 [framework-indicator-history-issues.md](./framework-indicator-history-issues.md)。

5. **策略不应长期维护指标历史缓存**  
   ATR 当前用 `state.extra` 缓存 KDJ 是临时绕法。长期应由框架提供可靠的 `indicator_history()` 或 `indicator_series()`。

6. **失败 run 也要记录设计含义**  
   `r4` 暴露指标读取/预热问题，`r7` 证明时间止损能释放持仓，`r13` 证明密度改善但未达标。这些失败比单次收益更能决定下一轮方向。

## 5. 当前决策

| 决策 | 说明 |
|------|------|
| 继续 ATR 信号密度 loop | 当前从 `1 / 1 / 1` 改善到 `3 / 1 / 5`，说明结构仍可继续验证 |
| 暂停 ATR 参数搜索 | 交易节奏未达标，搜索会拟合噪声 |
| 保留 ATR 风控能力 | ATR 止损、止盈、移动止盈仍适合作为策略风控模块 |
| 暂不引入 DMI/ADX | 基础信号仍稀疏，趋势过滤应后置 |
| 不追当前正收益样本 | `DCE.m2605` 的 `5` 笔正收益不足以支持调参追结果 |
| 下一轮退出条件 | 若补出场释放/再入场后仍无法接近月度 `12~20` 笔，ATR 主线降级为 baseline/风控模块 |

## 6. 下一轮任务

优先做能判断“交易节奏是否可修复”的工作：

1. 增加持仓周期统计：平均持仓、最长持仓、1 日内退出比例、超过 2 日退出比例。
2. 检查出场释放：时间止损、反向退出、平仓后冷却、是否允许同方向再入场。
3. 只在真实成交密度仍不足时继续调整触发结构，不先优化 ATR 参数。
4. 若交易数接近每月 `12~20` 笔，再评估胜率、盈亏比、成本敏感性和回撤形态。
5. 若仍无法达标，转向通道突破、Donchian、布林带突破或回踩均线后再启动等新触发结构。

## 7. 后置事项

这些事项暂不进入下一轮主线：

| 事项 | 后置原因 |
|------|----------|
| ATR 参数搜索 | 交易节奏未达标，搜索会过拟合 |
| ATR 风险仓位 | 需要先确认策略有足够样本和基本收益质量 |
| DMI/ADX | 只适合做过滤，基础信号稀疏时会继续压低样本 |
| Walk-Forward | 当前交易数不足，不具备样本条件 |
| 多品种组合验证 | 需要先在当前样本内形成可评价的单品种交易节奏 |
| 框架指标历史修复 | 已单独记录，长期应修，但 ATR 实验短期可用策略层缓存继续推进 |

# 算法设计备选方案

> 当前实验计划以**多分辨率共识 CUSUM + 状态机 RLL** 为主要路径。但这不是唯一方案。本文件整理所有备选路线的原理、优缺点和适用条件。
>
> **如何在 P1 结束后选择主路径**：参见 §[P1 分叉决策树](#p1-分叉决策树)。

---

## 路线 A：多分辨率共识 CUSUM + 状态机 RLL（当前主方案）

**核心思想**：对 W ∈ {20, 40, 80, 160} 四个窗口独立运行 CUSUM 断点检测，≥ 3 个窗口在 ±20h 内同时检出断点才确认一次 regime 切换。确认后在 LOW/MID/HIGH 三态状态机中转移，受最小停留 80h + 滞后确认 40h 约束。

| 维度 | 评价 |
|---|---|
| 理论基础 | Frequentist 变点检测 |
| 复杂度 | 中（4 窗口 CUSUM + 共识逻辑 + 状态机） |
| 误检控制 | 好（P(false)³ << P(false)） |
| 响应速度 | 慢（等 4 窗口都确认至少需要 20h+） |
| 调参量 | 多（每窗口 CUSUM 阈值 h、共识窗口 Δt、状态机 RLL 参数） |
| 可解释性 | 极高（每步都可以 debugs） |
| 不确定性输出 | 无（硬 0/1 判决） |
| 多品种联合 | 不支持独立模型，品种间无信息共享 |

**适用条件**：
- ACF 显示 regime 切换是**断点分明**的（sharp break，而非缓慢漂移）
- 需要完全的审计可追溯（每笔仓位调整都能说清是哪个分辨率触发的）
- 作为 baseline 和对照组，其他方案必须跑赢它才有切换价值

**工程成本**：约 200-300 行 Python，CUSUM 有成熟实现（`ruptures` 库），共识逻辑需要自己写。

---

## 路线 B：贝叶斯在线变点检测 (BOCPD)

**核心思想**：不硬断断点，而是对输入序列 $y_t = \hat{x}(t)$ 维护一个"当前距上次变点已过多少步"的 run-length $r_t$ 的后验分布。每来一个新观测，递归更新：

$$P(r_t \mid y_{1:t}) \propto \sum_{r_{t-1}} P(r_t \mid r_{t-1}) \cdot P(y_t \mid r_t, y_{(t-r_t):(t-1)}) \cdot P(r_{t-1} \mid y_{1:t-1})$$

其中 $P(r_t \mid r_{t-1})$ 是变点 hazard（先验风险率），$P(y_t \mid \dots)$ 是分段预测似然。

| 维度 | 评价 |
|---|---|
| 理论基础 | Bayesian 序贯推断 |
| 复杂度 | 高（需要 Sequential Monte Carlo / Particle Filter） |
| 误检控制 | 好（概率阈值控制） |
| 响应速度 | 中（在线序贯更新，响应延迟可调） |
| 调参量 | 中（hazard rate 先验、预测分布参数） |
| 可解释性 | 低（后验在粒子集中演化，难以逐笔追溯） |
| 不确定性输出 | **原生支持**（$P(\text{变点} \mid \text{数据})$ 是直接输出） |
| 多品种联合 | 难扩展 |

**适用条件**：
- 切换是**渐进式**而非断点式（均值在一段区间内连续漂移）
- 实盘需要"概率化仓位"而非硬切换（如：HIGH regime 概率 > 0.6 → 逐步加仓，而非瞬间加满）
- 愿意接受黑盒调式成本

**核心痛点**：BOCPD 通常用于一维序列的均值变点检测。输入 $\hat{x}_{W=80}(t)$ 单序列即可，不需要 4 窗口共识——因为 BOCPD 天然通过 run-length 后验区分局部波动和持久切换。但这意味着它**放弃了一部分多分辨率的结构化信息**。

**工程成本**：Python 无成熟 BOCPD 生产库（`bayesian_changepoint` 是学术 toy），需要自己实现 Particle Filter，约 400-600 行。

## 路线 C：隐马尔可夫模型 (HMM)

**核心思想**：假设观测 $\hat{x}(t) \mid S(t) \sim \mathcal{N}(\mu_k, \sigma_k)$，背后隐藏的 regime 状态 $S(t) \in \{1,2,3\}$ 服从一阶 Markov 链。通过 Baum-Welch (EM) 从全历史数据中学习转移矩阵 $A_{ij}$ 和发射分布参数 $(\mu_k, \sigma_k)$，在线运行时用 Viterbi 或 forward-filtering 解码最可能的当前状态。

**三品种联合 HMM 扩展**（路线 C 的独特优势）：

```
           板块隐藏状态 S_sector(t) ∈ {LOW, MID, HIGH}
                  /         |         \
          玉米 x̂(t)    淀粉 x̂(t)    豆粕 x̂(t)
```

玉米和淀粉共享同一个 $S_{sector}(t)$，但各自的观测噪声不同。豆粕由于驱动不同，可以完全不共享或部分共享状态。这种结构化信息是目前 CUSUM 方案完全无法表达的。

| 维度 | 评价 |
|---|---|
| 理论基础 | 状态空间模型 + EM 最大似然 |
| 复杂度 | 中高（Baum-Welch 有成熟实现，但诊断收敛困难） |
| 误检控制 | 中（依赖转移矩阵 $A$，后验概率可做二次过滤） |
| 响应速度 | 中（forward-filter 在线更新） |
| 调参量 | **少**（状态数 K 需先验，其余参数自动学习） |
| 可解释性 | 中（转移矩阵 $A$ 直观可读："LOW→MID 的月概率是 8%"） |
| 不确定性输出 | **原生支持**（$P(S_t=k \mid \text{data})$ 是直接输出） |
| 多品种联合 | **原生支持**（联合 HMM 是自然扩展） |

**适用条件**：
- P1 ACF 显示 regime 有**长记忆**（AR(1) 系数 > 0.5）
- 需要**多品种联合建模**（特别是玉米+淀粉共享板块状态）
- 不介意 EM 可能收敛到局部最优
- 有足够的历史数据（每个品种至少 2000 bar，目前都有）

**工程成本**：使用 `hmmlearn` 库，约 150-250 行 Python。联合 HMM 需要自定义，约 300-400 行。

## 路线 D：直接波动率 Regime（80/20 极简方案）

**核心思想**：$|\nu|/\sigma$ 的 regime 本质上由波动率 $\sigma$ 主导（分子有方向但绝对值往往与波动率同方向变动）。直接用滚动 $\sigma$（波动率）替代 $|\nu|/\sigma$ 做 regime 分类：

| Regime | 条件 | 行为 |
|---|---|---|
| LOW | $\sigma < \sigma_{P30}$ | 不开仓 |
| MID | $\sigma_{P30} \leq \sigma \leq \sigma_{P70}$ | 使用 KF-27 全时段参数 |
| HIGH | $\sigma > \sigma_{P70}$ | 使用 KF-27 high-RR 参数 |

$\sigma$ 估计使用 W=80h 滚动标准差：$\hat{\sigma}_t = \sqrt{\frac{1}{W-1}\sum_{i=t-W+1}^t (r_i - \bar{r})^2}$

| 维度 | 评价 |
|---|---|
| 理论基础 | 经验主义（无严格概率框架） |
| 复杂度 | **极低**（一行 pandas：`df['close'].pct_change().rolling(80).std()`） |
| 误检控制 | **差**（高波震荡市会大量误报 HIGH regime） |
| 响应速度 | **快**（$\sigma$ 实时更新，无确认延迟） |
| 调参量 | **最少**（1 个窗口长度 + 2 个分位阈值） |
| 可解释性 | **极高** |
| 不确定性输出 | 无 |
| 多品种联合 | 简单（品种间做相关性分析） |

**适用条件**：
- 仅作为 **baseline**——其他方案必须跑赢波动率方案才值得采用
- 品种的 $\sigma$ 和 $|\nu|/\sigma$ 的 Spearman 秩相关系数 r > 0.7（P1 给出实证）
- 预算有限，只想要一个"够用"的解

**风险**：实盘中 $\sigma$ 和 $|\nu|$ 可能完全脱钩——高波无方向的市场（如事件驱动的双向大幅震荡）会被误判为 HIGH regime，导致在该开仓的时候大量亏损。需要 $\sigma$ 和 $|\nu|/\sigma$ 的变异度一致性验证（P1）。

**工程成本**：约 50 行 Python，两小时写完。

## 路线 E：双层 CUSUM（当前方案的直接改进）

**核心思想**：当前方案需要等 4 窗口共识，响应慢。替代方案是将 CUSUM 分层：

```
Layer 1: CUSUM on x̂_{W=20}(t)
         → 快速检测候选变点（高灵敏度，高误报）
Layer 2: CUSUM on Layer 1 的报警累积量
         → 只有当报警持续累积才确认
```

2 层确认等效于 4 窗口共识但没有"等最慢窗口"的问题。Layer 2 的 CUSUM 输入是 Layer 1 的报警强度（不一定要二进制 0/1，可以是偏离度）。

| 维度 | 评价 |
|---|---|
| 理论基础 | Frequentist（与路线 A 同一家族） |
| 复杂度 | 低（2 个 CUSUM 串联） |
| 误检控制 | 中（双层过滤，弱于 4 窗口共识但强于单窗口） |
| 响应速度 | **中快**（比 4 窗口共识快，比单窗口慢） |
| 调参量 | 少（每层 CUSUM 阈值 h1, h2，共 2 个参数） |
| 可解释性 | 高（两个 CUSUM 过程可分别追踪） |

**适用条件**：
- P1 显示切换是断点式的（sharp break）
- 路线 A 的响应速度不满足要求
- 想保留 Frequentist 框架但简化工程

**工程成本**：约 100-150 行 Python。

## 路线 F：神经网络隐状态编码（高风险高回报）

**核心思想**：直接绕开 "$\hat{x}(t)$ → regime 标签 → 参数映射"的手工链路，用 LSTM/Transformer 将 $\hat{x}(t)$ 的历史序列编码为一个隐状态向量 $h_t$，直接预测下一步的最优塑形容器参数 $(K_S, K_T, \tau)_t$。

```
输入: [x̂(t-L+1), ..., x̂(t)]  →  LSTM  →  h_t  →  [K_S, K_T, τ]_t
                                                    ↓
                                         代入 KF-27 E_gross 公式
                                                    ↓
                                         损失: -E_gross[h_t 对应的参数]
```

| 维度 | 评价 |
|---|---|
| 理论基础 | 无（纯数据驱动） |
| 复杂度 | **极高**（训练管线 + 超参搜索 + 过拟合防控） |
| 可解释性 | **极低**（黑盒预测参数，无法审计为什么是这个 K_S） |
| 调参量 | 极多（LSTM 层数、隐状态维度、学习率、正则化…） |
| 数据需求 | 极大（需要 10^5+ 样本级的 $\hat{x}$ 序列做训练） |

**不推荐**：数据量不足（玉米全历史约 6000 bar → ~3000 个滑动窗口样本，对于神经网络太少），不适合小样本研究项目。

**唯一价值**：如果 P1 发现 $\hat{x}(t)$ 的时序结构极其复杂（高阶非线性依赖），传统方法搞不定，可以考虑。但对商品期货 1h 数据来说不太可能。

## P1 分叉决策树

P1（$|\nu|/\sigma$ 时间序列分析）的结果直接决定应该走哪条路线。设计如下决策树：

```
P1 产出:
  (a) x̂_{W=80}(t) 序列的 AR(1) 系数
  (b) ACF 半衰期（自相关衰减到 0.5 所需的滞后数）
  (c) 断点尖锐度（CUSUM 统计量在断点处的跳跃高度 vs 基线波动的比值）
  (d) σ 与 |ν|/σ 的 Spearman 秩相关系数

决策:

1. 如果 (d) > 0.7
   → 路线 D（波动率 regime）作为必跑 baseline
   → 其他路线必须跑赢路线 D 才有切换价值

2. 如果 (a) > 0.5 且 (b) > 80h（长记忆 regime）
   → 优先选路线 C（HMM），状态空间天然适配长记忆过程

3. 如果 (c) > 3.0（sharp break，断点分明）
   → 优先选路线 A（共识 CUSUM）或路线 E（双层 CUSUM）
   → 路线 A 更稳健（4 窗口共识），路线 E 更快（双层串联）

4. 如果既不是长记忆、也不是 sharp break（渐进漂移 + 无强记忆）
   → 优先选路线 B（BOCPD），Bayesian 框架最适配渐变过程

5. 任一情况，路线 A（共识 CUSUM）保持不变作为对照 baseline
   → 所有路线的产出格式统一（regime 标签 + 切换时间戳 + 参数表）
   → P4 回测阶段可直接插入不同路线结果做横向对比
```

**为什么保留路线 A 作为全场景 baseline**：虽然它不是最优的，但它有一个其他方案都没有的属性——**完全透明，每步可审计**。当实盘出问题时（必然出事），路线 A 的方案可以逐笔追溯"这笔仓位切换是 W=20 触发的还是 W=80 触发的"，而 HMM 只会说"后验概率 0.73，我也不知道为什么"。

## 关于"最优方案"的说明

本文列出的 6 条路线没有一条是"最优"的——因为**在不知道 $|\nu|/\sigma$ 的实际时序结构之前，不存在理论上的最优先验选择**。P1 的实证数据决定了哪条路线对玉米/淀粉/豆粕拟合得最好。

目标不是找到"最优算法"，而是**在所有方案中挑选出对实盘最稳健、最少过拟合、最可调试的一个**。这个优先级排序意味着：

- 可解释性 > 复杂度（优先选路由 A / D / E）
- 不确定性输出 > 点估计（验证 B / C 是否能带来平滑过渡的收益）
- 多品种泛化 > 单品种极致（验证 C 的联合 HMM 是否能捕获板块共享 regime）

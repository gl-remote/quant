# va-asymmetry-composite · 策略定义（通俗版 · 归档 v1.0）

> 本文是归档版 v1.0 交易策略的**通俗语言定义**，章节结构与严格数学规格
> （`../../archive/strategy-research/2026-07-10-va-asymmetry-composite/strategy-math-spec.md`）
> 一一对应，供后续逐节翻译为数学语言。**当前状态：通俗版审校中，待翻译成严格数学版。**

策略在物理上分为两层，后者依赖前者：

- **A 层 · 盘前分类器**：每个交易日开盘前，给每个合约算出"今天做多候选 / 做空候选 / 不参与"，结果当天固定。
- **B 层 · 日内交易执行**：交易日当天，按 A 层给出的清单执行开仓、止损、平仓与仓位管理。

## §1 分类器：输入、输出与归类（A 层 · 盘前 · 日频）

### §1.1 原始输入（3 个原始量 · 定义方式）

对每个合约，基于近 **10 日**行情（偏度取前一日 5m K 线）算出 3 个**原始量**。注意：这些原始量**本身不是排名**——排名只是 §1.3 里多种归一化手段之一，可替换；原始量未经归一化、量纲/分布随品种与时段而异，不能直接用于阵营判定。

三个原始量及其**定义方式**（计算口径继承 poc 分类器，见 `classifier-math-spec`）：

1. **量加权价格偏度 A3_skew（原始）**：取前一交易日 `skew_win` 指定粒度的 K 线（见 §0.1），按成交量加权算价格分布的**三阶中心矩标准化偏度**（poc §4）。这是核心信号——"昨天成交量堆在价格高位还是低位"，原始取值为偏度本身（可正可负，典型 |A3_skew| < 3）。
2. **真实波幅 ATR（原始）**：该合约前 `L`（=`atr_entry_win`，见 §0.1）日平均真实波幅，原始单位为价格或 bps（poc §5.2，`daily_atr_L_bps`）。
3. **收益率方向 / 趋势（原始）**：该合约前 `M`（=`trend_entry_win`，见 §0.1）日累计对数收益 `trend_ret_M = log(C_d / C_{d−M+1})`（poc §5.3），原始取值为收益本身（可正可负）。

> 注：策略名含 "value-area"，但该成分不参与实际计算；真正使用的是上述量加权偏度。三者均为原始量，是否及如何归一化、映射到可比尺度由 §1.3 决定。定义方式中的窗口——原始量窗口 `atr_entry_win`/`trend_entry_win`（即 `L`/`M`）、`skew_win`（偏度 K 线粒度），以及三者各自的 **rank 窗口 `skew_rank_win` / `atr_rank_win` / `trend_win`（天数可独立设定）**——均为可调超参，具体值与候选集见 §0.1。

### §1.2 主攻阵营（6 类 · 标签 + 逻辑，不含参数选择）

基于上述三维特征，本策略主攻 **6 个互斥阵营（tier）**，由 (偏度域, 波动率制度, 趋势制度) 三维组合唯一确定，再加 transition 轴决定"稳定/转换期"适用。6 类 = 多头 3 + 空头 3。下面**只讲每类的逻辑（为什么这个组合是可交易环境），不写具体参数区间**——参数区间在 §1.3 按归一化方式给出。

**元逻辑（来自 poc 阶段研究，非本 spec 杜撰）**：

- **偏度是"顺势放大器"而非独立均值回归**（KF-05）：偏度本身不单独产生 alpha，必须叠加趋势 + 波动过滤；无 trend filter 会在跌段亏钱。
- **ATR 是过滤器而非放大器**，与偏度近乎独立（KF-06，Spearman |ρ| < 0.05）；低 ATR 空头甚至会反转。
- **ATR × Trend 正交**（KF-19 洞察 S）：ATR = 强度，Trend = 方向，联合筛选是核心 filter 架构。
- **多头 3 种反弹机制**（KF-16 洞察 P）：低 ATR = 日常均值回归、中 ATR = 震荡后秩序恢复、高 ATR = 恐慌 V 反弹。
- **空头单机制**（KF-17 洞察 Q）：只有"崩盘前奏"一种，必须高 ATR；低 ATR 空头反转。

**各阵营逻辑（标签 + 为什么可交易）**：

| 阵营 | 方向 | 逻辑 | transition 适用 |
|:---|:---:|:---|:---|
| **L_seg3_lowmid_up** | 多 | 温和右偏（量堆在价格中低位、非极端底厚/顶厚）+ 低中波动 + 强涨势。偏度作顺势放大器确认涨势可持续性；含 KF-23 段3·低ATR 甜蜜点（平静市 + 底厚 → 慢反弹）。 | 全期（稳定/转换均可） |
| **L_seg12_high_up** | 多 | 近中性/极端底厚偏度 + **高波动** + 强涨势。对应洞察 P 的高 ATR"恐慌 V 反弹"机制：高波动中极端底厚 + 强涨的急反弹。 | 仅转换期（稳定期失效） |
| **L_seg2_low_flat（已淘汰，见 §11）** | 多 | 低偏度 + **低波动** + 平稳趋势。低波动平稳市的温和多头；KF-26 表明平稳期 alpha 仅存在于转换期。（注：该 tier 塑形后信息比率为负，已全品种淘汰） | 仅转换期 |
| **S_seg12_high_dn** | 空（核心） | 极端右偏（**顶厚**=量堆在价格高位=上方供给堆积）+ **高波动** + 走弱/跌。空头唯一机制"崩盘前奏"（KF-17），高波动中趋势转弱时最强反转做空信号；三 period 全 A，最铁复合信号（KF-29）。 | 全期，偏好 |τ| 大（转换期更强，KF-28） |
| **S_seg34_high_dn** | 空 | 次极端右偏（顶厚但略弱）+ 高波动 + 跌。同"崩盘前奏"机制、顶厚程度稍低的主力空头。 | 全期，偏好 |τ| 大 |
| **S_seg2_mid_dn** | 空 | 中高右偏（顶厚，程度弱于 S_seg12）+ **中波动** + 跌。高波动以外的顶厚做空（KF-23 段3·ATR中 新甜蜜点）；仅转换期有效。注：其 skew 区间 `(0.81,0.91]`（§1.3.1）为 S_seg12 `[0.81,1]` 的偏弱子集，故偏度程度低于 S_seg12，称"中高"而非"极端"。 | 仅转换期 |

> 注：阵营逻辑依据见 poc `research-status.md`（KF-05/06/16/17/19/26/28/29）与 `parameter-selection-spec §8 下游指南`。**互斥性**：6 类在 (skew × ATR × trend) 三维空间内完全互斥，同一事件最多命中 1 类（边界刻意不重叠，区间见 §1.3）。多头与空头严格分域：偏度 ≤ 0.30 多域、≥ 0.60 空域、中间 [0.30, 0.60] 不参与（具体边界见 §1.3.1）。

### §1.3 参数归一化与阵营参数范围

阵营判定在**归一化后的特征**上做。归一化手段是可替换超参；不同手段下，同一阵营对应的参数范围不同。下面先给每个参数的候选归一化方式，再在每种方式下给出各阵营的参数范围。

#### §1.3.0 归一化候选总览（主特征）

| 候选 | 名称 | 做法 | 输出尺度 | 适用原始量 |
|:---|:---|:---|:---|:---|
| A（默认 B0） | 滚动排名 rank | 在各指标 `*_rank_win` 窗口内（见 §0.1，skew/atr/trend 天数可独立设定）对原始量滚动排名 | **0~1（均匀）** | 偏度 / ATR / 趋势 均可（分布稳健） |
| B | z-score 标准化 | `(x − μ)/σ`，μ、σ 取窗口内 | **无界 ℝ（中心 0）** | 偏度 / ATR / 趋势 均可；但 D2 已实证**跨 0、窗内混合符号、跨合约尺度差 2×**，固定 z 阈值不可移植，仅作下游 ℝ 尺度变体（详见 §1.3.0 注） |
| C | 分位数 / 百分位 | `percentile(x)` 在窗口经验分布中的位置 | **0~1（均匀）** | 偏度 / ATR / 趋势 均可（≈A） |
| D | 绝对阈值分档 | 直接对原始量设硬阈值（如 ATR 以 bps 绝对档） | **离散档** | ATR 最自然（有绝对量纲）；偏度/趋势需先定尺度 |

> **尺度一致性**：§1.3.1~1.3.3 的阵营范围写在 **0~1 尺度**上，故仅 **A / C 可直接套用**。B（ℝ）、D（离散）须先对齐尺度（见各小节）。
> **分布假设（已实证，基于 `dataset_full.parquet` 的 36,625 个 A3_skew 事件，即 D2）**：区分两层分布——(D1) 偏度所概括的**日内量加权价格分布**（偏度是刻画它的统计量，天然可非对称，这正是 skew≠0 的意义）；(D2) **偏度统计量自身的时间序列分布**（逐事件每个合约得一个 skew 值，跨事件/跨合约构成，即我们真正归一化的对象）。归一化作用在 D2 上，与 D1 的形状**无关**。"偏度度量非对称性、用对称假设归一化它自相矛盾"是**范畴错误**——两者不是同一分布。**实证结果**：(i) **D2 跨 0**（P(skew>0)=49.3%、均值≈0、中位≈0、范围[−3.1,+2.2]）→ "值>0/右偏"假设不成立，log 等正支撑变换失效；(ii) **D2 近似对称**（自身偏度≈+0.02）、**中度重尾**（超额峰度+0.77、拟合 Student-t df≈12，非极端重尾，且 t 与正态在中间分位几乎重合、差异仅在极尾）→ z-score 的对称前提**满足**，但重尾使 σ 不稳定；(iii) **逐合约尺度差异大**（std 范围[0.32,0.66]、JB 普遍显著非正态）→ 绝对阈值(D)不具跨品种可移植；(iv) **各 N 日窗内 D2 普遍同现正负（D2 中心≈0）** → 固定 z/绝对阈值在窗内语义混乱。→ **结论：A/C（排名/百分位）是 D2 上唯一经得起实证检验的归一化**（非参数、零分布假设、跨窗口/跨合约稳健）；B 仅在下游需要无界 ℝ 尺度时作待校准变体（对称前提已满足，但各窗内跨 0 + 跨合约尺度差使其**不能用于 tier 判定**）；D 仅单品种内可作绝对档。10 日窗口（仅约 10 个日频观测）无法可靠拟合 t 参数（scipy.t.fit 在多个合约退化至天文数字），故参数化 PIT 不纳入候选。
> **同合约窗内归一化（A 层盘前 · 日频 · N 天参数化）**：本策略归一化即**逐合约（within-contract）滚动**，且按 spec 设计在 **A 层盘前、日频**完成——§1.4 明确 A 层"开盘前基于前 N 日行情算好、当日清单当天不再变化"，B 层（日内执行）全天复用、**不重新计算 skew**。故归一化作用在**每日 1 个 skew 观测**构成的序列上（每合约每天 1 值、日内恒定），§1.3.0 的跨合约尺度差（std∈[0.32,0.66]）不影响判定。这恰与被否决的**旧实现**相反：旧管线（stage4/stage2）对 `A3_skew` 按**逐事件行**滚 `ROLLING_EVENTS=100`，因每日约 6 行且日内 skew 恒定，等效仅 ~17 交易日、且同日重复行引入 ≤0.05 的秩 jitter——混淆了"日内事件"与"日频信号"，是已知缺陷；正确实现须先 `drop_duplicates(event_date)` 再去按 N 日滚（与 atr/trend 同口径；该正确口径将在后续独立脚本实现，不沿用被否决的旧 stage4 实现）。**窗口长度 N 为已参数化超参（`skew_rank_win` / `atr_rank_win` / `trend_win` 三者天数独立可调，见 §0.1；§1.4 以"前 N 日"为示例（N 见 §0.1 各 `*_rank_win`）），后续微调将遍历各天数**。结合 D2 已实证为对称、中度重尾 t(ν≈12)、中心≈0，且归一化落在每日序列上，选型结论为：(i) **首选 = 逐合约滚动百分位/排名（A/C）**：与 spec 以百分位定义的阵营边界（0.09..0.91）精确对应、模型零风险；(ii) **N 天窗口的尾部分位分辨率指引（供天数微调参考）**：0.09/0.91 每侧仅约 `0.09×N` 个观测——N=20（spec 示例）≈1.8 个、偏短；N=60≈5.4、N=100≈9 更稳。故 **skew** 的 `skew_rank_win` 微调遍历时应将 ≥60 天纳入候选、勿止步于 20 天（atr/trend 边界居中、20 天默认已够，不在此限）；(iii) **若下游需无界 ℝ 尺度，用稳健 loc/scale = 中位数 + 1.4826·MAD（非 σ，重尾会撑大 σ）**，并可将 `(x−med)/(1.4826·MAD)` 过已知 t(ν=12) 的 CDF 得形状校正后的 0~1（参数化 PIT）；(iv) **参数化 t-PIT 仅在窗口缩到 <~40 观测（如 10–20 天）时才相对 rank 有增量**（借已知形状对极值做收缩、稳 0.09/0.91），在常规 N≥60 天窗下 rank 已够平滑、参数化只增模型风险。
> **关键结论**：只有 A/C（排名/百分位）给出**固定、可移植**的阵营边界（如 skew 边界 0.81、atr 边界 0.67）；B/D 的边界要么随窗口漂移、要么随品种/时段变化，不具固定可移植性——这正是 B0 默认 A 的根因。

#### §1.3.4 转换轴(transition)归一化与阵营适用范围

`transition_flag` / `transition_intensity_signed` 是对 ATR 维度的**另一种归一化**（波动率制度切换检测），独立于上面 A~D 主特征，单独列出。与主特征同理支持多候选，**B0 默认 T1（session-ATR 桶 crossover，继承 poc §6.4）**：

| 候选 | 名称 | 做法 | 输出 |
|:---|:---|:---|:---|
| T1（默认 B0） | session-ATR 桶 crossover | 相邻交易日 session-ATR 档位（low/mid/high）跨档切换检测 + 3 日衰减 | `flag` ∈{0,1} + `τ_signed` ∈[−1,+1] |
| T2 | ATR 原始动量 | 对 session-ATR 水平/rank 做归一化差分 `m = tanh(k·Δ/W)` | 有符号连续 ∈[−1,+1]；需另设阈值 `|m| > Θ_m` 才得二元 `flag`，衡量 ATR 动量/突变量（非严格 crossover，单日尖峰也会被记为转期） |
| T3 | ATR z-score 突破 | ATR 滚动 z-score 跨 ±Θ_z 即 flag=1，符号表方向 | `flag` ∈{0,1} + 符号 |

> 已排除 HMM（原 T4）：数据量（需数年日线，单合约换月仅数月）、批量 vs 在线、概念错位（平滑回望式滞后 vs 即时 crossover）三项冲突，不构成合理备选。
> 候选只改变"ATR → 转期信号"映射，**不改变下方阵营适用范围语义**。

**候选 T1（默认）详细定义**：session 级 ATR 桶近 3 日是否 crossover，产出 `transition_flag` 与 `τ_signed ∈ [−1,+1]`（0=稳定，+1=低→高波动扩张，−1=高→低波动收缩，3 日衰减）：

`transition_intensity_signed(s,t) := Δ_recent × max(0, 1 − age / W)`

- `Δ_recent = level(current) − level(prev_at_crossover)`，档位 `level(low)=0, mid=0.5, high=1` ⇒ `Δ_recent ∈ {−1, −0.5, 0, 0.5, 1}`；
- `age` = 距最近 crossover 交易日数，`W = n_transition_window_days`（=3，可延长）；
- **关键不变性**：`τ_signed = 0 ⟺ 稳定期`（距最近 crossover **≥ W 日**、含 age=W 时 `max(0,1−W/W)=0` 已归零）。故 `τ_signed ≠ 0` 仅在 crossover 后不足 W 日（age < W）成立、稳定期恒为 0，**非恒成立**；
- `flag = 1 ⟺ τ_signed ≠ 0`；供 B 层仓位缩放与定向出场。

**各阵营的 transition 适用范围（在 T1 及任意候选下语义一致）**：

| 阵营 | transition 适用范围（τ_signed） |
|:---|:---|
| L_seg3_lowmid_up | `τ_signed ∈ [−1, +1]`（full，含稳定期 τ=0） |
| L_seg12_high_up | `τ_signed ≠ 0`（= transition_flag=1，仅转期） |
| L_seg2_low_flat | `τ_signed ≠ 0`（仅转期） |
| S_seg12_high_dn | `τ_signed ∈ [−1, +1]`，偏好 |τ_signed| 大（核心） |
| S_seg34_high_dn | `τ_signed ∈ [−1, +1]`，偏好 |τ_signed| 大 |
| S_seg2_mid_dn | `τ_signed ≠ 0`（仅转期） |

> 注：与最原始分类器一致（poc §6.4 / §7.2 / §4 白名单 6 类 × 3 period 全列）——全部 6 类均计算并参与 stable/trans/full 拆分。原 whitelist 仅有 stable/trans/full 三档，**未分 τ_signed 的符号方向**（低→高 + / 高→低 −），故"偏好 |τ| 大"类目前只知道转期 vs 稳定期取舍，不知偏好扩张还是收缩方向，需待 τ_signed 实跑回测校准（B 层待定旋钮）。功能重要性依据见 workbench `va-asymmetry-composite-r1-transition-flag-scope.md`。

**阵营判定规则（汇总）**：

- 偏度 ≤ 0.30 进入多域，≥ 0.60 进入空域，中间 [0.30, 0.60] 为**不参与**；
- 三维（skew × ATR × trend）组合区间刻意不重叠，faction 间互斥，保证**命中 faction 的合约至多落一类**；部分 (skew, ATR, trend) 组合无对应 faction，归为不参与（partial 分类，域内允许有覆盖空隙）；
- 多域与空域严格互斥，不存在同时命中；
- 区间均写在 0~1 归一化尺度（候选 A/C 直接适用；B/D 见各小节对齐规则）。

### §1.4 分类结果的时效

A 层在**开盘前**基于各指标前 N 日行情（见 §0.1 各 `*_rank_win`，skew/atr/trend 天数可独立设定）算好，输出一份"当日清单"：哪些合约是多头候选、哪些是空头候选、
各自属于哪个 tier。这份清单**当天不再变化**，供 B 层全天使用。

### §1.5 波动率（ATR）的计量单位（硬约束）

策略中所有"距离 / 止损 / 仓位 / 成本"的统一度量，都是**入场前 `atr_entry_win` 日化 ATR**（窗口长度见 §0.1），并归一化为
"占价格的万分之几（bps）"。所有后续计算都使用同一把尺子，便于直接比较。

该日化 ATR 同时充当**日波动基准 σ_day**：§3.3 的持仓退出以它为时钟单位，把"日历时间"换成"波动率时间"，
使快 / 慢品种在风险尺度上对齐（scale-invariant）。

## §3 交易执行 · 塑形（B 层 · 日内 · 小时级）

### §3.1 入场

- 时钟：交易日内以 `base_tf` 为基准周期推进；入场判定在 `entry_tf` 候选周期上检查（B0 默认见 §0.1）。
- **入场 = 双层条件同时满足**：(1) A 层当日清单里该合约是某 tier 的多/空候选；(2) 该 tier 的 `entry_mode(tier)` 在 `entry_tf` K 线上触发（见 §3.1.1）。
- **方向**：由 tier 决定（多域 tier → 做多，空域 tier → 做空）；`entry_mode` 的触发方向须与 tier 方向一致（映射见 §3.1.2，禁止反向开仓）。
- **数量**：由 §5 的仓位公式决定（B0 下 `W(tier)=W0`，见 §0.1；只受 §7.2 名义上限压仓影响）。

#### §3.1.1 候选开仓时机（`entry_mode`）

`baseline` 为当前基线（候选即首根 `entry_tf` K 线开仓，无额外过滤）；其余为待验证候选，均在 `entry_tf` 上计算、仅当其触发方向与 tier 方向一致时开仓：

- **`baseline`**：无额外条件，命中候选即开（基线对照）。
- **`boll`**：价格在 `entry_tf` 上触/破**下带** → 多触发；触/破**上带** → 空触发（`boll_win` / `boll_k` 见 §0.1）。
- **`macd`**：`DIF` 上穿 `DEA`（金叉）→ 多触发；下穿（死叉）→ 空触发（`macd_f` / `macd_s` / `macd_sig` 见 §0.1）。
- **`kdj`**：`K` 上穿 `D` 且 `K < kdj_os`（超卖区金叉）→ 多触发；`K` 下穿 `D` 且 `K > kdj_ob`（超买区死叉）→ 空触发（`kdj_*` 见 §0.1）。
- **`rsi`**：`RSI < rsi_os` → 多触发；`RSI > rsi_ob` → 空触发（`rsi_*` 见 §0.1）。
- **`breakout`**：价格突破前 `breakout_n` 根 `entry_tf` K 线**最高** → 多触发；跌破**最低** → 空触发。
- **`prevhi`**：价格突破**前一交易日最高**（`prevbar` 取日线 high）→ 多触发；跌破**前一交易日最低**（low）→ 空触发（趋势突破类）。
- **`openrange`**：今日起始 `openrange_min` 分钟内形成的**开盘区间最高/最低**（B0 默认 `openrange_min=15`，见 §0.1）；`openrange_min` 分钟之后价格**上破**该区间高 → 多触发，**下破**该区间低 → 空触发（开盘区间突破，ORB 类）。

#### §3.1.2 阵营 → 开仓时机映射（建议默认，待验证）

映射原则：tier 的偏度形态决定"顺势 / 反转"风味，开仓时机须**同向且风味匹配**。`顶厚`（右偏）阵营天然对应"逢高做空（fade the top）"，`底厚`阵营对应"逢低做多（buy the dip）"，`强涨势`动量阵营对应"趋势确认"。

| tier | 方向 | 逻辑风味 | 候选 `entry_mode`（均待回测） |
|:---|:---:|:---:|:---|
| L_seg3_lowmid_up | 多 | 动量延续（强涨势） | `macd`（金叉确认）、`breakout`（新高）、`prevhi`（昨日高突破）、`openrange`（开盘区间上破）、`rsi`（超卖，实验） |
| L_seg12_high_up | 多 | 反转（极端底厚 · 恐慌 V 反弹） | `boll`（下带）、`kdj`（超卖金叉）、`rsi`（超卖） |
| L_seg2_low_flat（淘汰） | 多 | —（已淘汰，见 §11） | — |
| S_seg12_high_dn | 空 | 反转（极端顶厚 · 逢高做空） | `boll`（上带）、`kdj`（超买死叉）、`rsi`（超买）、`breakout`（新低）、`prevlo`（昨日低突破）、`openrange`（开盘区间下破），均实验 |
| S_seg34_high_dn | 空 | 反转（次极端顶厚 · 逢高做空） | `boll`（上带）、`kdj`（超买死叉）、`rsi`（超买）、`breakout`（新低）、`prevlo`（昨日低突破）、`openrange`（开盘区间下破），均实验 |
| S_seg2_mid_dn | 空 | 反转（中高顶厚 · 逢高做空） | `boll`（上带）、`kdj`（超买死叉）、`rsi`（超买）、`breakout`（新低）、`prevlo`（昨日低突破）、`openrange`（开盘区间下破），均实验 |

> B0 默认全 tier `baseline`（保留当前"不择时"基线作对照）。上表为开启择时候选时的**候选挂法清单**：`boll` / `kdj` / `rsi` 属反转极值类（与顶厚/底厚阵营同向），`macd` / `breakout` / `prevhi` / `openrange` 属趋势确认/突破类（与动量阵营同向）；其中 `prevhi`/`prevlo` 为前一交易日高低突破、`openrange` 为今日开盘 `openrange_min` 分钟区间突破，均与 S_* 反转逻辑存在风味冲突，标"实验"、须逐 `entry_mode` 回测验证增益后再固定，未验证前不得声称择时有效。

### §3.2 止损（SL）

各 tier 止损距离为 `K_SL(tier) × ATR`（ATR 见 §1.5），`K_SL(tier)` 的 B0 取值与候选见 §0.1（多域 tier 与空域 tier 分别取多/空域候选）。价格反向触及止损价即平仓。

### §3.3 时间退出（主平仓机制 · 波动率时间）

持仓不按日历时间计，而按**波动率时间**计：入场后逐 `base_tf` K 线累计已实现波动，
累计量达到该 tier 的**波动率预算 `H_vol(tier)`**（单位为 σ_day 倍数，σ_day 见 §1.5）时，
在下一根 `base_tf` K 线收盘平仓。

```
累计已实现波动 = Σ(入场后各 `base_tf` K 线的波动增量)
当  累计已实现波动 ≥ H_vol(tier) × σ_day  时触发时间退出
```

| tier | 波动率预算 `H_vol(tier)`（× σ_day） |
|:---|:---:|
| L_seg3_lowmid_up | `B_L` |
| L_seg12_high_up | `B_L` |
| L_seg2_low_flat（淘汰） | `B_L` |
| S_seg12_high_dn | `B_S` |
| S_seg34_high_dn | `B_S` |
| S_seg2_mid_dn | `B_S` |

`B_L / B_S` 的 B0 取值**校准到复现原中位持仓**（锚点见 §0.1；快品种日历时间自动更短、慢品种更长）。
优先级：**止损 > 时间退出**（同时触发取止损）。

### §3.4 移动止损（不启用）

本策略默认不启用移动止损。

### §3.5 硬止盈（不启用）

本策略默认不设硬止盈目标。

## §4 品种准入（按 tier 白名单，B0 = 全品种）

### §4.1 设计原则

品种成员资格**按 tier 独立配置**，而非全局一刀切。每个 tier 持有一份**品种白名单 `sym(tier)`**，
只有落在白名单内的品种才允许进入该 tier 的开仓逻辑。这把"哪些品种能交易"从参数轴降级为
**准入配置（governance）**：默认全开，事后按表现个别裁剪，不作为预优化搜索轴。

> 与早期 S2（按 A/B/C 类型全局筛选）的区别：S2 是把"类型"当优化轴去挑，实测净夏普增量
> 为负（反向拖累，见 §11）已被否决；`sym(tier)` 默认全品种，仅作风险治理式的个别裁剪。

### §4.2 默认配置（B0）

所有 6 个 tier 的白名单均 = 全品种：

```
sym(tier) ≡ Universe   （∀ tier）
```

即等价于旧 S1。参数仍只按 tier 分组（§0.1 的 `K_SL / H_vol / W`），品种不入参。

### §4.3 裁剪机制（事后治理）

测试后如发现某品种表现不稳定，可从对应 tier 的白名单中剔除该品种；如发现某整类 tier
（如 L_seg2_low_flat）信息比率为负，可将整个 tier 移出 `tier_def` 活跃集。两类裁剪均为
配置变更，不影响其余 tier 的参数与逻辑。

### §4.4 归因维度

- **按分类（tier）归因**：6 个 tier 各自的收益 / 信息比率 / 交易次数。
- **按品种归因**：逐 symbol 的收益 / 信息比率，定位不稳定个体。
- （可选）**按产品组 A/B/C 类型交叉归因**：仅用于事后分析，不参与入参筛选（区别于已否决的旧 S2）。

## §5 仓位与加权（B0 = 等权 × 等权）

### §5.1 信号强度加权 → 等权

强度权重按 tier 索引为 `W(tier)`。B0 下所有 tier 均取等权方案 `W0`，即 `W(tier) ≡ 1`（不做按信号强弱的加权）。

### §5.2 多空权重 → 等权

B0 下多头与空头的权重相同（均为 1），不做多空倾向性加权。

### §5.3 单品种单 tier 的目标仓位（压仓前）

目标名义仓位 = 方向权重 × 强度权重 ×（单笔可承受风险 ÷ 止损距离）× 账户权益。

其中：

- 单笔可承受风险 = 账户权益的 **2%**（见 §7.1）；
- 止损距离 = `K_SL(tier)` × ATR 占价格的比例（B0 各 tier 取值见 §0.1）；
- 方向权重、强度权重 `W(tier)` 在 B0 下取 `W0`（见 §0.1）。

含义：固定每笔最多亏 `RiskPerTrade`，止损越远（波动越大）则下的注越小。

## §6 交易成本（硬约束 · 真实成本口径）

### §6.1 单边成本

每笔单边成本 = 佣金 + 滑点，按下述方式计算（具体数值查合约规格表 `contract_specs.py`）：

- 佣金：**交易所手续费 ×（1 + broker_markup）**，再折算为 bps。
  - 交易所手续费：固定值（元/手）或成交额费率（`is_rate=True`，即价格 × 数量 × 乘数 × 费率）。
  - 期货公司加收比例由各品种在 `ContractSpec`（`contract_specs.py`）中携带，属实现/config 数据、非策略旋钮，不在本规格的参数表里。
    例外：`ap/cu/lh/j/pm/ic/im/au/wh/ec/zc` 等 11 个交易所基准手续费本就偏高、4 跳难覆盖成本的品种已下调至优惠档，
    避免高基准上失真。以上均为占位值，实盘以账户结算单为准逐品种校准。
- 滑点：按"滑点档数 × 最小变动价位 × 数量 ÷ 价格"折算为 bps（具体档数见 `contract_specs.py`，非本规格关心）。

成本单位统一为"占名义本金的 bps"，与 §1.5 的 ATR 同口径。

### §6.2 成本相对波动的倍率（诊断用）

成本相对波动倍率 = 单边成本 bps ÷ 入场 ATR bps，用于诊断摩擦大小。

### §6.3 净收益

净收益 = 毛收益 − 入场成本 − 出场成本（近似为毛收益 − 2 × 单边成本）。

## §7 风控

### §7.1 单笔止损 = `RiskPerTrade` × 权益（硬约束，见 §0.1）

这是 §5.3 仓位公式的输入（每笔最多亏 `RiskPerTrade`），同时也是事后保护：若实际损失超过约 `RiskPerTrade` 对应阈值，
按止损价执行。

### §7.2 总名义暴露上限（默认见 §0.1 之 `Cap`）

所有持仓的名义金额之和 ≤ **`Cap` × 账户权益**。超出时按以下优先级砍仓：

1. 先砍强度权重最低的持仓（B0 下全为 1，退化为顺序/随机）；
2. 再砍离到期最近的持仓；
3. 最后按先进先出（FIFO）。

### §7.3 保证金约束（自动满足）

期货保证金率约 5%~12%，`Cap` 名义暴露对应 5%~12% 保证金占用，低于约束上限，不主动管理。

### §7.4 单日最大损失熔断（可选 · 默认见 §0.1 之 `cb`）

若当日累计亏损 ≤ `cb` 阈值 × 权益，则当日剩余时间不再开新仓（已有持仓按原规则退出）。

## §8 事件并发与去重

### §8.1 合约内去重

同一合约在去重窗口内只接受第一个有效信号，后续信号忽略。B0 默认窗口见 §0.1 之 `dedup`。

### §8.2 多 tier 并发

同一时刻同一合约只能匹配唯一 tier（§1.2 逻辑 + §1.3 区间，完全互斥），不存在多 tier 并发。
跨品种可并发，但受 §7.2 名义上限聚合压仓。

### §8.3 多空互斥

同一合约同一时刻不可能同时触发多域与空域（偏度 ≤0.30 与 ≥0.60 严格互斥），
不做自对冲；跨品种多空可并存。

## §9 归因口径（硬约束）

任何声称"某方案平均收益为正"的结论，必须同时报告以下四项：

- 平均毛收益（bps）；
- 毛收益方差；
- Itô 修正后的真实漂移 = 平均收益 − 方差/2；
- 真实漂移 > 0 的概率（基于"合约—日期"为单位的聚类自助法）。

只有真实漂移 > 0 且概率 ≥ 0.95，才可判为"真实正 edge"。

## §10 输出：成交明细字段

每笔实验输出交易级明细，至少包含：合约、品种、分类(tier)、入场/出场时间、方向、阵营、
入场/出场价、出场原因（止损 / 时间 / 压仓）、入场 ATR、压仓前后数量、两个权重、
毛收益、两次成本、净收益（bps 与计价币）、出入场前后权益。

## §11 本策略明确不采用的方案（审计）

以下方案经测试未通过，不作为 B0 的活动配置，仅留作审计：

- **全局按类型筛选（旧 S2）**：把各 tier 白名单整体收窄到单一 A/B/C 类型，相对 B0 净夏普增量为负，反向拖累；已由 `sym(tier)` 默认全品种取代，不采用。
- **信号强度加权（W1/W2/W3）**：按偏度距离 / 波动匹配 / 三维乘积加权，均未达增量门槛，不采用。
- **多空加权（VW1/VW2）**：按信息比率比例 / 频率平衡加权，未达增量门槛，不采用。
- **L_seg2_low_flat 阵营**：塑形后信息比率为负，全品种淘汰。

## §12 策略内可调参数

B0 之外，本策略仅预留两处可调旋钮（不改动上述硬约束）：

1. **总名义暴露上限 `Cap`**：可在其候选集间调节（主可调轴，候选见 §0.1）。
2. **合约内去重窗口 `dedup`**：可在其候选集间调节（次可调轴，候选见 §0.1）。
3. 塑形参数（止损倍数 `K_SL(tier)`、波动率持仓预算 `H_vol(tier)`）可在 ±30% 范围内做描述性敏感性，不作为主调节。

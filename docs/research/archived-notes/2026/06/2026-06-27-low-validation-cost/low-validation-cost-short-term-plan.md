## 1. 短期目标

短期目标不是盲选 Donchian、布林带或其他入口信号，也不是继续扩大 MA/ATR 参数搜索，而是把研究框架落到一组可迭代的短期实验中。

短期实验不是单轮决策，而是多轮候选价格区间验证。每轮实验完成后，实验记录归档到 [`docs/research/archived-notes`](../)，短期计划只保留下一步实验方向和共同评价标准。

当前阶段实验优先验证一件事：

```text
是否存在一类低验证成本价格区间，
能够用清晰失败边界判断方向选择是否错误，
价格在边界处是否表现出足够接受 / 拒绝质量，
目标空间是否足以覆盖手续费和滑点，
并且有限止损放宽能够改善成本后期望，
而不只是提高表面胜率。
```

本阶段不要求立即得到可上线策略，也不直接追求“高胜率”。每一轮都必须先证明：候选价格区间确实比随机入场更容易低成本知道自己错了。

## 2. 实验哲学

本阶段延续 [策略长期共识：共识价格区间下的账户风险结构塑形框架](../../../roadmap/strategy-research-framework.md) 的交易哲学，采用“低验证成本价格区间验证”作为短期实验哲学。

| 组成      | 定位                       |
| ------- | ------------------------ |
| 候选价格区间  | 研究对象，不是预测信号              |
| 方向假设    | 用于定义交易方向，不假设能稳定预测未来      |
| 严格失败边界  | 理论上证明方向假设失败的位置           |
| 接受 / 拒绝质量 | 判断价格是否真的在边界处形成结构证据，而不是机械触碰指标线 |
| 成本空间    | 回测前预筛目标空间是否足以覆盖手续费和滑点   |
| 实际止损边界  | 为吸收正常噪声后设置的退出位置          |
| 止损放宽    | 诊断是否能改善成本后期望，不作为默认优化方向  |
| 小目标止盈   | 用于观察高胜率低盈亏比结构是否成立        |
| 时间退出    | 入场后无法快速验证则退出，避免长期占用样本    |
| 冷却 / 暂停 | 连续亏损或异常波动后停止交易，保护小资金生存能力 |

核心判断不是：

```text
这个入场信号是否更准？
```

而是：

```text
这个价格区间是否更容易低成本判断方向选择是否失败？
价格对边界的接受 / 拒绝质量是否足够？
目标空间是否足以覆盖手续费和滑点？
止损放宽带来的变化，是否改善成本后期望而不只是提高胜率？
```

## 3. 当前实验命名

下一阶段优先实验命名为：

```text
low-validation-cost-r3
```

含义：

```text
用一类清晰价格结构作为候选区间，
定义方向假设、严格失败边界和实际止损边界，
验证该区间是否具备低验证成本，
验证边界接受 / 拒绝质量和成本空间是否足够，
以及止损放宽是否能改善成本后期望。
```

N 周期突破、布林带、均线回踩等都只是候选价格区间表达，不是研究主角。

## 4. 每轮实验核心问题

每轮实验按顺序回答以下问题：

```text
是否存在明确的候选价格区间
→ 该区间的方向假设是否清楚
→ 严格失败边界是否可定义
→ 入场到严格失败边界的验证成本是否足够低
→ 严格失败边界快速再触及率是否可接受
→ 价格接受 / 拒绝质量是否足够
→ 目标空间是否足以覆盖手续费和滑点
→ 实际止损边界是否只是有限放宽
→ 放宽止损后成本后期望是否改善
→ 单笔尾部亏损和亏损簇是否仍可承受
→ 收益是否不依赖单品种、单参数或单时段
```

如果这些问题无法回答，或回答是否定的，不进入参数优化。

## 5. 候选区间来源

### 5.1 成熟体系只作为结构语言

每轮可以借用成熟交易体系寻找候选区间，但不能直接采纳它们的预测解释。

| 来源体系                     | 可提取的候选区间                         | 本项目中的用途        |
| ------------------------ | -------------------------------- | -------------- |
| Price Action             | 前高前低、区间边界、趋势线、K 线反转区             | 定义接受 / 拒绝和失败边界 |
| Wyckoff                  | spring、upthrust、test、吸筹 / 派发区边界  | 定义假突破或测试失败     |
| Market Profile / Auction | VAH / VAL、POC、Initial Balance 边界 | 定义价值区接受 / 拒绝   |
| 均值回归                     | 布林带外、VWAP 偏离、Z-score 极端区         | 定义继续扩散即失败      |
| 突破交易                     | 通道边界、前高前低、压缩后突破位                 | 定义突破接受 / 失败    |
| 趋势回踩                     | 均线回踩、前突破位回踩、趋势通道回踩               | 定义趋势延续失败边界     |

每个候选区间都必须完成结构塑形翻译：

```text
传统交易理由是什么
→ 对应哪个价格结构
→ 方向假设是什么
→ 严格失败边界在哪里
→ 入场到失败边界的验证成本是多少
→ 价格接受 / 拒绝证据是什么
→ 严格边界是否容易被正常噪声快速再触及
→ 目标空间是否足以覆盖手续费和滑点
→ 是否需要放宽止损吸收正常噪声
→ 放宽止损后的成本后期望是否改善
```

### 5.2 当前优先候选

每轮只选一个最简单区间，不同时堆叠多个复杂体系。

已完成的 `low-validation-cost-r1/r2` 说明，普通指标边界和普通前 N 高低点虽然容易定义，但容易落入“边界清楚、接受质量不足、成本空间不足”的陷阱。下一阶段优先转向更可能被市场共同观察的结构边界。

当前优先候选：

| 候选区间 | 验证问题 |
| -------- | -------- |
| Initial Balance 高 / 低点 | 日内早盘边界是否能提供更清晰的接受 / 拒绝证据，假突破后是否能低成本验证失败 |
| 前日高 / 低点、昨收、开盘区间边界 | 共识边界附近的突破失败或重新接受是否比普通前 N 高低点更稳定 |
| VAH / VAL / POC 等价值区边缘 | 市场是否接受更高 / 更低价值区，边界外失败是否能提供短线反向机会 |
| 支撑 / 阻力 / 密集成交区边缘 | 密集成交区边缘的接受 / 拒绝是否能降低快速反向 MAE |

暂不优先继续：

| 候选区间 | 原因 |
| -------- | ---- |
| 布林带收缩后直接突破 | 已验证成本后未通过，不继续围绕同一入口调参 |
| 布林带突破后回踩边界再启动 | 样本减少且止损放宽不稳定改善，不继续微调 |
| 普通前 N 高 / 前 N 低突破后首次回踩 | 低验证成本成立，但严格边界快速再触及率过高 |
| 当前趋势均线回踩定义 | 强重新接受有价值，但当前结构仍未转正，需重新定义候选区而非调参数 |

## 6. 实验结构

### 6.1 区间定义

每个实验必须显式定义：

| 项目     | 说明                  |
| ------ | ------------------- |
| 候选区间   | 例如 Initial Balance 高 / 低点、前日高低点、价值区边缘 |
| 方向假设   | 例如突破失败后重新接受区间内部，或边界外接受后短距离延续 |
| 严格失败边界 | 例如假突破极值、重新跌回边界内、重新接受价值区另一侧等 |
| 严格失败距离 | 入场价到严格失败边界的距离       |
| 接受 / 拒绝证据 | 例如重新收回边界内、边界外无法维持、强重新接受、失败测试等 |
| 目标空间 / 成本 | 目标止盈空间相对手续费和滑点的倍数 |
| 实际止损边界 | 为吸收正常噪声后设置的止损位置     |
| 止损放宽倍数 | 实际止损距离 / 严格失败距离     |
| 目标止盈   | 固定小目标、波动倍数或区间目标     |
| 时间退出   | 未在预期时间内验证成功则退出      |

### 6.2 止损放宽测试

每轮至少比较两类设置：

| 设置       | 目的               |
| -------- | ---------------- |
| 严格失败边界止损 | 观察原始低验证成本区间的基础表现 |
| 适度放宽止损   | 诊断吸收正常噪声后成本后期望是否改善 |

重点不是找到最优止损，而是回答：

```text
止损放宽是否真的改善成本后期望，
还是只是在购买噪声并提高表面胜率？
```

若放宽止损只提高胜率，但平均亏损、最大亏损、亏损交易 MAE 或组合净期望恶化，则该结构不进入参数搜索。

### 6.3 暂不处理

每轮不做：

- 大规模 Optuna 搜索；
- Walk-Forward；
- 仓位优化；
- 多体系条件叠加；
- 亏损后加仓摊低；
- 为了提高胜率取消或无限放宽止损；
- 为了凑交易数放宽候选区间定义；
- 根据单次收益结果频繁切换候选区间。

## 7. 评价顺序

每轮按以下顺序判断：

```text
候选价格区间是否清晰
→ 严格失败边界是否可定义
→ 严格失败距离是否足够低
→ 严格失败边界快速再触及率是否可接受
→ 接受 / 拒绝质量是否足够
→ 目标空间是否足以覆盖成本
→ 实际止损放宽是否可控
→ 放宽后成本后期望是否改善
→ 单笔尾部亏损是否可承受
→ 交易机会是否足够评价
→ 参数和品种是否不依赖单点偶然性
```

收益率、夏普、最大回撤和最优参数只作为后置参考，不作为每轮主证据。

## 8. 关键观察指标

每轮实验完成后，需形成实验记录并归档到 [`docs/research/archived-notes`](../)。

| 字段          | 说明                          |
| ----------- | --------------------------- |
| 实验版本        | 例如 `low-validation-cost-r3` |
| 候选价格区间      | 例如 Initial Balance 高 / 低点、前日高低点、价值区边缘 |
| 结构来源        | Price Action、布林带、趋势回踩等      |
| 传统解释        | 该体系通常如何解释这个位置               |
| 结构塑形解释      | 本项目为什么认为它是低验证成本区间           |
| 方向假设        | 本轮交易方向假设                    |
| 严格失败边界      | 理论上方向假设失效的位置                |
| 严格失败距离      | 入场到严格失败边界的距离                |
| 严格边界快速再触及率 | 判断边界是否过近、是否主要承受正常噪声 |
| 接受 / 拒绝证据 | 价格在边界处形成的结构证据，例如重新接受、失败测试、无法维持 |
| 接受质量诊断 | 接受证据是否降低快速反向 MAE 和严格边界再触及 |
| 目标空间 / 单笔成本 | 回测前预筛小目标是否足以覆盖手续费和滑点 |
| 实际止损边界      | 真实止损位置                      |
| 实际止损距离      | 入场到实际止损的距离                  |
| 止损放宽倍数      | 实际止损距离 / 严格失败距离             |
| 候选信号数       | 区间定义后产生的候选机会                |
| 实际成交数       | 真实开仓次数和完整交易数                |
| 严格止损胜率      | 使用严格失败边界止损时的胜率              |
| 放宽止损胜率      | 使用实际止损边界时的胜率                |
| 胜率提升        | 放宽止损后胜率提升幅度                 |
| 盈亏比下降       | 放宽止损后平均亏损扩大幅度               |
| 盈亏平衡胜率      | 根据盈亏比和成本计算的最低胜率             |
| 胜率安全边际      | 实际胜率 - 盈亏平衡胜率               |
| 平均盈利        | 金额、tick、相对成本倍数              |
| 平均亏损        | 金额、tick、相对平均盈利倍数            |
| 最大单笔亏损      | 会回吐多少笔平均盈利                  |
| 亏损簇         | 最大连续亏损次数、最差连续亏损金额           |
| 成本影响        | 成本 / 平均盈利、毛收益到净收益差异         |
| MAE         | 进场后最大不利波动，用于判断止损是否合理        |
| MFE         | 进场后最大有利波动，用于判断止盈是否过早或过晚     |
| 期望净变化       | 止损放宽后成本后期望是否改善              |
| exit reason | 止盈、严格失败、实际止损、时间退出、冷却暂停等分布   |
| 结论          | 区间有效、需调整边界、需调整止损放宽、还是淘汰     |

## 9. 通过标准

每轮不是要求立即盈利，而是要求看到“低验证成本区间 + 接受质量 + 成本空间 + 有限止损放宽诊断”的结构塑形迹象。

| 维度    | 通过标准                           |
| ----- | ------------------------------ |
| 区间清晰度 | 候选区间可以客观定义，不依赖主观画线             |
| 失败边界  | 严格失败边界清楚，且距离入场较近               |
| 边界噪声  | 严格失败边界快速再触及率不能过高，不能主要依赖放宽止损修复 |
| 接受质量  | 价格在边界处有可定义的接受 / 拒绝证据，且能降低快速反向 MAE |
| 成本空间  | 目标空间和平均盈利显著大于手续费和滑点             |
| 止损放宽  | 放宽倍数有限，且事先定义                   |
| 期望净变化 | 放宽止损后成本后期望改善，而不只是胜率提高          |
| 单笔质量  | 平均盈利显著大于手续费和滑点                 |
| 尾部亏损  | 最大亏损不会回吐过多平均盈利                 |
| 亏损簇   | 连续亏损在小资金可承受范围内                 |
| 交易机会  | 不依赖极低频单笔样本，也不靠放宽区间定义凑交易数       |
| 参数邻域  | 边界和止损放宽小幅变化不应断崖式退化             |
| 可解释性  | 能说明收益来自价格区间、失败边界、止损放宽、止盈还是时间退出 |

未通过时，不继续围绕同一入口调参，而是先判断失败来自哪一层：

```text
候选区间无法客观定义
→ 严格失败边界不清楚
→ 验证距离过大
→ 严格失败边界快速再触及率过高
→ 接受 / 拒绝质量不足
→ 目标空间不足以覆盖成本
→ 放宽止损没有改善成本后期望
→ 平均盈利太接近成本
→ 止损放宽后尾部亏损不可承受
→ 品种或周期不适合小资金
```

## 10. 短期执行顺序

```text
实现 low-validation-cost-r3
→ 选择一个更有市场共识的候选价格区间：Initial Balance、前日高低点、开盘区间或价值区边缘
→ 完成结构塑形翻译：传统解释、方向假设、严格失败边界、接受 / 拒绝证据
→ 回测前检查目标空间 / 单笔成本
→ 跑严格失败边界止损版本
→ 跑有限放宽止损版本
→ 输出严格失败距离、严格边界快速再触及率、接受质量诊断
→ 输出实际止损距离、止损放宽倍数
→ 输出严格止损胜率、放宽止损胜率、胜率提升、盈亏比下降
→ 输出盈亏平衡胜率、胜率安全边际、成本后期望净变化
→ 判断低验证成本区间、接受质量和成本空间是否同时有效
→ 若有效，再做边界邻域和多窗口检查
→ 若无效，先判断失败层级，再决定是否更换候选价格区间
```

## 11. 区间替换原则

只有在当前区间完成完整诊断后，才比较不同候选区间。

区间替换的目的不是寻找神奇信号，而是寻找更适合结构塑形的低验证成本实验场。

可替换区间包括：

| 区间                | 用途                 |
| ----------------- | ------------------ |
| Initial Balance 高 / 低点 | 检查日内早盘共识边界的接受 / 拒绝和假突破 |
| 前日高 / 低点、昨收、开盘区间边界 | 检查更高共识度价格边界是否优于普通前 N 高低点 |
| VAH / VAL / POC 等价值区边缘 | 检查市场是否接受更高 / 更低价值区 |
| 支撑 / 阻力 / 密集成交区边缘 | 检查密集成交区边缘的接受 / 拒绝是否有清晰边界 |
| 通道边界附近            | 检查通道延续或回落的验证距离是否较短 |
| 重新定义后的趋势回踩区域 | 只在引入更强趋势阶段过滤或结构级失败识别后重启 |
| 重新定义后的布林带边界 | 只在解决指标边界共识不足和目标空间不足后重启 |

若收益分布只在单一入口、单一参数、单一品种或单一时段上成立，不视为结构有效。

## 12. 暂停条件

出现以下情况时，不继续投入当前结构：

1. 找不到可客观定义的低验证成本价格区间；
2. 严格失败边界无法定义，或距离入场过远；
3. 严格失败边界快速再触及率过高；
4. 接受 / 拒绝质量不足，无法降低快速反向 MAE；
5. 目标空间不足以覆盖手续费和滑点；
6. 放宽止损不能改善成本后期望，只是提高表面胜率；
7. 胜率提升无法覆盖盈亏比下降和成本；
8. 平均盈利太小，成本 / 平均盈利过高；
9. 单笔最大亏损会回吐过多平均盈利；
10. 亏损簇超出小资金可承受范围；
11. 为维持胜率需要无限放宽止损、加仓摊低或长期扛亏；
12. 交易数只能靠放宽区间定义获得；
13. 收益主要来自单合约、单时段或尖峰参数；
14. 无法解释收益分布改善来自哪一层结构。

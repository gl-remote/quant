## 1. 目标

围绕 [ma_strategy.py](../../../workspace/strategies/ma_strategy.py)，验证 MA 是否能形成可迁移、可解释、扣除成本后仍具备长期正期望的中低频趋势策略。

当前一个多月测试数据下，目标策略形态是：

| 维度 | 目标 |
|------|------|
| 交易频率 | 单品种每周约 3~4 次进场，每月约 10 多次进场 |
| 持仓周期 | 平均持仓 1~2 天 |
| 成本敏感性 | 不被手续费和滑点吞噬 |
| 鲁棒性 | 不依赖单品种、单窗口、单笔持仓或尖峰参数 |
| 可解释性 | 能说明改善来自入场、出场、过滤、仓位还是成本 |

这个频率区间的意义：低于该区间，一个多月数据难以观察策略成色；高于该区间，容易退化为 1m 噪声交易，被滑点和手续费吞噬。

## 2. 当前结论

当前 MA 在本地豆粕短样本上**没有找到可继续主线投入的候选区域**。

核心判断：

1. 不加反向退出时，MA 基本只有开仓，没有足够完整交易样本。
2. 加同级别即时反向退出后，交易数恢复，但容易过度交易、成本放大、品种分化。
3. `ma8` 改成 `15m` 背景 + `5m` 触发后，噪声明显下降，但频率整体不足。
4. 冷却、最短/最长持仓、反向确认能改善交易质量，但不能创造足够信号。
5. 15m MA 参数没有形成稳定平台，继续微调 `sma_short/sma_long` 价值不高。
6. 当前阻塞是 **alpha / 触发密度不足**，不是优化器、仓位或 trial 数不足。

因此：**不建议继续在 MA 上扩大 trial。** 如果继续 MA，应先扩大历史窗口或品种；如果仍局限当前数据，更值得转向通道突破、Donchian、布林带突破、回踩均线后再启动等新触发结构。

## 3. 实验摘要

### 3.1 Baseline 与退化解保护

初始 MA baseline 使用 `v2.0.0-ma7`，在 `DCE.m2601 / DCE.m2603 / DCE.m2605` 上跑多品种搜索。

| 事项 | 结论 |
|------|------|
| 初始 baseline | 每条回测均只有 1 笔交易，不能作为正期望证据 |
| 低交易活跃度惩罚 | [optimizer.py](file:///Users/gaolei/Documents/src/quant/workspace/backtest/optimizer.py) 加入低交易次数惩罚后，所有低活跃度 trial 得分被压到 `-999` |
| 暴露的问题 | 优化器保护生效，但 MA 入场/出场结构无法产生足够样本 |
| 决策 | 不继续扩大 trial，先拆解信号和出场结构 |

设计含义：优化器只能避免选择退化解，不能创造有效交易机会。若策略结构只产生单笔持仓，参数搜索没有意义。

### 3.2 信号消融与反向退出

为拆解信号，MA 增加 `signal_profile`：`sma_only` / `sma_macd` / `sma_kdj` / `full`。

第一次消融结果：

| profile | 结果 | 结论 |
|---------|------|------|
| `sma_only` | 每品种 1 笔 | 只有开仓，样本不足 |
| `sma_macd` | 每品种 1 笔 | MACD 改变方向，但不解决样本不足 |
| `sma_kdj` | 每品种 1 笔 | KDJ 未改善交易活跃度 |
| `full` | 每品种 1 笔 | 看似收益为正，但仍是单笔持仓 |

随后增加 `exit_on_reverse_signal`，持仓时遇到当前 profile 的反向信号则先平仓、不反手。

| profile | 总交易数 | 结果 | 结论 |
|---------|----------|------|------|
| `sma_only` | 1251 | 严重亏损并爆仓 | 信号太敏感，成本和噪声吞噬账户 |
| `sma_macd` | 265 | 两个品种正收益，一个品种爆仓 | 有一定候选价值，但缺少约束 |
| `sma_kdj` | 35 | 交易数较合理，但整体负收益 | 频率接近，但 alpha 不足 |
| `full` | 3 | 仍然极稀疏 | 样本不足 |

设计含义：反向退出可以恢复完整交易样本，但同级别即时反向退出太敏感，会形成“慢一点没交易，快一点过度交易”的断崖。

### 3.3 三轮周期/参数频率收敛

目标：只通过 profile、均线快慢和降仓位，把 MA 拉到目标频率区间。

统一设置：`position_ratio=0.1`，`exit_on_reverse_signal=true`，关闭参数搜索，回测 3 个豆粕合约。

| 轮次 | 设置 | 结果 | 结论 |
|------|------|------|------|
| 1 | `sma_macd`，`10/40` | 进场次数 `5 / 3 / 1` | 频率偏低，持仓未达 1~2 天 |
| 2 | `sma_macd`，`5/20` | 进场次数 `29 / 3 / 1` | DCE.m2601 过度交易，品种分化严重 |
| 3 | `sma_kdj`，`5/20` | 进场次数 `15 / 6 / 3` | DCE.m2601 频率接近目标，但持仓只有数小时且跨品种不稳 |

设计含义：均线参数可以调节频率，但不能稳定解决“频率合适 + 持仓舒适 + 跨品种一致”。后续不应继续盲目微调 `sma_short/sma_long`。

### 3.4 ma8 九轮验证

[ma_strategy.py](file:///Users/gaolei/Documents/src/quant/workspace/strategies/ma_strategy.py) 升级为 `ma8`：

- 方向结构改成 `15m` SMA 背景 + `5m` MACD/KDJ 触发；
- 去掉 `1m` 方向确认；
- 新增 `trend_macd` / `trend_pullback` / `trend_full`；
- 新增 `reverse_confirm_bars`、`min_hold_bars`、`max_hold_bars`、`entry_cooldown_bars`、`trend_gap_atr`；
- 测试通过：`ruff check`、`ruff format`、`uv run pytest workspace/tests/strategies/test_ma_strategy.py --tb=short`。

统一设置：`position_ratio=0.1`，关闭参数搜索，单次回测 3 个豆粕合约。

| 轮次 | 设置 | 结果 | 结论 |
|------|------|------|------|
| 1 | `trend_macd 20/60` | DCE.m2603 持仓接近目标，整体频率偏低 | 有降噪效果，但样本不足 |
| 2 | `trend_pullback 20/60` | 更稀疏 | 回调触发未形成样本 |
| 3 | `trend_full 20/60` | 更稀疏，DCE.m2603 小幅为正 | 双确认过严 |
| 4 | `trend_macd + reverse_confirm=6` | 改善 DCE.m2603，频率更低 | 反向确认提升质量但压低样本 |
| 5 | `trend_macd + cooldown=240` | DCE.m2603 收益改善，频率不足 | 冷却有效但不创造信号 |
| 6 | `trend_macd + min/max hold` | 持仓更接近目标，交易数不足 | 持仓约束有效但不增频 |
| 7 | 第 6 轮 + `trend_gap_atr=0.2` | DCE.m2603 保留，其他品种更少 | 趋势过滤应后置 |
| 8 | 第 6 轮 + 快参数 `10/40` | 退化为单笔样本 | 快参数没有形成平台 |
| 9 | 第 6 轮 + 慢参数 `30/90` | 部分品种持仓接近 1~2 天，频率仍不足 | 慢参数也无稳定平台 |

设计含义：`ma8` 基本排除了仓位、1m 噪声和即时反向退出带来的伪问题；剩下的主要问题是 MA 在当前样本中没有足够有效触发。

## 4. 设计沉淀

1. **频率目标要前置为硬约束**  
   当前短样本下，单品种每周 3~4 次、每月 10 多次、平均持仓 1~2 天，是判断策略成色的最低可观察区间。

2. **均线参数只能调频，不能解决结构问题**  
   均线放慢会交易不足，放快会在部分品种上过度交易；若品种间频率分化明显，不应继续微调参数。

3. **同级别即时反向退出过于敏感**  
   它能恢复交易数，但会带来高频反复开平。更合理的是更高周期反向、连续 N 根确认、最短持仓或平仓后冷却。

4. **15m 背景是降噪器，不是频率修复器**  
   慢周期背景能减少 1m 噪声和爆仓噪声，但也会压低交易次数，不能单独满足目标频率。

5. **冷却/持仓约束改善交易质量，但不能创造信号**  
   `entry_cooldown_bars`、`min_hold_bars`、`max_hold_bars` 能改善个别品种收益和持仓形态，但基础触发不足时只会筛选交易。

6. **趋势过滤应后置**  
   `trend_gap_atr` 能提升个别交易质量，但在基础信号稀疏时会进一步压低样本量。

7. **MA 当前没有稳定参数平台**  
   `10/40`、`20/60`、`30/90` 都无法同时满足频率、持仓和跨品种一致性。

8. **profit/loss 语义必须明确**  
   止盈表达式应使用 `profit_*`，止损表达式应使用 `loss_*`；不要使用绝对盈亏幅度同时驱动止盈和止损，否则会污染出场归因。

9. **样本形态未达标前，不应扩大 trial**  
   如果交易频率、持仓周期和成本占比未进入可评价区间，扩大优化次数只会放大退化解和过拟合风险。

## 5. 当前决策

| 决策 | 说明 |
|------|------|
| 暂停 MA 主线优化 | 当前阻塞是 alpha / 触发密度不足，不是 trial 数不足 |
| 保留 `ma8` baseline | 用于后续和 ATR、通道类策略对照 |
| 若继续 MA | 先扩大历史窗口或品种，确认是否只是豆粕短窗口限制 |
| 若仍使用当前数据 | 优先尝试通道突破、Donchian、布林带突破、价格回踩均线后再启动 |
| 退出条件 | 扩大窗口/品种后仍无法满足频率、持仓和成本约束，则 MA 仅保留 baseline 维护 |

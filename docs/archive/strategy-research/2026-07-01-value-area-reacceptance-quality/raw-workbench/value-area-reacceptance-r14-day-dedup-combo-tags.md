## 1. 实验问题

R13 显示：

```text
bad = poc_edge_bucket == edge or migration_bucket == away
```

是很强的坏结构组合标签。

但 R13 的 41 笔清算样本包含 2 ticks / 3 ticks 对同一天、同方向、同一结构的重复记录。因此本轮验证：

```text
按 symbol / trading_day / direction 做日级去重后，
edge_or_away 是否仍然是有效风险标签？
```

本轮不重新跑回测，不改变交易信号，只处理 R12 的清算样本。

## 2. 数据与去重口径

数据来源：

```text
DCE.m2601: 450 / 451
CZCE.SR601: 452 / 453
SHFE.rb2601: 454 / 455
```

原始清算样本：

```text
raw_clearings = 41
```

去重键：

```text
symbol + trading_day + direction
```

去重后日级结构数：

```text
day_structures = 23
```

为了避免单一选样本偏差，本轮同时使用两种口径：

1. **优先 2 ticks**
   同一日级结构内，优先保留 2 ticks 样本；若没有 2 ticks，再保留最早样本。

2. **同结构平均 PnL**
   同一日级结构内，将 2 ticks / 3 ticks 的 PnL 取平均，标签使用该结构的首个样本标签。

## 8. 判断

R14 确认：

```text
edge_or_away 不是重复样本造成的假象；
日级去重后仍是有效风险标签。
```

但它目前更适合作为：

```text
强风险诊断标签 / 候选过滤器
```

还不适合立刻固化为实盘过滤规则。

原因：

1. 日级结构仍只有 23 个；
2. DCE.m 的样本最好，但 bad 只有 1 个；
3. SR 有改善，但仍要考虑成本和 POC 兑现率；
4. rb 的 not_bad 仍为负，说明该品种需要额外左尾过滤或继续作为负面对照；
5. 目前没有做跨合约、跨月份、跨更长历史窗口验证。

## 9. 阶段结论

R14 的结论比 R13 更稳：

```text
去掉 2/3 ticks 重复结构后，
edge_or_away 仍然显著负向，
not_bad 仍显著优于全样本。
```

这说明 POC 质量标签已经从“解释变量”进入“过滤候选”的阶段。

但下一步不应马上改交易规则，而应做：

```text
R15：候选过滤器影子评估。
```

即：

```text
1. 保持实际交易信号不变；
2. 在 diagnostics / clearing 层标记 would_filter=edge_or_away；
3. 报告同时展示原始策略与影子过滤后结果；
4. 若多个品种 / 更大样本下仍稳定，再考虑是否实现为真实过滤条件。
```

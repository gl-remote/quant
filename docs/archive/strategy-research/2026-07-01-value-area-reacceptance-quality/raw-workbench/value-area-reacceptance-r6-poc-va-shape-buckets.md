## 1. 实验问题

R5 已确认：

```text
当前 POC / VA 定义具备一定解释力，
但不足以作为严格、稳定的共识价格区间定义。
```

本轮继续诊断，不调参、不改代码，重点看：

```text
1. 盈利/亏损样本在 POC 位置、POC 靠边程度、entry 深度上有什么差异；
2. POC 靠近 VA 边缘是否是风险信号；
3. 重新接受进入更深是否真的更好；
4. DCE.m 主线样本里，哪些逐笔形态更像“可回归共识区间”。
```

## 2. 数据来源与字段

使用与 R5 相同的 5m 回测样本：

```text
DCE.m2601: 401 / 402
CZCE.SR601: 403 / 404
SHFE.rb2601: 409 / 410
```

派生字段：

```text
VA width = VAH - VAL；
target_to_va = abs(target - entry) / VA width；
entry_from_edge_pct = 重新接受后 entry 从突破边界进入 VA 的比例；
poc_pct = (POC - VAL) / VA width；
poc_edge_distance_pct = min(poc_pct, 1 - poc_pct)；
```

分桶定义：

```text
POC 靠边：poc_edge_distance_pct < 0.20；
POC 中偏边：0.20 <= poc_edge_distance_pct < 0.35；
POC 居中：poc_edge_distance_pct >= 0.35。

entry 浅确认：entry_from_edge_pct < 0.10；
entry 中等确认：0.10 <= entry_from_edge_pct < 0.18；
entry 深确认：entry_from_edge_pct >= 0.18。
```

## 7. 关键发现

### 7.1 POC 靠边是明显风险信号

三个品种里，`poc_near_edge` 的表现都很差：

```text
DCE.m2601: poc_near_edge 样本全部亏损；
CZCE.SR601 / 3 ticks: poc_near_edge 3 笔全部亏损；
SHFE.rb2601: poc_near_edge 6 笔全部亏损。
```

这说明当前 `close-profile POC` 如果贴近 VA 边缘，可能并不代表“可回归共识价格”，而更像：

```text
价值区本身偏斜；
POC 与失败边界距离过远或路径过长；
单点 POC 受 close-profile 偏移影响；
重新接受后回到 POC 的短期兑现概率低。
```

### 7.2 POC 居中/不靠边时，结构明显更好

DCE.m 的 `poc_central` 样本表现最好：

```text
DCE.m2601 / 2 ticks / poc_central: 2 笔，100% 胜率，net_pnl = 3425.6；
DCE.m2601 / 3 ticks / poc_central: 1 笔，100% 胜率，net_pnl = 2581.6。
```

SR 和 rb 的居中/中偏边样本也比靠边样本好。这说明：

```text
POC 是否远离 VA 边缘，可能比单纯 reaccept ticks 更能解释样本质量。
```

### 7.3 entry 更深不一定更好

R3 里 2~3 ticks 有正反馈，但本轮按 entry 深度分桶后发现：

```text
entry_deep 并不稳定；
entry_deep 往往伴随 target_to_va 变大；
过深确认可能已经消耗 POC 回归空间。
```

这进一步支持：

```text
reaccept 深度不是独立越深越好；
它必须和 POC 位置、VA 宽度、target_to_va 一起解释。
```

### 7.4 DCE.m 的主线优势来自少数形态清晰样本

DCE.m 最关键盈利样本是：

```text
2025-10-22 long：
entry_pct = 0.077；
poc_pct = 0.513；
poc_edge_pct = 0.487；
target_to_va = 0.436；
raw_rr = 1.889；
net_pnl = 2581.6。
```

这个样本更像理想结构：

```text
重新接受刚进入 VA；
POC 接近 VA 中部；
到 POC 有中等而非过远的空间；
raw_rr 足够；
快速 hit POC。
```

相比之下，亏损样本常见形态是：

```text
POC 更靠 VA 边；
target_to_va 偏大；
虽然 raw_rr 看起来不低，但 POC 没有被兑现。
```

### 7.5 POC / VA 提供可兑现盈亏比时，结构更强

本轮进一步明确：

```text
POC / VA 的价值不是“让目标越远越好”，
而是同时给出较近失败边界、适中可兑现目标、以及足够价格原始盈亏比。
```

可以把当前阶段性理解表述为：

```text
当 POC / VA 给出一个结构合理、距离适中、可兑现的目标，
同时失败边界较近时，value_area_reacceptance 效果更强。
```

这解释了为什么 DCE.m 当前强于 SR：

```text
DCE.m 的 raw_rr 更好，且部分样本 POC 不靠边、target_to_va 适中；
SR 的成本和 POC 兑现更不稳定；
rb 即使 POC 有时居中，也不能保证方向 edge 成立。
```

因此，后续不应把 POC / VA 只当作空间过滤器，而应把它们视为结构定义本身：

```text
失败边界是否清楚；
POC 是否是可回归共识锚；
entry 到 POC 是否有足够但不过远的空间；
目标能否在短持仓窗口内兑现。
```

### 7.6 tick 敏感性可以被“POC 靠边 + target_to_va”部分解释

R3 观察到 1~2 ticks 很敏感，本轮进一步说明：

```text
如果 POC 靠边或目标距离占 VA 比例过大，
1~2 ticks 的入场差异会显著改变 target_to_va 和 raw_rr，
但这不是市场对精确 tick 的稳定响应，
而是 POC/VA 单点锚本身脆弱。
```

## 8. 对后续研究的含义

本轮支持将以下变量提升为质量标签候选：

```text
poc_edge_distance_pct；
target_to_va；
entry_from_edge_pct；
VA width；
POC 是否居中 / 靠边。
```

但当前仍不建议马上改策略过滤。更好的下一步是：

```text
1. 对 DCE.m 的关键盈利/亏损样本做图形复盘；
2. 观察 close-profile POC 是否与肉眼看到的成交/震荡中心一致；
3. 如果 POC 靠边确实经常失败，再考虑把 poc_edge_distance_pct 作为质量标签；
4. 如果 target_to_va 过大经常失败，再考虑目标从单点 POC 改成 POC band 或内部目标区。
```

## 9. 本轮结论

```text
POC/VA 形态比 fixed ticks 更接近当前结构质量的解释变量。
```

具体结论：

```text
POC 靠边是明显风险信号；
POC 居中或不靠边的样本更容易兑现；
entry 更深不一定更好，过深可能损失 POC 回归空间；
POC / VA 真正有效时，不只是提供更高 raw_rr，
而是提供“较近失败边界 + 适中可兑现目标 + 足够原始盈亏比”；
当前 2~3 ticks 正反馈更像 POC/VA 锚点脆弱下的经验补偿，
不是应继续优化的精确参数。
```

阶段建议：

```text
继续保持 5m 执行周期；
继续保留 DCE.m 为主线观察；
下一步优先做关键样本图形复盘；
在图形复盘确认前，不急于把 poc_edge_distance_pct 或 target_to_va 写成硬过滤。
```

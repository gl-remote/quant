## 1. 研究问题

本轮验证 `low-validation-cost-r1` 的趋势回踩候选区间：

```text
趋势背景下的回踩区，是否能提供客观、近距离的严格失败边界，
并允许用有限止损放宽购买足够胜率？
```

候选结构：

| 项目 | 定义 |
| --- | --- |
| 策略代号 | `lvc_pullback` |
| 候选区间 | `15m` 趋势背景下，`5m` 回踩到趋势均线附近后重新接受 |
| 多头假设 | `15m` 快均线在慢均线上方，`5m` 回踩后重新上破并收回趋势均线上方 |
| 空头假设 | `15m` 快均线在慢均线下方，`5m` 反弹后重新跌破并收回趋势均线下方 |
| 严格失败边界 | 多头为本轮回踩低点；空头为本轮反弹高点 |
| 实际止损边界 | 严格失败距离按固定 `stop_widen_multiplier` 有限放宽 |
| 目标止盈 | 按实际止损距离的 `target_r_multiple` 小目标 |
| 时间退出 | 超过 `max_hold_bars` 未触达止盈或实际止损则退出 |

## 2. 最终结论

```text
趋势回踩候选区能产生足够样本，且强重新接受能明显改善质量；
但在当前结构下，止损放宽和目标倍数调整都未能让组合稳定转正。
该方向继续微调的边际价值不高，应暂停。
```

保留的有效发现：

1. 趋势回踩不是“无信号”方向，候选区间能产生足够样本。
2. 强重新接受比“上一根 K 线突破”更有效。
3. 扩大止损不是正确优化方向，主要会放大失败交易亏损。
4. 提高目标倍数对不同合约分化明显，不具备稳定组合优势。
5. 失败交易的核心问题是入场后反向 MAE 偏大，不是目标太近或持仓太短。
6. 后续若重新研究趋势回踩，应优先改变候选区定义，而不是继续调当前参数。

## 3. 关键证据

### 3.1 样本不是问题

最初固定参数回测出现 `0` 交易，后续确认不是候选区没有信号，而是诊断字段类型导致回测异常：

```text
ValueError: Unknown format code 'f' for object of type 'str'
```

原因是 `signal.diagnostics` 中包含字符串字段，而 vn.py bridge 按 float 格式化所有诊断字段。移除非数值诊断字段后，策略正常产生交易。

候选分层统计显示，原始定义下最终候选数量足够：

| 方向 | 最终候选数 |
| --- | ---: |
| 多头 | `99` |
| 空头 | `131` |

因此本轮结论不是“趋势回踩没有样本”，而是“当前样本质量不足”。

### 3.2 止损放宽无效

对比严格止损与 `1.5x` 放宽止损：

| 变体 | 总交易记录数 | 平均胜率 | 平均单笔期望 | 合计收益率 |
| --- | ---: | ---: | ---: | ---: |
| `strict_stop` | `210` | `38.96%` | `-541.82` | `-129.57%` |
| `widen_1_5` | `223` | `40.28%` | `-654.66` | `-151.08%` |

结论：

```text
1.5 倍止损放宽只带来很小胜率提升，但显著恶化平均期望。
```

这说明当前方向不能靠“给信号更多空间”解决。

### 3.3 MAE/MFE 指向入场质量问题

MAE/MFE 诊断显示：

| 变体 | 完整交易数 | 胜率 | 平均 PnL | 平均 MAE | 平均 MFE | 亏损交易平均 MFE | 亏损交易平均 MAE |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| `strict_stop` | `104` | `40.38%` | `-455.96` | `5.91` | `4.57` | `2.16` | `8.76` |
| `widen_1_5` | `111` | `41.44%` | `-622.25` | `8.34` | `6.72` | `3.48` | `11.82` |

关键观察：

- 失败交易入场后很快出现较大反向波动；
- 亏损交易的有利波动很小，不像“差一点到目标”；
- 放宽止损会增加 MFE，但 MAE 增幅更大；
- 时间退出样本没有体现“错过大行情”。

因此问题优先不在止损倍数、目标距离或持仓时间，而在重新接受质量不足。

### 3.4 强重新接受有效但不足

强重新接受定义：

| 方向 | 原重新接受 | 强重新接受 |
| --- | --- | --- |
| 多头 | 收盘上破前一根 `5m` 高点并站回趋势均线上方 | 收盘上破整个回踩窗口的前序高点并站回趋势均线上方 |
| 空头 | 收盘跌破前一根 `5m` 低点并回到趋势均线下方 | 收盘跌破整个反弹窗口的前序低点并回到趋势均线下方 |

强确认后结果明显改善：

| 变体 | 总交易记录数 | 平均胜率 | 平均单笔期望 | 合计收益率 |
| --- | ---: | ---: | ---: | ---: |
| `widen_1_5` | `223` | `40.28%` | `-654.66` | `-151.08%` |
| `strong_reaccept` | `146` | `45.22%` | `-58.13` | `-59.23%` |

结论：

```text
强重新接受确实过滤了一部分弱信号，说明“重新接受质量”是有效改进方向；
但组合仍未整体转正，不能判定趋势回踩方向通过。
```

### 3.5 提高目标倍数未通过

在 `strong_reaccept` 基础上测试 `target_r_multiple = 1.0 / 1.2 / 1.4`：

| 变体 | 总交易记录数 | 平均胜率 | 平均单笔期望 | 合计收益率 |
| --- | ---: | ---: | ---: | ---: |
| `target_1_0` | `146` | `45.22%` | `-34.34` | `-59.23%` |
| `target_1_2` | `146` | `40.42%` | `-309.44` | `-78.55%` |
| `target_1_4` | `146` | `39.09%` | `-59.53` | `-62.94%` |

MAE/MFE 也不支持继续调高目标：

| 变体 | 完整交易数 | 胜率 | 平均 PnL | 平均 MAE | 平均 MFE | planned hit ratio |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `target_1_0` | `73` | `43.84%` | `-67.40` | `8.70` | `8.81` | `48.21%` |
| `target_1_2` | `73` | `38.36%` | `-332.05` | `9.26` | `8.92` | `44.23%` |
| `target_1_4` | `73` | `36.99%` | `-118.08` | `9.41` | `9.48` | `43.14%` |

结论：

```text
提高目标倍数没有带来组合级改善，反而降低触达率并增加 time_exit。
```

## 4. 工程发现

本轮研究过程中发现并修复/规避了几个工程问题：

1. `signal.diagnostics` 不应包含非数值字段，除非 bridge 支持类型化格式化。
2. 字符串/类别策略参数需要能被持久化，不能只依赖 `param_value REAL NOT NULL`。
3. 策略私有参数不应集中写死在通用配置 schema 中，应由具体策略参数模型负责解释。
4. `no-search` run 也应生成稳定的 report artifact；否则前端/契约测试会把缺失文件误判为生成失败。
5. 前端参数类型需要接受 `number | string | boolean | null`，不能假设所有策略参数都是数值。

## 5. 后续建议

暂停当前趋势回踩方向，不继续围绕 `lvc_pullback` 做参数微调。

下一候选区间建议切换到：

```text
前高 / 前低接受失败：
价格突破前高 / 跌破前低后无法维持，重新回到边界内，
以突破极值作为严格失败边界，验证假突破后的反向短线机会。
```

如果未来重新研究趋势回踩，建议满足以下条件之一，而不是复用本轮参数微调路径：

- 重新定义回踩区间，而不是只改变止损/目标；
- 引入更强的趋势阶段过滤，避免趋势末端回踩；
- 增加结构级失败识别，过滤入场后快速反向的样本；
- 使用更明确的成交量、波动压缩或高周期位置约束。

## 跨阶段方法论约束

以下约束由前置阶段的发现产生，适用于所有后续阶段。写入本节以避免每
个阶段各自遗漏。

- **【阶段 1 触发】必须使用 distance-matched baseline**：POC 天然回归属性
  （~0.55 baseline）远大于结构 edge。任何"vs same_direction 随机"的
  判据都会大幅高估 edge，必须叠加 vs distance_matched 判据。
- **【阶段 1.5 A/A2 触发】必须使用 ATR 归一化的距离**：ticks 是绝对单位
  跨品种不可比。所有距离档判据须以 ATR 单位（rolling 20-bar TR SMA）
  表达。
- **【阶段 1.5 A3 触发】必须使用"成本后期望净值"，不能仅用 reach_rate**：
  reach_rate 高 ≠ 期望收益高（metals 4.0+ ATR 案例：reach_rate 相对
  edge +0.04，但期望净值 -0.39 ATR）。所有策略判据须包含成本估算与
  真实交易路径模拟。
- **【阶段 1.5 §5.5.7 触发】结构 × 距离档二维联合优化**：策略定位不能
  只在最简交易结构（单一目标 + 固定 stop）下判定。至少覆盖 2-3 种止盈
  止损结构做敏感性对照。rolling vs fixed-window 对照（阶段 4）必须在
  **最优结构下**做。

## 逻辑支点链

```text
POC/VA 反映共识价格（假设 A）
  → 滚动刷新能更好捕获共识（假设 B）
    → 价格回到 VA 内部有方向信息（假设 C）
      → 重新接受深度改善风险空间（假设 D）
        → POC 目标可兑现（假设 E）
          → 跨品种普遍成立（假设 F）
```

## 阶段 1：方向信息（成本最低，Gatekeeper）

**验证目标**：价格回到 VA 内部是否有方向信息，**并识别生效边界**（哪些
品种显著、哪些不显著、边界的宏观特征是什么）。

**方法**：广度优先的事件研究。
- **取 10+ 个品种**的 5m 数据（覆盖不同板块：黑色 / 有色 / 能化 / 农产品 /
  股指等），计算前日 VA/POC（先用最简 fixed-window，不需要 rolling）
- 扫描事件：价格从 VA 外侧穿回内侧（bar close 回到 VAH/VAL 之间）
- 统计事件后 N 个 bar 内价格往 POC 方向走的比例
- 对照：same-direction 随机入场、random-direction 随机入场
- **逐品种输出胜率 + 随机基准差值**，按板块 / 波动率 / 成交量分组观察差异

**判断标准**：不看"整体是否显著"，看"生效品种集合"。
- 至少一个板块 / 品种子集在成本后显著优于随机基准
- 生效边界可描述（"什么类型的品种有信号"），而不是随机散布

**成本**：10+ 品种 × 每品种 2 分钟数据获取 + 批量跑脚本 ≈ 1 小时以内。
广度优先在这里成本几乎为零，没理由先只跑 1-2 个。

**不通过退出路径**（分档）：

fixed-window 在 jump-process 假设下本身就是最差近似（见 README §8），
因此阶段 1 只能作为"信号是否存在"的粗筛，不能作为"信号强度"的定论。

- **所有品种都无信号** → 放弃整个策略方向，不进入阶段 4
- **少数品种弱信号但无板块规律** → 保留进入阶段 4，让 rolling 有机会
  在被采样时钟糊掉的 edge 上放大信号
- **明确的板块 / 品种子集有强信号** → 阶段 4 只需在该子集上验证
  rolling 至少不劣于 fixed-window，阶段 5 时间 OOS 也在该子集上做

### 阶段 1 最终结论

**判定**：进入下一阶段，但**结论发生重大修正**（引入 distance-matched
baseline 分离 POC 天然吸引力后）。

- **核心发现**：POC 本身的价值回归属性（~0.55 到达率基线）是主要 edge
  来源；VA reacceptance 结构的独立贡献只在部分板块上约 +0.03-0.06。
  之前 vs same_direction 观察到的 +0.09 中约 2/3 是 POC 距离效应。
- **修正后的生效板块（vs distance-matched）**：
  - 最强：**metals**（+0.055）—— 之前被距离效应完全掩盖
  - 中：agri_czce、black、energy_chem（+0.027 ~ +0.029）
  - 弱：agri_dce（+0.010）
- **策略含义**：
  - 结构 edge 主要体现在"重塑距离-到达率函数"（尤其 metals 上远距离到达率被
    reacceptance 显著抬升），不是"简单加一个高 baseline"
  - 短止损（N ≤ 10）杀 edge，策略退出必须留 20-40 bar 时间预算
- **强制方法论更新**：**阶段 2-5 所有判据必须使用 vs distance-matched
  baseline**，不能只看 vs same_direction

详见 [stage1-direction-info.md](value-area-rolling-reacceptance-stage1-direction-info.md)
§4.1 - §4.2 与 §7.2。

## 阶段 1.5：POC 回归属性专项研究（阶段 1 触发的新增阶段）

**触发原因**：阶段 1 通过 distance-matched baseline 发现 POC 天然回归
属性 (~0.55) 是主要 edge 来源，VA reacceptance 结构独立贡献只有 ~+0.03
（约 1/20 量级）。在把 reacceptance 作为策略主入场信号之前，必须先独
立回答"POC 本身是不是一个稳定的、可利用的回归目标"。

**验证目标**（独立于 reacceptance 触发）：

1. **距离-到达率函数形状**：不同距离档 (0-5 / 5-15 / 15-30 / 30+ ticks)
   下 POC 到达率如何变化？是线性衰减 / 指数衰减 / 有拐点？
2. **时间尺度**：到达率随 N ∈ {5, 10, 20, 40, 80, 160} 的曲线，POC 回归
   有没有典型时间尺度？
3. **POC 触及后行为**：到 POC 后 20 bar 内价格是穿越 / 反弹 / 震荡？如果
   "触即穿"，则 POC 作为策略目标不成立
4. **POC 稳定性**：POC 跳变（新旧 POC 距离超过 X ticks）前后回归属性是否
   一致？跳变过渡期是否可交易？
5. **触发器对比**：同一距离档下，reacceptance vs 其他候选触发器（长实
   体拒绝 / 成交量脉冲 / 简单距离阈值）的到达率差异

**方法**：无条件的全 bar 扫描 + 事件研究。
- 对每个品种，扫描全部 bar，按"当前距 POC 距离桶"分组
- 输出 (distance_bucket, N) → reach_rate / hit_first_side_ratio 二维表
- POC 跳变定义：每日 POC 与前一日 POC 距离 ≥ 5 ticks
- 触发器候选筛选：至少 3 种独立触发器 vs reacceptance 对比

**判据**：
- 若 POC 回归属性稳定、可参数化 → 策略核心可以定位为"POC 目标"
- 若 POC 回归高度依赖具体品种 / 距离档 / 时段 → 需要品种特定 POC 参数
- 若 POC 触即穿或回归属性不显著 → 主题需要重新定位

**成本**：1-2 小时开发新扫描脚本 + 批量跑，产出复用阶段 1 的 CSV。

**不通过**：
- POC 触即穿或无稳定回归 → 主题需重新定位，考虑放弃"POC 目标"设定，
  改用其他退出方式（时间止盈 / VA 中位 / ATR 目标）
- POC 回归属性存在但 reacceptance 相对其他触发器无优势 → 主题降级为
  "POC 目标 + 更优触发器"，reacceptance 只是候选之一

### 阶段 1.5 最终结论

_待阶段 1.5 完成后回填。回填内容：POC 回归函数形状、时间尺度、稳定性、

## 阶段 2：POC 目标可兑现率（成本次低，可与阶段 1 合并）

**验证目标**：以 POC 为目标的到达率是否足够，**并识别到达率的品种边界**
（是否与阶段 1 的生效品种集合一致）。

**方法**：在阶段 1 的 10+ 品种事件集上追加统计。
- 从重新接受点到 POC 的 MFE
- 到达 POC 的到达率（%），输出**成本前 / 成本后**两版
- 平均到达距离 vs 目标距离
- 成本估算：单边佣金 + 滑点（保守取 1-2 tick，或按品种历史平均 spread）
- **逐品种输出**，与阶段 1 的方向信号强度并列观察

**判断标准**：**成本后**到达率仍足够高，使得目标不是虚设。
- 对阶段 1 生效品种子集：到达率明显高于其他品种，构成"方向 + 兑现"的
  双重生效边界
- 若阶段 1 生效但到达率在同品种上普遍偏低 → 提示需要换退出方式（不是
  以 POC 为目标）

**成本**：在阶段 1 脚本上加 10 行统计 + 成本参数，边际成本趋近于零。

**不通过**：所有品种成本后到达率都过低 → 目标不可靠，需要换退出方式或放弃。

## 阶段 3：重新接受深度改善（成本中等偏低，可与阶段 1 合并）

**验证目标**：在 VA 内侧更深的位置入场，是否改善风险空间，**并观察深度
改善效应在品种间是否一致**。

**方法**：在阶段 1 的 10+ 品种事件集上按深度分桶。
- 桶 1：bar close 刚好在 VA 边界（±1 tick）
- 桶 2：回到 VA 内侧 2-3 ticks
- 桶 3：回到 VA 内侧 4+ ticks
- 统计各桶的 MAE 分布和 MFE 分布
- **逐品种输出各桶指标**，观察深度效应的普遍性 vs 品种特异性
- 若某品种样本量太少无法分桶 → 单独标注，不强行合并

**判断标准**：深度增加确实降低 MAE 或提高 MFE。
- 生效品种子集内深度效应一致 → 支持"深度过滤是稳健特性"
- 深度效应仅在个别品种成立 → 提示这是品种特性而非策略普适规律，谨慎
  纳入主规则

**成本**：在阶段 1 脚本上加分桶逻辑，边际成本低。

**不通过**：入场深度过滤无效，最简配置即可（边界入场）。

## 阶段 4：Rolling vs Fixed-Window 业务对照（成本中等）

**验证目标**：同一事件口径下，rolling 锚判定的业务指标是否**至少不劣于**
fixed-window 锚。这是逻辑支点 B（"滚动刷新能更好捕获共识"）的直接检验。
**同时观察 rolling 相对 fixed-window 的优势是否有品种边界**（例如是否
只在高波动 / 高成交量品种上放大 edge）。

**方法**：横向对照，覆盖阶段 1-3 的 10+ 品种，不看"稳定度"，只看业务指标。
- 实现最简 rolling profile/POC 计算（每 bar 重算前 N 个 bar 的 volume profile）
- 用 rolling 锚重新扫一遍阶段 1 的事件（价格从 VA 外侧穿回内侧，用 bar 自身
  那一刻的 rolling VAH/VAL 判定）
- 在**同一批事件口径 + 同一批品种**上对比 rolling 锚 vs fixed-window 锚：
  - 阶段 1 指标：结构入场胜率 vs 随机基准
  - 阶段 2 指标：POC 到达率（成本后）
  - 阶段 3 指标：深度分桶下的 MAE / MFE 分布
- **逐品种输出对照差值**，识别 rolling 优势的品种边界
- 附加观察（辅助，非判据）：跳变时刻两种锚的响应差异 —— 帮助解释业务
  指标差异的来源，但不能反过来当作通过条件

**判断标准**：
- **整体判据**：rolling 锚在**至少一项**业务指标上显著优于 fixed-window，
  且在其余指标上**不劣于** fixed-window
- **边界判据**：rolling 优势集中在阶段 1 生效品种子集内 → 支持"rolling 是
  在有信号的地方放大 edge"的理论解释；若优势与生效品种正交，需要重新审视
  优势来源是否真实

不接受"稳定度更好但业务指标持平或更差"的结果 —— rolling 的价值必须体现
在业务层面，而不是"看起来更平滑"。

**成本**：需要写 rolling profile 计算逻辑，1-2 小时；对照跑本身复用阶段
1-3 脚本。

**不通过**：rolling 相对 fixed-window 无业务优势，README §8 的理论论证与
数据脱节，需要回到 §8.6 的平滑变体（B+ debouncing / B++ EWMA）或退回
feature-only 出口。

## 阶段 5：生效边界确认 + 时间样本外（成本最高）

**验证目标**：阶段 1-4 已在广度上识别生效品种子集，本阶段做**边界确认 +
时间维度 OOS**，避免品种边界与时间稳定性的耦合失败。

**方法**：品种边界扩样确认 + 时间维度 OOS。
- **品种边界扩样**：若阶段 1-4 覆盖已达 10+ 品种，本阶段可扩至 15-20 品种
  或纳入之前未覆盖的板块（例如加入贵金属 / 软商品），检验生效边界的
  可外推性
  - 生效边界稳定 → 支持策略普适性
  - 新增品种全部失败 → 生效边界是原样本内的特性，需要收缩策略适用范围
- **时间维度 OOS**：对生效品种子集，每个品种把数据切成前段（训练）/
  后段（验证），先在前段做参数选择（若有），再在后段独立验证；简化版
  可用固定比例（例如前 70% / 后 30%）。参数选择不能触碰后段数据
- 广度扫描阶段（阶段 1-4）不做时间 OOS 是有意的 —— 先确认信号存在，再
  验证信号在时间上稳健

**判断标准**：
- 品种维度：生效品种子集在扩样后仍多数（≥ 60%）成本后有信号；生效边界
  可用宏观特征（板块 / 波动率 / 成交量）描述
- 时间维度：前段生效的品种在后段方向 edge 不显著恶化（允许幅度下降但
  方向不能反转）

**成本**：数据获取快（2 分钟/合约），主要是批量跑脚本。

**不通过**：
- 品种维度不通过 → 策略适用范围过窄，降级为品种特定策略或放弃
- 品种维度通过但时间维度不通过 → 可能存在时效性 / 环境依赖，暂停并
  分析样本内 vs 样本外差异来源（是否某段行情结构变化 / 是否高波动期
  才成立）

## 执行建议

## 与 S0-S6 路线图的衔接

假设验证（本计划阶段 1-5）与工程化路径（S0-S6）是**并列关系**，不是
包含关系：假设验证决定"是否值得做工程化"，工程化路径本身不重新验证
假设。

| 本计划阶段 | 与 S0-S6 的关系 | 说明 |
|-----------|----------------|------|
| 阶段 1-3 | **S0 之前**（前置判断） | 决定是否值得写 spec；三阶段合并跑 |
| 阶段 4 | **S0 之前**（前置判断） | Rolling vs Fixed-Window 业务对照，决定 rolling 建模是否成立 |
| 阶段 5 | **对应 S4**（Stage A 结构对齐） | 跨品种 + 时间双 OOS，需要 S1-S2 工程化产物支撑 |
| — | S0-S3 | 工程化路径（Spec / Indicator / 策略实现 / Smoke test），阶段 4 通过后启动 |
| — | S5-S6 | Stage B 泛化 + 参数选择归档，阶段 5 通过后启动 |

阶段 1 不通过（完全无信号）→ 直接放弃 rolling 主题，不需要启动 S0。
阶段 4 不通过 → 走 README §8.6 平滑变体或 feature-only 出口，不进入
S1-S2 工程化。

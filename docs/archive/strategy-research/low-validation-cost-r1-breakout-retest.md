## 1. 核心问题

本轮验证：

```text
前高 / 前低突破后，首次回踩到突破边界附近，
是否能形成一个低验证成本价格区间；
若方向假设错误，是否能用较近的严格失败边界快速识别；
适度放宽止损是否能购买到足够胜率，覆盖盈亏比下降和交易成本。
```

本轮不测试普通突破，也不把突破当成预测信号。

## 2. 实验定义

| 项目 | 定义 |
|---|---|
| 候选价格区间 | 前 N 高 / 前 N 低突破后的首次回踩边界附近 |
| 传统解释 | 突破后回踩确认，原阻力变支撑 / 原支撑变阻力 |
| 结构塑形解释 | 突破边界提供清晰接受 / 失败边界，回踩入场使验证距离收缩 |
| 方向假设 | 回踩后价格重新接受突破方向，随后短距离延续 |
| 严格失败边界 | 突破价本身，即重新回到原区间内部 |
| 严格失败距离 | 入场价到突破价的距离 |
| 实际止损边界 | 严格失败距离 × `stop_loss_expansion` |
| 目标止盈 | 严格失败距离 × `take_profit_r` |
| 时间退出 | 入场后 `max_hold_bars` 根 K 线未验证成功则退出 |

多头结构：

```text
价格突破前 N 根高点
→ 不追多
→ 等首次回踩到突破价附近
→ 收盘重新接受在突破价上方
→ 做多
→ 若跌回突破价下方，方向假设失败
```

空头结构反向。

## 5. 回测收益口径

本轮排查确认：`backtest_trades.pnl` 与 `backtest_daily.net_pnl` 不是同一口径。

| 数据 | 口径 | 用途 |
|---|---|---|
| `backtest_trades.pnl` | 逐笔毛盈亏，不扣手续费和滑点 | 分析价格运动、胜负结构、单笔空间 |
| `backtest_trades.commission` | 该成交本侧手续费，open/close 各自记录 | 成本拆解 |
| `backtest_daily.net_pnl` | 日度净盈亏，包含手续费、滑点和期末持仓重估 | 真实权益变化 |
| `backtests.total_net_pnl` | `backtest_daily.net_pnl` 汇总 | 策略收益主口径 |

DCE.m2601 run 30 对账：

```text
逐笔毛盈亏合计        720.00
- 总手续费         10,193.11
- 总滑点           18,280.00
+ 期末未平仓浮盈        80.00
= 日度净盈亏合计    -27,673.11
```

长期规则：真实收益、成本后期望、盈亏平衡胜率、安全边际以 `backtest_daily.net_pnl` / `backtests.total_net_pnl` 为准；逐笔交易表的 `pnl` 只用于毛盈亏结构分析。

## 6. 结论

```text
当前 1m / 5m / 15m 前高 / 前低突破后首次回踩接受 / 失败结构未通过。
不进入 Optuna，不继续围绕 breakout_window、retest_tolerance_atr、take_profit_r 做参数搜索。
```

最终失败层级：

```text
候选区间：客观，可定义，通过；
严格失败边界：清晰，通过；
低验证成本：成立，但距离过低；
回踩接受质量：未通过，严格边界快速再触及比例过高；
止损放宽：能提升胜率，但主要是在购买噪声，不足以覆盖成本；
成本后期望：未通过。
```

后续若重启该方向，只能作为新实验：更强接受条件、更大目标空间，或不同市场结构边界。不要把当前版本作为参数优化对象继续续调。

## 7. 对后续研究的价值

1. “边界清楚”和“低验证成本”不等于结构有效；如果严格失败边界太近，止损放宽可能只是在购买噪声。
2. 首次回踩接受需要额外证明“接受质量”，仅靠收盘回到边界上方 / 下方不足。
3. 更高周期能降低严格边界触及率，但可能同步压缩相对 MFE 尾部，不能默认改善期望。
4. 小目标结构必须优先检查成本 / 平均盈利；成本占比过高时，胜率提升很难覆盖。
5. 后续更适合转向更有结构含义的边界，例如 Initial Balance、高低价值区、密集成交区边缘，而不是普通前高 / 前低。

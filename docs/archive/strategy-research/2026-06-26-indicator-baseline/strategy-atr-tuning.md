## 1. 研究目标

本轮研究不是验证“ATR 是否能产生 alpha”，而是验证：

> 以 ATR 风控承载的“趋势背景 + 触发入场”结构，是否能形成可观察、可迁移、扣除成本后仍有潜力的中低频 CTA 策略。

目标形态：

| 维度 | 目标 |
|------|------|
| 交易频率 | 单品种每周约 3~5 次进场，一个月约 12~20 次进场 |
| 持仓周期 | 平均持仓 1~2 天 |
| 成本敏感性 | 单笔平均盈亏显著大于手续费和滑点 |
| 鲁棒性 | 不依赖单品种、单窗口、单笔持仓或尖峰参数 |
| 可解释性 | 能区分改善来自入场、出场、ATR 风控、持仓释放还是成本控制 |

判断顺序：

```text
交易节奏达标 → 持仓周期达标 → 成本后仍不过度恶化 → ATR 参数搜索 → Walk-Forward
```

因此，在交易频率和持仓周期未进入可评价区间前，不把收益率或单次 Optuna 最优结果作为正期望证据。

## 2. 长期结论

ATR 当前不适合继续作为主触发 / 主 alpha 方向推进。

更准确的定位是：

```text
ATR 主触发方向：降级 / 暂停
ATR 风控模块：保留 / 复用
```

核心原因：

1. 当前问题不是优化器、trial 数或 ATR 参数范围，而是交易样本形态不达标。
2. 原始 5-AND 结构、`15m` 背景 + `5m` MACD/KDJ 结构、`15m` 背景 + `5m` KDJ 中线触发结构，都没有稳定进入目标交易频率。
3. 时间止损可以释放持仓，但不能创造稳定 alpha。
4. 平仓后冷却归零、缩短时间止损、反向信号退出，都没有把交易密度推入目标区间。
5. 当前瓶颈已从“是否出不来 / 是否再入场被挡住”收敛为“触发结构本身无法持续产生足够机会”。

因此：**不建议继续围绕 ATR 止损、止盈、移动止盈参数做搜索。** ATR 应从主线降级为后续策略的风控和波动归一模块。

## 3. 关键证据

| 阶段 | 设置 | 交易数 | 长期含义 |
|------|------|--------|----------|
| Baseline | 原始 5-AND：5m SMA + 1m/5m MACD + 1m/5m KDJ | `1 / 1 / 1` | 单笔样本不能作为正期望证据；多条件 AND 入场过严 |
| P1 | `15m` 趋势背景 + `5m` MACD/KDJ 回调触发 | `1 / 1 / 1` | 慢趋势背景能降噪，但不是频率修复器 |
| 并行搜索验证 | Bayesian search，`3 trials × 3 合约` | `1 / 1 / 1` | 工具链跑通，但交易数不足时搜索结果只验证流程 |
| P1.5 / `r7` | 加入 `time_stop_bars=48` | `3 / 3 / 1` | 时间止损能释放仓位，但不能创造稳定 alpha |
| P1.5 / `r13` | 修正 KDJ lookback，使用 `15m` 背景 + `5m` KDJ 中线触发 | `3 / 1 / 5` | 较早期改善，但仍远低于月度 `12~20` 笔/品种目标 |
| Release loop | 冷却归零、时间止损缩短、反向退出 | 总入场约 `7~8` 笔/三合约 | release tweak 不能修复当前触发结构的交易密度 |

## 4. release loop 沉淀

release tweak 指不改变核心入场 alpha，只调整持仓释放或再入场释放机制，例如：

- 平仓后冷却；
- 时间止损；
- 反向信号退出；
- 同方向再入场约束。

本轮 release loop 的长期结论：

| 假设 | 验证方式 | 结论 |
|------|----------|------|
| 交易少是因为冷却挡住再入场 | 冷却从 10 分钟降到 0 | 总入场不变，冷却不是主要瓶颈 |
| 交易少是因为持仓太久挡住新机会 | 时间止损从 48 bars 缩到 24 bars | 持仓缩短，但总入场不增，收益更差 |
| 交易少是因为反向机会出现但未及时退出 | 开启反向信号退出 | 只释放部分持仓，没有改善总交易密度 |

反向退出的含义：当前持有多单时，如果出现完整空头背景 + 空头触发，则平掉多单；当前持有空单时反之。它只改变退出时机，不直接创造新入场信号。

结论：当前 ATR 结构不是“出不来”的问题，而是“可进场的有效机会本来就不够”。

## 5. 设计沉淀

1. **ATR 不是独立 alpha**  
   ATR 更适合做止损、止盈、移动止盈、风险仓位和波动归一，不应单独承担方向发现。

2. **参数搜索必须后置**  
   交易数不足时，Optuna / Bayesian search 只会拟合噪声。先让交易节奏和持仓周期进入可评价区间，再搜索 ATR 参数。

3. **慢周期背景不是频率修复器**  
   `15m` 趋势背景能降低噪声，但不会自动增加机会。若触发层过稀疏，慢周期只会进一步压低样本。

4. **入场放宽和真实成交变多不是一回事**  
   交易频率由入场触发和出场释放共同决定。release loop 用来判断是否被持仓占用或冷却卡住，但它不能替代 alpha。

5. **失败 run 也要记录设计含义**  
   `r4` 暴露指标读取/预热问题，`r7` 证明时间止损能释放持仓，`r13` 证明密度改善但未达标，release loop 证明释放机制不是主要瓶颈。

6. **指标历史问题应框架化处理**  
   KDJ / STOCH 需要可靠的历史序列和 warmup 语义。策略层缓存只能作为临时绕法，长期应由框架提供 `indicator_history()` 或 `indicator_series()`。

## 6. 后续方向

下一步不应继续微调 ATR release 参数，而应换触发结构，保留 ATR 风控。

优先级：

1. **Donchian / 通道突破**  
   用价格突破近期高低点作为趋势启动信号，和 ATR 止损、移动止盈天然匹配。

2. **布林带突破**  
   用波动收缩后扩张捕捉启动，但需要重点防假突破。

3. **回踩均线后再启动**  
   保留趋势背景，用价格行为替代 KDJ 中线触发。

新触发结构仍沿用本轮节奏统计：

- 入场数 / 出场数；
- 平均持仓 / 最长持仓；
- 1 日内退出比例 / 超过 2 日退出比例；
- 退出原因；
- 收益、回撤、成本敏感性。

只有新触发结构接近月度 `12~20` 笔/品种后，才评估胜率、盈亏比、ATR 参数搜索和 Walk-Forward。

## 7. 暂停事项

| 事项 | 暂停原因 |
|------|----------|
| ATR 止损/止盈参数搜索 | 交易节奏未达标，搜索会过拟合 |
| 继续微调冷却、时间止损、反向退出 | release loop 已验证不能修复密度 |
| DMI / ADX 过滤 | 基础信号已经稀疏，过滤会进一步压低样本 |
| ATR 风险仓位 | 需要先确认新触发结构有足够样本和基本收益质量 |
| Walk-Forward | 当前交易数不足，不具备样本条件 |
| 多品种组合验证 | 需要先形成可评价的单品种交易节奏 |

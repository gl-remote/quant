# structural-alpha-deepening-r4-value-area-time-exit-realization

> 类型：Workbench / 结构入口深耕实验  
> 状态：第四轮完成，time_exit 与 POC 兑现效率完成初步诊断  
> 日期：2026-06-29  
> 前序：[价值区机制诊断 r3](./structural-alpha-deepening-r3-value-area-mechanism-diagnostics.md)

## 1. 本轮问题

r3 发现价值区深度重新接受后，`time_exit` 是主要退出来源之一。

本轮专门验证：

```text
在 min_reaccept_ticks = 2~3 下，
time_exit 交易到底是小亏、小赚，还是错过 POC 后回吐？
```

并检查：

```text
延长 max_hold_bars 是否能改善 POC 兑现？
还是会放大左尾风险？
```

## 2. 实验设计

固定参数：

```text

stop_widen_multiplier = 1.5

```

变量：

```text
symbol ∈ [DCE.m2601, CZCE.SR601]
min_reaccept_ticks ∈ [2, 3]
max_hold_bars ∈ [6, 12, 18, 24]
```

输出文件：

```text

project_data/research/random_baseline/value_area_deepening_r4_time_exit_summary_20260629_000723.json
```

口径说明：

```text
MFE target coverage = time_exit 前最大 MFE / 入场到前日 POC 的目标距离。
```

第一版 runner 曾用退出价反推目标距离，导致覆盖率可能大于 100%；本轮已修正为按 5m close profile 重建前一交易日 POC，使用同源 POC 目标距离估算覆盖率。

仍需注意：该 POC 重建口径用于诊断，不直接替代策略内部状态；若后续要做正式指标，应把 entry 时的 target_price 写入成交诊断。

### 3.3 DCE 判断

DCE 上不是简单“time_exit 导致利润兑现不充分”。更准确是：

```text
6 bars：过早退出，能获得小盈利，但基本没有 POC 兑现；
12 bars：允许部分交易打到 POC，是当前最优；
18/24 bars：不是改善兑现，而是放大 stop_loss / force_flat 左尾。
```

`max_hold_bars = 12` 下，time_exit 本身平均亏损，且 time_exit 前 MFE 只覆盖 POC 目标约 `22.69%`，没有证据说明大量 time_exit 曾接近 POC 后回吐。

因此 DCE 的下一步不是简单延长持仓，而是：

```text
保留 12 bars 附近；
研究 time_exit 的主动小止盈 / 分段兑现；
避免把未达 POC 的交易拖到 18/24 bars。
```

### 4.3 SR 判断

SR 与 DCE 一致支持：

```text
max_hold_bars = 12 比 6 更合理；
18/24 不是更好兑现，而是明显放大左尾风险。
```

但 SR 的 time_exit 质量整体弱于 DCE：

```text
多数 time_exit 目标覆盖率只有 12%~43%；
12 bars 下 time_exit 仅接近小赚；
延长后 time_exit 平均 pnl 转为明显负值。
```

因此 SR 不应靠延长时间等待 POC，而应考虑更主动的部分兑现或更近目标。

## 5. 横向结论

### 5.1 time_exit 不是“错过 POC 后回吐”的主证据

本轮更支持：

```text
time_exit 交易多数没有接近 POC；
它们不是“曾经到目标附近但没卖”，
而是“方向有一定 MFE，但不足以完成 POC 回归”。
```

典型目标覆盖率：

```text
DCE 12 bars: 约 22.69%
SR 12 bars: 约 31%~37%
```

因此，当前 POC 目标对部分交易偏远。

### 5.2 12 bars 是当前较合理持仓窗口

`6 bars`：

```text
退出太早，TP 几乎没有发生，更多是小赚/小亏兑现。
```

`12 bars`：

```text
允许一部分高质量交易触达 POC，整体收益和回撤最好。
```

`18/24 bars`：

```text
没有显著增加有效 POC 兑现，反而放大 stop_loss / force_flat 左尾。
```

因此不建议简单延长持仓。

### 5.3 下一步应研究“目标兑现层”，不是时间放宽

当前瓶颈不是时间太短，而是：

```text
POC 目标对部分交易太远；
但完全缩短持仓又会牺牲能触达 POC 的交易。
```

更合理方向是：

```text
保留 POC 作为完整盈利上界；
增加主动兑现层，例如 0.3~0.5 POC 距离、固定 R、或 MFE 回撤退出；
检查是否能把 time_exit 的小 MFE 转化为稳定收益，
同时保留少数能到 POC 的右侧收益。
```

## 6. 口径风险

本轮 SR 在 `max_hold_bars >= 18` 的若干回测出现 vnpy 平仓未配对警告。

因此：

```text
SR 18/24 bars 的巨亏数值只作为风险信号，暂不作为精确统计；
但“延长持仓明显恶化”的方向判断仍成立，因为收益、回撤、胜率和配对警告同时变差。
```

后续若要正式比较 18/24 bars，需要先复核 vnpy 成交配对口径或限制跨夜 / force_flat 行为。

## 7. 阶段结论

本轮结论：

```text
max_hold_bars = 12 附近仍是当前合理窗口；
time_exit 不是大量错过 POC 后回吐，而是多数交易 MFE 不足以接近 POC；
简单延长持仓到 18/24 会放大左尾风险；
调整退出策略有潜在价值，但本阶段不继续展开。
```

价值区主线继续保留：

```text
value_area_reacceptance
+ POC 空间
+ price_raw_rr 预筛
+ min_reaccept_ticks 2~3
+ max_hold_bars ≈ 12
```

## 8. 后续保留线索：退出策略深耕

本轮已经足以确认：

```text
退出层会显著影响方向 edge 的兑现质量。
```

但现阶段不继续围绕退出策略做新一轮实验，原因是当前主任务仍是确认结构入口和风险空间是否有研究价值，而不是过早进入止盈 / 时间退出优化。

后续若重新打开该方向，优先研究：

```text
partial_target_ratio = 0.3 / 0.5 / 0.7 of POC distance
take_profit_r = 0.5 / 0.75 / 1.0
MFE 回撤退出
time_exit 前小 MFE 主动兑现
```

评价重点：

```text
是否提高 time_exit 前 MFE 的兑现；
是否牺牲过多 POC 右侧收益；
成本/平均盈利是否改善；
亏损簇和最大单亏是否继续受控。
```

当前处理：

```text
退出策略深耕暂缓；
作为后续研究线索保留；
不在本阶段继续迭代。
```

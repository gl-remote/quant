## 1. 核心问题

本轮实验要回答：

```text
Initial Balance 高 / 低点作为日内共识边界
→ 边界外假突破后重新接受 / 拒绝
→ 是否能给出清晰严格失败边界
→ 是否存在可预估的短期盈利上界
→ 是否能在 2%~3% 单次账户风险约束下提供足够账户原始盈亏比
→ 主动止盈、时间退出或有限止损放宽是否改善胜率 / 盈亏比转化效率
```

本轮不回答：

```text
不做 Optuna
不做 Walk-Forward
不叠加 MACD / KDJ / 均线确认
不同时引入 VAH / VAL / POC
不为了交易数放宽 IB 定义
```

## 2. 实验定义

| 项目 | 定义 |
| --- | --- |
| 实验版本 | `structural-alpha-r1` |
| 策略代号 | `ib_reacceptance` |
| 候选共识价格区间 | 开盘后固定分钟数形成的 Initial Balance High / Low |
| 结构来源 | Auction / Market Profile / Price Action |
| 传统解释 | 早盘区间突破失败后，价格重新回到早盘接受区，边界外追单失败 |
| 结构塑形解释 | IB 边界提供市场共同观察位置；假突破极值提供严格失败边界；IB 中轴 / 对侧边界提供短期盈利上界 |
| 方向假设 | 上破 IB High 后重新收回 IB 内部做空；下破 IB Low 后重新收回 IB 内部做多 |
| 严格失败边界 | 假突破极值外加 `failure_buffer_ticks` |
| 严格失败距离 | 入场价到严格失败边界的距离 |
| 目标止盈 | IB 中轴 / IB 对侧边界 / 按 R 倍数止盈 |
| 时间退出 | 入场后 `max_hold_bars` 或日内 `force_flat_time` |
| 入场方式 | 边界外触发突破状态后，收盘价重新回到 IB 内部并满足最小突破幅度 |
| 退出方式 | 严格失败边界、主动止盈、时间退出、日内强平 |

### 2.1 多头结构

```text
开盘后形成 IB High / IB Low
→ 价格向下跌破 IB Low 至少 min_breakout_ticks
→ 价格收盘重新回到 IB Low 上方
→ 做多
→ 严格失败边界 = 假突破低点 - failure_buffer_ticks
→ 盈利上界 = IB 中轴或 IB High
```

### 2.2 空头结构

```text
开盘后形成 IB High / IB Low
→ 价格向上突破 IB High 至少 min_breakout_ticks
→ 价格收盘重新回到 IB High 下方
→ 做空
→ 严格失败边界 = 假突破高点 + failure_buffer_ticks
→ 盈利上界 = IB 中轴或 IB Low
```

### 5.4 失败层级定位

本轮不是失败在“是否存在共识边界”或“是否能定义失败边界”。这两点成立。

失败层级主要是：

```text
接受 / 拒绝质量不足
→ 盈利上界过近或 MFE 转化不足
→ 毛收益安全边际不足
→ 手续费 + 滑点吞噬全部边际
→ 跨品种不稳定
```

按 roadmap 暂停条件，对应：

1. 价格原始盈亏比不足或转化效率不足；
2. 平均账户盈利过小；
3. 成本后期望无法转正，安全边际不足；
4. 收益改善主要来自少量参数变化，未形成稳定平台。

## 7. 临时结论

当前判断：

```text
本轮 IB 高 / 低点假突破重新接受方向暂停；结构边界清晰，但成本后期望和安全边际未通过。
```

主要已确认事项：

1. 第一轮方向选用 IB 高 / 低点假突破后的重新接受 / 拒绝。

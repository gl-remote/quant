> 类型：Roadmap / 框架缺陷记录

## 1. 背景

`low-validation-cost-r1` 需要比较严格止损与放宽止损后的尾部亏损、最大连续亏损、出场原因和成本后净收益。

在复查结果时发现三个框架级问题：

1. 最大连续亏损统计曾把未闭合开仓单纳入亏损序列；
2. 回测过程中仍出现“平仓有余量未配对”告警；
3. 策略声明 `1m` 数据需求，但实验 runner / 回测入口原先按 `backtest.interval=5m` 加载和执行，零交易结果可能被误读为策略无样本。

这类问题会影响策略研究可信度，因此单独记录为框架缺陷，不放入短期实验 roadmap 正文。

## 2. Issue 1：最大连亏统计口径污染

状态：已修正并补回归测试。

### 2.1 现象

`VnpyBacktestEngine._calculate_trade_stats` 原统计逻辑遍历格式化后的全部 trade 记录。

未闭合开仓记录的 `pnl=0` 会进入非盈利分支，从而被当作亏损序列的一部分。这会导致最大连续亏损被明显放大。

在 R1 实验中，早期最大连亏曾出现异常偏大的值；修正后 `DCE.m2601` 三组结果均为 `5`。

### 2.2 影响

| 影响面 | 说明 |
|--------|------|
| 尾部亏损判断 | 最大连亏被放大，可能错误淘汰策略结构 |
| 小资金承受能力 | 连亏簇指标失真，影响仓位和暂停规则设计 |
| 止损放宽比较 | 严格止损和放宽止损的风险对比被污染 |
| 实验结论可信度 | 研究记录中的 tail-risk 指标不可直接信任 |

### 2.3 修正方向

已将统计逻辑改为：

- 只用平仓交易计算连赢连亏；
- `pnl < 0` 计入亏损；
- `pnl == 0` 不计入亏损，但会切断连亏 / 连盈序列。

已补回归测试，覆盖未闭合开仓、平盘交易、连续亏损和盈亏切换。

## 3. Issue 2：平仓有余量未配对告警

状态：已修正并通过短窗口回测验证。

### 3.1 现象

R1 回测过程中出现类似告警：

```text
平仓有余量未配对
```

即使策略暂时限制为 long-only，仍观察到配对余量告警。

### 3.2 影响

该问题可能影响：

| 影响面 | 说明 |
|--------|------|
| 开平仓配对 | closed trade 的入场价、出场价、数量可能不完整 |
| PnL 统计 | 净收益、平均盈利、平均亏损可能被错误归因 |
| 短侧统计 | 多空同时启用时，短侧结果更容易被污染 |
| exit reason | 出场原因分布可能无法可靠映射到完整交易 |
| 成本统计 | 手续费和滑点可能与配对交易不一致 |

已修正：

- `_parse_trades` 改为按开仓方向维护独立 FIFO 队列，避免多空开仓队列互相污染；
- 平多时只消耗 long open queue，平空时只消耗 short open queue；
- 补充开多 / 平多配对回归测试，确认不会误报“平仓有余量未配对”；
- R1 短窗口 `DCE.m2603 / DCE.m2605` 回测未再出现该告警。

## 4. Issue 3：策略数据需求与实验加载周期不一致

状态：已修正并通过短窗口验证。

### 4.1 现象

`LowValidationCostStrategyCore.data_requirements()` 声明需要 `1m` 周期，但 `low_validation_cost_r1.py` 原先通过 `bc.interval` 加载数据；当前 backtest 配置中 `interval=5m`。此外，即使 runner 改为加载 `1m` 数据，如果回测配置仍以 `5m` 初始化 vnpy engine，也会出现 scan 有信号但回测 0 交易。

已修正：

- R1 runner 默认加载 `1m`，并用 `1m` 覆盖本次回测配置的 interval；
- 正式 vnpy search / walk-forward 入口会读取策略 `data_requirements` 的最小周期，并在配置周期更粗时自动覆盖；
- DataFeed 内存 / 磁盘缓存命中时校验 `base_period`、required periods 和日期范围；
- 短窗口 `DCE.m2603 / DCE.m2605` 已能正常产生成交。

### 4.2 影响

| 影响面 | 说明 |
|--------|------|
| 跨品种结论 | 可能把数据周期不匹配误判为结构跨品种无效 |
| 策略实验 runner | 固定参数实验绕过正式 CLI 时，容易忽略策略 `data_requirements` |
| DataFeed 语义 | 策略声明的最小周期与外部加载的基础周期不一致时，应显式失败或自动加载所需周期 |

### 4.3 验证

- DataFeed 缓存周期校验已补回归测试；
- CLI 正式入口已完成策略最小周期覆盖；
- R1 短窗口验证显示 `DCE.m2603 / DCE.m2605` 能正常产生成交。

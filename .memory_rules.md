# 知识图谱记忆规则

> 版本: 0.2.0-dev | 更新日期: 2026-05-25

---

## 一、目的

使用 Knowledge Graph Memory 持久化记录项目的关键信息，使 AI 助手在跨会话中保持对项目结构、模块关系、问题状态的准确认知。

---

## 二、实体类型

### 2.1 Module（模块）

项目中的核心代码模块。

- **命名**: 模块路径（如 `backtest/vnpy_backtest_engine`）
- **描述**: 模块职责与核心功能
- **关键属性**: 文件路径、依赖模块、所属子系统

### 2.2 Strategy（策略）

策略核心算法与桥接器。

- **命名**: 类名（如 `MaStrategy`、`VnpyStrategyBridge`、`TqsdkStrategyBridge`）
- **描述**: 策略类型、运行模式
- **关键属性**: 所属层级（core/bridge）、依赖框架

### 2.3 Issue（问题/缺陷）

在 plan.md 或其他审计中识别的问题。

- **命名**: issue_id（如 `DEF-01`、`ARC-01`）
- **描述**: 问题摘要
- **关键属性**: 严重度（critical/high/medium/low）、状态（open/fixed）、影响文件

### 2.4 Fix（修复记录）

已完成的代码修复。

- **命名**: `fix-{issue_id}`（如 `fix-DEF-BT16`）
- **描述**: 修复方式描述
- **关键属性**: 关联问题 ID、变更文件列表、版本号

### 2.5 Config（配置项）

关键配置参数及其含义。

- **命名**: 配置路径（如 `backtest.initial_capital`、`trading.sma_short`）
- **描述**: 参数说明与默认值
- **关键属性**: 生效范围、约束条件

### 2.6 Document（文档）

项目中的关键文档。

- **命名**: 文档路径（如 `doc/architecture`、`AI_BEHAVIOR_RULES`）
- **描述**: 文档用途与目标读者
- **关键属性**: 版本号、最后更新日期

## 三、关系类型

| 关系 | 主体 → 客体 | 含义 |
|------|------------|------|
| `imports` | Module → Module | 模块导入依赖 |
| `implements` | Bridge → Core | 桥接器实现核心策略接口 |
| `depends_on` | Module → Framework | 模块依赖的外部框架 |
| `belongs_to` | Module → Subsystem | 模块所属子系统 |
| `exposes` | Subsystem → Module | 子系统对外暴露的模块 |
| `found_in` | Issue → File | 问题所在的源文件 |
| `resolved_by` | Issue → Fix | 问题被修复记录解决 |
| `tracks` | Document → Issue | 文档跟踪的问题 |
| `controls` | Config → Module | 配置项控制的模块行为 |
| `relates_to` | Document → Document | 文档间关联引用 |

## 四、写入时机

### 每次必须写入

| 场景 | 操作 |
|------|------|
| 项目首次加载 | 创建所有 Module + Strategy 实体，建立 import/depends_on 关系 |
| 发现新问题 | 创建 Issue 实体 |
| 修复问题 | 创建 Fix 实体 + resolved_by 关系 |
| 新增模块/类 | 创建对应实体并建立关系 |
| 文档变更 | 更新 Document 实体的 observations |

### 可选写入

| 场景 | 操作 |
|------|------|
| 配置参数说明 | 按需创建 Config 实体 |
| 模块间临时关系 | 仅在关系稳定后写入 |

---

## 五、命名约定

```
实体名格式:
  Module:    {子系统}/{模块名}           例: backtest/vnpy_backtest_engine
  Strategy:  {ClassName}                 例: MaStrategy
  Issue:     {issue_id}                  例: DEF-01
  Fix:       fix-{issue_id}              例: fix-BT01
  Config:    {section}.{key}             例: backtest.initial_capital
  Document:  {文件路径去除扩展名}          例: doc/architecture
```

## 六、必须跟踪的项目信息

以下实体在项目加载时应优先确保存在于知识图谱中：

### 6.1 子系统架构

```
main.py (CLI 入口 — export/backtest/tq-backtest/test/live/report)
├── strategies/       策略子系统 (框架无关)
│   ├── core/         Strategy ABC + Bar/Signal/Fill/Position 类型
│   ├── bridges/      vnpy 桥接 + 天勤 SDK 桥接
│   └── ma_strategy.py 双均线策略实现
├── backtest/         回测引擎子系统
│   ├── vnpy_backtest_engine.py  批量回测 + Walk-Forward
│   ├── tq_backtest_engine.py    天勤 GUI 回测
│   ├── data_loader.py           CSV 加载 / 窗口切分
│   └── types.py                 回测结果类型
├── data/             数据持久化子系统
│   ├── database.py   SQLite + peewee ORM
│   ├── models.py     peewee 模型 + 完整 TypedDict 体系
│   └── exporter.py   天勤 → Qlib CSV 导出
├── report/           报告子系统 (从 backtest 和 data 解耦)
│   ├── dataset_reporter.py     单数据集 JSON 报告
│   ├── comparison_reporter.py  多品种横向对比 + 排名
│   └── sql_reporter.py         SQLite 只读报告
├── common/           纯函数工具层 (零 I/O、零副作用)
│   ├── metrics.py    max_drawdown / sharpe_ratio 等指标
│   ├── stats.py      rank_by_key / summary_stats
│   └── formatting.py 百分比/浮点数格式化
└── config/           配置管理 (YAML 分层合并)
    └── config_manager.py
```

### 6.3 策略清单

| 类名 | 层级 | 文件 |
|------|------|------|
| `Strategy` (ABC) | core — 策略抽象接口 | `strategies/core/base.py` |
| `MaStrategy` | core — 双均线策略实现 | `strategies/ma_strategy.py` |
| `VnpyStrategyBridge` | bridge — vnpy 桥接 | `strategies/bridges/vnpy_bridge.py` |
| `TqsdkStrategyBridge` | bridge — 天勤 SDK 桥接 | `strategies/bridges/tqsdk_bridge.py` |

架构约束: Strategy 不依赖任何外部框架；Bridge 负责将 vnpy/tqsdk 数据转为 Bar，将 Signal 转为框架指令。

### 6.4 当前缺陷清单

来源于 `plan.md` 第三、四节。所有 Bug 已修复，以下为存量缺陷：

**存量缺陷 (10 项):**

| 编号 | 分类 | 摘要 | 影响文件 |
|------|------|------|---------|
| DEF-01 | 测试 | 覆盖率排除核心文件 (66% 非真实值) | `pyproject.toml` |
| DEF-02 | 测试 | Bridge 层零测试覆盖 (~300 行) | `strategies/bridges/` |
| DEF-03 | 测试 | 无集成/端到端测试 | `tests/` |
| DEF-04 | 策略 | 止损/止盈使用固定比例 | `strategies/ma_strategy.py` |
| DEF-05 | 策略 | 信号优先级隐式 (if/elif 顺序) | `strategies/ma_strategy.py` |
| DEF-06 | 数据 | money 字段为近似值 (close*volume) | `data/exporter.py` |
| DEF-07 | 数据 | 缺少数据完整性校验 | `backtest/data_loader.py` |
| DEF-08 | 风控 | 无实盘风控模块 | 无 |
| DEF-09 | 风控 | 无仓位风险检查 | 无 |
| DEF-10 | 运维 | 缺少行情数据源健康检查 | `strategies/bridges/tqsdk_bridge.py` |

**指标/架构缺陷 (3 项):**

| 编号 | 分类 | 摘要 |
|------|------|------|
| DEF-11 | 指标 | calc_sharpe_ratio 未扣除无风险利率 |
| DEF-12 | 架构 | context 为空时硬编码策略类型回退 |
| DEF-13 | 指标 | 缺少 Sortino/Calmar/基准对比指标 |

**架构层面发现 (5 项):**

| 编号 | 摘要 | 严重度 |
|------|------|--------|
| ARC-01 | `_InjectedStrategy` 闭包注入脆弱，并发会竞态 | 中 |
| ARC-02 | `data_loader.py` 覆盖率仅 37% | 中 |
| ARC-03 | `vnpy_backtest_engine.py` 覆盖率 11% (依赖 vnpy 运行时) | 中 |
| ARC-04 | `main.py` 678 行单一入口，可拆分 `cli/` 包 | 低 |
| ARC-05 | `sql_reporter.py` 覆盖率 19% | 低 |

# Quant Research Methodology

本 skill 定义量化策略研究的**方法论流程与方向性约束**，帮助研究决策不走偏。它是研究方向校准的参考，不是硬性门禁——具体数值和阈值应在各策略主题的实验计划中根据实际情况设定。

## 触发时机

- 设计新实验或规划研究方向时
- 评估实验结果、判断信号是否真实时
- 决定继续 / 放弃 / 切换研究方向时
- 规划跨品种扩样时
- 准备写 math spec 或工程化之前
- 需要判断"当前做到哪一步了、下一步该做什么"时

## 与其他 skill 的边界

| Skill | 管什么 | 不管什么 |
|-------|--------|---------|
| `strategy-research-framework.md` | 交易哲学、正期望来源、退出条件、长期评价标准 | 具体实验操作顺序 |
| 本 skill | **实验操作顺序、证据门槛、验证流程、扩品种规则** | 交易哲学、文档格式、代码风格、CLI 命令 |

## 1. 研究顺序：广度优先，深度在后

对具体策略假设，采用**广度优先验证**而非深度优先调参。

### 流程

```text
假设生成 → 最简规则广度扫描 → 品种边界确认 → 深度优化 → 工程化
```

### 规则

- **最简规则**：用最少的参数、最简的退出（固定止损 + 固定止盈或 POC 目标），不加质量过滤、不加复杂状态。
- **广度扫描**：在尽可能多的品种上跑最简配置，确认基本假设（方向 edge）是否普遍存在。
- **品种边界确认**：区分哪些品种有信号、哪些没有，确定策略的生效范围。
- **只在有信号范围内深入**：不在无信号的品种上浪费时间调参。
- **深度优化**：品种边界确认后，才做参数敏感度、退出优化、风控塑形。

### 禁止

- 在少数品种上深度调参后的完整策略直接去扩品种——那样的扩样几乎必然失败。

## 2. 证据门槛：随机对照

**随机入场对照是必选项，不是可选项。** 没有通过随机对照的结论不能作为"有信号"的证据。

### 规则

- 任何声称"有方向 edge"的结论，必须经过随机入场基准对照。
- 对照基准：same-direction（同方向随机入场）和 / 或 random-direction（随机方向随机入场）。
- 结构入场胜率必须显著高于随机基准，具体阈值在各策略实验计划中设定。
- 对照结果写入实验记录，作为结论的一部分。

## 3. 样本量要求

- 广度扫描阶段不设硬门槛，观察趋势即可。
- 深度验证阶段需要足够的 trades 数量，具体阈值在各策略实验计划中设定。
- 样本量不足的结论必须标注"信度不足"，不作为硬决策依据。
- 优先通过扩展品种 / 延长数据窗口来增加样本，而非放宽入口条件凑数。

## 4. 参数稳健性

广度扫描通过后，**必须**做参数敏感度测试。

### 规则

- 核心结论（方向 edge、正期望）不应随参数在合理范围内剧烈变化。
- 测试方式：在最优参数附近扫多组参数，观察结果是否稳定。
- 如果最优参数 vs 次优参数的结果差异巨大 → 过拟合风险高，暂停并回到广度扫描或放弃该方向。

## 5. 成本现实化

- 广度扫描阶段可忽略成本，判断信号是否存在。
- 深度验证阶段必须加入交易成本和滑点估算。
- 成本后仍然有正期望才可继续进入样本外验证。
- 成本后消失的信号，可降级为质量标签或放弃。

### 5.1 成本模型选择（跨品种硬约束）

- **跨品种 gatekeeper 判决必须用每合约真实成本**：单边成本 = 佣金
  （按 entry 价估）+ 滑点（`size × tick × slip_tick`），按每笔
  entry_atr 换算为 ATR，随事件/合约/波动率变化。查表源
  `workspace/common/contract_specs.py`。
- **扁平 ATR 成本模型（如 0.05 ATR/单边）已知会跨品种系统性低估**：
  5m 期货实测扁平 0.05 vs 真实平均 0.225 单边，**低估 4.5 倍**；
  跨品种跨度从豆油 0.043 到螺纹 0.405（9 倍差）。仅可用于快速原型
  / 历史 baseline 对比 / debug，**不得作为最终判决口径**。
- **paired diff 判据在两种成本模型下等价**（同 event 双向抵消），
  但**绝对 mean 判决必须走真实成本**——扁平成本下 mean 翻正的结论
  在真实成本下常常消失（见 structural-shaping-alpha 主题 §8.7 KF-5）。
- Runner 层面：所有跨品种 no-signal / gatekeeper 类脚本，默认启用
  真实成本，扁平模式必须显式命名为 `--flat-cost-debug` 类开关。

### 5.2 显著性诊断护栏（避免 alpha 误判）

- **"少输"型 paired 显著性 ≠ 独立 alpha**：如观察到 paired diff CI
  排除 0 但 combo mean 仍<0，先做"绝对损益尺度归因"复核——紧止损/短
  距离档天然会缩小每笔绝对损益方差，让 paired diff CI 变窄，但这不
  改变期望方向，只是"更少的负数"（见 KF-4）。**必须做二维拆分**
  （单独变距离 vs 单独变时间/其他因子），确认显著性不是尺度伪影。
- **"1 ATR 强回归带"引发的 SCALE 稳健性对照，必须与 realistic-cost
  对照双向配套**——两条一起才干净，任一单独走都可能得到错判决。

## 6. 样本外验证

**需要品种维度和时间维度两个方向的样本外验证。**

### 品种维度

- 训练组品种上开发 / 调参，验证组品种上测试。
- 验证组品种不应参与任何参数选择。
- 验证组多数品种有方向 edge 或正收益才算通过。

### 时间维度

- 前段数据开发 / 调参，后段数据测试。
- 简单方案：固定比例切分（如前段训练、后段验证）。
- 进阶方案：Walk-Forward 滚动窗口。
- 时间样本外的方向 edge / 正收益不显著恶化才算通过。

### 判定

- 两个维度都通过 → 可进入模拟盘。
- 品种维度通过但时间维度不通过 → 策略可能有时效性或环境依赖，暂停。
- 品种维度不通过 → 策略逻辑本身有问题，放弃或回到假设生成。

## 7. 切换 / 降级决策

### 放弃条件

- 广度扫描后多数品种无信号。
- 随机对照未通过（结构入场不显著高于随机）。
- 成本后为负且无改善空间。

### 继续条件

- 有方向 edge 但退出兑现差 → 优化退出方向。
- 有方向 edge 但参数敏感 → 回到更简配置或放弃复杂度。

### 降级条件

- 不能作为独立策略但有信息价值 → 降为 feature / 质量标签。
- 降级后的 feature 仍应在后续实验中检验是否确实改善主策略。

## 8. 工程化时机

- 少数品种有信号时 → 不写完整 math spec，只写核心假设描述。
- 广度扫描通过（多数品种有信号）→ 可以开始写 math spec。
- 参数稳健性 + 成本现实化通过 → 可以开始实现工程化。
- 样本外双维度通过 → 可以进入模拟盘设计。

- spec 的精简程度应匹配验证阶段：广度扫描阶段只需简短核心定义，不需要完整候选矩阵。
- 工程化投入（indicator 抽取、框架改造等）只在样本外验证通过后才值得做。

## 9. 数据边界不可造假

**核心原则**：期货合约寿命有限、每个合约是独立市场——真实独立样本量往往
远小于表面 event 数。**承认它、诚实报告它、不要用"看似合理"的方法造精度**。

### 9.1 常见诱惑与陷阱

数据看起来不够时，容易想到的"补精度"手段，其实都在**破坏信号结构**：

| ❌ 错误做法 | 为什么破坏信号 |
|-----------|-------------|
| 跨合约池化 rank（合并全品种做分位）| 不同品种的尾部分布差异大（p05/p95 极差可达 0.5-0.7）· 触发阈值失去品种特异性 |
| 跨品种前缀池化（e.g. 合并 rb2410/2510/2601）| 即使同品种、不同月份的市场行为也不同（换月效应 / 波动率制度）· 空头等极端触发首先失效 |
| 用久远合约历史外推当前合约 | 每个合约都是独立市场 · 相隔一年的 rb2401 vs rb2601 不能视为同一分布 |
| Bayes shrinkage 用"分布不同"的数据做先验 | 引入系统性偏差 · 尾部信号被稀释 |
| Rolling window 超过合约寿命 | 无数据可用 · 相当于隐性地用别的合约填 |
| 用 event 数（1h/5m）当独立观察数 | 同日 event 共享日级特征（skew / ATR / trend）· event 数虚高 5-10 倍 |

### 9.2 正确的应对手段

**承认数据边界**：

- **诚实报告独立观察数**：区分"event 数"和"独立日 / 独立合约日"——报告时并列，
  例如 "n_events=142 · n_indep_days≈14"
- **正确 cluster bootstrap**：cluster 单位应对应真实独立单位（通常是
  `(contract, date)` 而非 `contract`）· 让 CI 反映真实不确定性
- **依赖对样本鲁棒的指标**：**mean / hit / payoff** 优先于 p 值 · 单笔 IR
  优于年化 Sharpe（后者对独立样本假设很敏感）
- **CI 上限保守决策**：小样本决策时不用 mean · 用 CI 下限（多头）或
  CI 上限（空头）· 避免过度自信
- **实盘偏差归因**：若实盘与预期偏差 · 优先假设是采样噪声 · 而不是"策略失效"

### 9.3 数据不够时可以做的

- **扩展合约月份**（同品种 · 每个合约独立）· 每个合约独立算 rank · **不池化**
- **扩展品种范围**（如果假设跨品种普适）· 每品种独立验证 · 报告"品种保留率"
- **延长历史**（如果合约寿命允许）· 但不外推到不同合约
- **降低时间粒度**（如果小时够就不要坚持 5m）· 让 event 时钟本身独立
- **接受"结果就是这样"**：拒绝"再挖挖看能不能更好"的诱惑

### 9.4 判别应用场景

**触发场景**：
- 每合约 rolling 分位样本量 <100 独立日
- 广度扫描发现 mean 显著但 CI 很宽
- 深度验证时想用"更多数据"改善显著性
- 需要写 math spec 前判断样本量约束

**判别流程**：
1. **诊断跨合约分布差异**：KS 检验 + 尾部分位（p05/p95）比较
2. **若中心相似、尾部差异明显**：意味着**触发依赖尾部的信号不能池化**
3. **若真的需要更多数据**：只能扩合约 · 扩品种 · 或降低复杂度
4. **反证实验**：如果确实想池化 · 先做"池化 vs per-contract"的严格对比
   （见 poc-value-area-asymmetry stage3 §12.12 · KF-22）· 若池化后核心
   信号降级 · 立即放弃池化

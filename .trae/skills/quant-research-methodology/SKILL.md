# Quant Research Methodology

本 skill 定义量化策略研究的**方法论流程与方向性约束**，帮助研究决策不走偏。它是研究方向校准的参考，不是硬性门禁——具体数值和阈值应在各策略主题的实验计划中根据实际情况设定。

## 触发时机

- 设计新实验或规划研究方向时
- 评估实验结果、判断信号是否真实时
- 决定继续 / 放弃 / 切换研究方向时
- 规划跨品种扩样时
- 准备写 math spec 或工程化之前
- 需要判断"当前做到哪一步了、下一步该做什么"时

## 1. 研究顺序：广度优先，深度在后

对具体策略假设，采用**广度优先验证**而非深度优先调参。

### 流程

```text
假设生成 → 最简规则广度扫描 → 品种边界确认 → 深度优化 → 工程化
```

### 规则

- **最简规则**：用最少的参数、最简的退出（固定止损 + 固定止盈或 POC 目标），不加质量过滤、不加复杂状态。
- **广度扫描**：在尽可能多的品种上跑最简配置，确认基本假设（方向 edge）是否普遍存在。
- **品种边界确认**：区分哪些品种有信号、哪些没有，确定策略的生效范围。
- **只在有信号范围内深入**：不在无信号的品种上浪费时间调参。
- **深度优化**：品种边界确认后，才做参数敏感度、退出优化、风控塑形。

### 禁止

- 在少数品种上深度调参后的完整策略直接去扩品种——那样的扩样几乎必然失败。

## 2. 证据门槛：随机对照

**随机入场对照是必选项，不是可选项。** 没有通过随机对照的结论不能作为"有信号"的证据。

### 规则

- 任何声称"有方向 edge"的结论，必须经过随机入场基准对照。
- 对照基准：same-direction（同方向随机入场）和 / 或 random-direction（随机方向随机入场）。
- 结构入场胜率必须显著高于随机基准，具体阈值在各策略实验计划中设定。
- 对照结果写入实验记录，作为结论的一部分。

## 3. 样本量要求

- 广度扫描阶段不设硬门槛，观察趋势即可。
- 深度验证阶段需要足够的 trades 数量，具体阈值在各策略实验计划中设定。
- 样本量不足的结论必须标注"信度不足"，不作为硬决策依据。
- 优先通过扩展品种 / 延长数据窗口来增加样本，而非放宽入口条件凑数。

## 4. 参数稳健性

广度扫描通过后，**必须**做参数敏感度测试。

### 规则

- 核心结论（方向 edge、正期望）不应随参数在合理范围内剧烈变化。
- 测试方式：在最优参数附近扫多组参数，观察结果是否稳定。
- 如果最优参数 vs 次优参数的结果差异巨大 → 过拟合风险高，暂停并回到广度扫描或放弃该方向。

## 5. 成本现实化

- 广度扫描阶段可忽略成本，判断信号是否存在。
- 深度验证阶段必须加入交易成本和滑点估算。
- 成本后仍然有正期望才可继续进入样本外验证。
- 成本后消失的信号，可降级为质量标签或放弃。

## 6. 样本外验证

**需要品种维度和时间维度两个方向的样本外验证。**

### 品种维度

- 训练组品种上开发 / 调参，验证组品种上测试。
- 验证组品种不应参与任何参数选择。
- 验证组多数品种有方向 edge 或正收益才算通过。

### 时间维度

- 前段数据开发 / 调参，后段数据测试。
- 简单方案：固定比例切分（如前段训练、后段验证）。
- 进阶方案：Walk-Forward 滚动窗口。
- 时间样本外的方向 edge / 正收益不显著恶化才算通过。

### 判定

- 两个维度都通过 → 可进入模拟盘。
- 品种维度通过但时间维度不通过 → 策略可能有时效性或环境依赖，暂停。
- 品种维度不通过 → 策略逻辑本身有问题，放弃或回到假设生成。

## 7. 切换 / 降级决策

### 放弃条件

- 广度扫描后多数品种无信号。
- 随机对照未通过（结构入场不显著高于随机）。
- 成本后为负且无改善空间。

### 继续条件

- 有方向 edge 但退出兑现差 → 优化退出方向。
- 有方向 edge 但参数敏感 → 回到更简配置或放弃复杂度。

### 降级条件

- 不能作为独立策略但有信息价值 → 降为 feature / 质量标签。
- 降级后的 feature 仍应在后续实验中检验是否确实改善主策略。

## 8. 工程化时机

- 少数品种有信号时 → 不写完整 math spec，只写核心假设描述。
- 广度扫描通过（多数品种有信号）→ 可以开始写 math spec。
- 参数稳健性 + 成本现实化通过 → 可以开始实现工程化。
- 样本外双维度通过 → 可以进入模拟盘设计。

- spec 的精简程度应匹配验证阶段：广度扫描阶段只需简短核心定义，不需要完整候选矩阵。
- 工程化投入（indicator 抽取、框架改造等）只在样本外验证通过后才值得做。
